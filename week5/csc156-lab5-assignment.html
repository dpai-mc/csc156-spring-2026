✨ Lab Overview & Objectives

Welcome to Lab Activity 5! This lab is your opportunity to apply the theoretical knowledge gained in Week 5's modules on file systems, deleted file recovery, and file carving. You will work with a provided disk image to identify various file systems, recover deleted data using both metadata-based techniques and file carving, and thoroughly document your findings.

🎓
CSLO Alignment: This lab directly advances **CSLO 2: Conduct thorough examinations of computer hard disk drives, and other electronic data storage media to recover information and electronic data.**

**Course Objective Alignment:** This lab directly addresses the course objective: *"Analyze file systems and recover deleted files."*

💡
Required Tools: Ensure you have **Autopsy**, **The Sleuth Kit (TSK) command-line tools** (mmls, fsstat, fls, istat, icat), **Scalpel** (or PhotoRec), and **MFTECmd** installed and ready in your forensic workstation environment. A hex editor is also highly recommended.

Total Points: 20 points

This lab is divided into three parts, each contributing to your overall score.

Part 1: File System Identification (6 points)

In this section, you will examine the provided disk image to identify the various partitions and the file systems present on them. Proper identification is the first critical step in any forensic analysis.

Tasks:

  1. Examine multiple partitions within the provided disk image.
    • Use mmls (TSK) to identify partition layout.
    • Use Autopsy's data source manager to view partitions.
  2. Identify the file system type (e.g., FAT32, NTFS, ext4, HFS+, APFS) for each relevant partition.
    • Use fsstat (TSK) and Autopsy's file system information views.
  3. For **at least three distinct file systems** identified, document the following key characteristics:
    • File System Type (e.g., NTFS, FAT32, ext4)
    • Volume Label (if present)
    • Total Size
    • Block/Cluster Size
    • Total Number of Files and Directories
    • Journaling Status (e.g., enabled/disabled, type of journal for ext4)
    • Any other notable features relevant to forensics (e.g., NTFS version, HFS+ Journaled)
  4. Provide **screenshots** as evidence for each identified file system, clearly showing the tool output and the documented characteristics.
🔍
Tip: Pay attention to the output of mmls for partition offsets, which are crucial for running subsequent TSK commands like fsstat. For example, if mmls shows a partition starting at sector 2048, you would use fsstat -o 2048 image.dd.

Part 2: Deleted File Recovery (8 points)

This section focuses on recovering logically deleted files using metadata-based techniques for different file systems. You will utilize both command-line and GUI tools.

Tasks:

1. FAT32 Volume:

  1. Identify a FAT32 partition within your disk image (from Part 1).
  2. List deleted files using fls -d -r -o [partition_offset] [image_file].
  3. Select **at least two deleted files** and attempt to recover them using icat -o [partition_offset] [image_file] [inode_number] > [output_filename].
  4. Document the recovery success/failure for each file, explaining the reasons. Note any filename character loss and how you handled it.
  5. Provide screenshots of the fls output showing deleted files and the successful recovery of at least one file.

2. NTFS Volume:

  1. Identify an NTFS partition.
  2. Add the NTFS partition as a data source in **Autopsy**. Ensure the "Deleted File Finder" and "File System Analysis" ingest modules are enabled.
  3. Navigate to the "Deleted Files" section in Autopsy and select **at least two deleted files** for recovery. Export these files.
  4. For one of the recovered NTFS files, compare its **$STANDARD_INFORMATION** and **$FILE_NAME** timestamps. Discuss any discrepancies and their forensic implications (e.g., signs of timestomping). Provide screenshots of the timestamp comparison.
  5. Attempt to identify any **Alternate Data Streams (ADS)** on a file (recovered or active) within the NTFS volume using Autopsy or a relevant TSK command. Document your findings.
  6. Export and analyze **$UsnJrnl entries** related to a deleted file (or any significant file activity) from the NTFS volume using **MFTECmd**. Document how this journal entry provides historical context.
  7. Provide screenshots of Autopsy's deleted file view, timestamp comparison, ADS identification, and MFTECmd output.

3. ext4 Volume (if provided in your disk image):

  1. Identify an ext4 partition.
  2. Examine the inode table for deleted files using fls -d and istat.
  3. Attempt to recover **at least one deleted file** using extundelete or a similar specialized tool.
  4. Document the challenges specific to ext4 recovery (e.g., cleared block pointers, filename loss) and whether your recovery was successful.
  5. Provide screenshots of your attempts and results.
⚠️
Important: Remember that working directly on the original disk image with tools like extundelete can modify it. Always work on a **copy** of your forensic image if using tools that write changes.

Part 3: File Carving Exercise (6 points)

In this section, you will practice file carving techniques to recover files from unallocated space or where file system metadata is no longer available.

Tasks:

  1. Configure **Scalpel** (or Autopsy's carving module if preferred).
    • Create or modify a scalpel.conf file to carve for the specified file types.
    • If using Autopsy, ensure the "File Carver" ingest module is enabled and configured for the relevant file types.
  2. Carve the provided disk image for the following file types:
    • JPEG images
    • PDF documents
    • Microsoft Office documents (DOCX, XLSX)
    • ZIP archives
  3. Review the carved files in your output directory (or Autopsy's "Carved Files" section). For **at least 3-5 unique carved files** (across different types):
    • Identify if they are **valid files** (open correctly) or **false positives**.
    • Determine if the recovery is **complete** or **partial** (e.g., due to fragmentation).
    • Document any observed **fragmentation effects** (e.g., extra data at end, corruption).
  4. Document your carving methodology (tool used, configuration file/settings).
  5. Briefly compare the results of file carving to your metadata-based recovery in Part 2. Discuss when each method is most appropriate.
  6. Provide screenshots of your carving configuration (e.g., scalpel.conf or Autopsy module settings) and examples of your reviewed carved files.
🛠️
Scalpel Configuration Example: 
# scalpel.conf snippet
# extension  case_sensitive  max_size  header                      [footer]
jpg          y               20000000  \xff\xd8\xff\xe0\x00\x10    \xff\xd9
pdf          y               50000000  %PDF                        %%EOF
docx         y               50000000  PK\x03\x04                  \x50\x4b\x05\x06
zip          y               100000000 PK\x03\x04                  \x50\x4b\x05\x06
Remember that DOCX (and XLSX, PPTX) files are essentially ZIP archives, so they share the same header and footer signatures.
