Overview

The Sleuth Kit (TSK) is an open-source collection of command-line tools for digital forensic analysis. Originally developed by Brian Carrier, TSK allows forensic investigators to examine disk images and recover evidence from file systems. It serves as the foundation for the popular Autopsy forensic browser.

📘 Learning Connection

This guide supports CSLO 4: Apply file system analysis techniques to examine storage media and recover digital evidence. TSK tools are referenced in NIST SP 800-86 as recommended forensic software for file system analysis.

Key Capabilities

  • Volume System Analysis - Examine partition layouts and volume structures
  • File System Analysis - Analyze FAT, NTFS, ext2/3/4, HFS+, and other file systems
  • Deleted File Recovery - Locate and recover deleted files and directories
  • Metadata Examination - Extract detailed file metadata including timestamps
  • Data Extraction - Extract file content by inode or block number

🔍 Non-Invasive

TSK operates in read-only mode, preserving the integrity of evidence during analysis.

📊 Comprehensive

Supports multiple file systems and provides detailed metadata extraction capabilities.

⚙️ Scriptable

Command-line interface enables automation and integration with other forensic tools.

Installation

Linux Installation

Ubuntu/Debian (Package Manager)

Bash 
# Update package list
sudo apt update

# Install The Sleuth Kit
sudo apt install sleuthkit

# Verify installation
mmls -V

From Source (Latest Version)

Bash 
# Install dependencies
sudo apt install build-essential automake libtool libafflib-dev libewf-dev zlib1g-dev

# Download and extract source
wget https://github.com/sleuthkit/sleuthkit/releases/download/sleuthkit-4.12.1/sleuthkit-4.12.1.tar.gz
tar -xzf sleuthkit-4.12.1.tar.gz
cd sleuthkit-4.12.1

# Configure and build
./configure
make
sudo make install

# Update library cache
sudo ldconfig
💡 Tip

The package manager version may not be the latest. For the newest features, compile from source or use the SIFT Workstation.

Windows Installation

Pre-compiled Binaries

  1. Visit the official download page: sleuthkit.org/sleuthkit/download.php
  2. Download the Windows ZIP package (e.g., sleuthkit-4.12.1-win32.zip)
  3. Extract to a folder (e.g., C:\Tools\sleuthkit)
  4. Add the bin folder to your system PATH

Adding to System PATH

PowerShell (Administrator) 
# Add TSK to PATH for current session
$env:PATH += ";C:\Tools\sleuthkit\bin"

# Verify installation
mmls -V

# To make permanent, add to system environment variables:
# System Properties > Advanced > Environment Variables > Path > Edit
⚠️ Note

Windows Defender may flag forensic tools. You may need to add an exclusion for the TSK folder.

Core TSK Tools

The following five tools form the foundation of file system analysis with The Sleuth Kit. Each tool serves a specific purpose in the forensic examination workflow.

Summary of core TSK command-line tools
Tool Function Primary Use
mmls Display partition layout Identify partitions and calculate offsets
fsstat File system statistics Examine file system metadata and configuration
fls List files and directories Browse file system contents including deleted files
istat Display inode/MFT entry details Examine file metadata and timestamps
icat Extract file by inode number Recover file content from disk image

mmls displays the partition layout of a disk image. It's typically the first tool used when examining a disk image to identify partitions and their sector offsets.

Syntax

Bash 
mmls [options] image_file

Common Options

  • -t type - Specify partition type (dos, gpt, mac, sun)
  • -o offset - Sector offset to volume system
  • -B - Display sizes in bytes
  • -r - Recurse into nested partition tables
  • -V - Display version

Example Usage

Bash 
# List partitions in a disk image
mmls evidence.dd

# Example output:
DOS Partition Table
Offset Sector: 0
Units are in 512-byte sectors

      Slot      Start        End          Length       Description
000:  Meta      0000000000   0000000000   0000000001   Primary Table (#0)
001:  -------   0000000000   0000002047   0000002048   Unallocated
002:  000:000   0000002048   0001026047   0001024000   NTFS / exFAT (0x07)
003:  000:001   0001026048   0002097151   0001071104   Linux (0x83)
🔑 Key Concept

The Start sector value is crucial—it becomes the -o (offset) parameter for other TSK tools when analyzing specific partitions.

fsstat displays detailed information about a file system, including type, size, cluster configuration, and metadata structures.

Syntax

Bash 
fsstat [options] -o offset image_file

Common Options

  • -o offset - Sector offset to file system (required for partitioned images)
  • -f fstype - Force file system type (ntfs, fat, ext, etc.)
  • -i imgtype - Image format type (raw, ewf, aff)

Example Usage

Bash 
# Get file system statistics (using offset from mmls)
fsstat -o 2048 evidence.dd

# Example output for NTFS:

--------------------------------------------

Volume Serial Number: 1234567890ABCDEF
OEM Name: NTFS    
Volume Name: Evidence

--------------------------------------------

--------------------------------------------

📘 Forensic Value

The cluster size is important for understanding slack space. The MFT location helps identify where file metadata is stored.

fls lists files and directories in a file system, including deleted entries. This is essential for identifying files of interest and locating deleted files for recovery.

Syntax

Bash 
fls [options] -o offset image_file [inode]

Common Options

  • -o offset - Sector offset to file system
  • -r - Recurse into subdirectories
  • -d - Show only deleted entries
  • -u - Show only undeleted entries
  • -p - Display full path
  • -m prefix - Output in mactime format (for timeline)
  • -l - Long listing (more details)

Output Format

Each line shows: type/name inode: filename

  • r/r - Regular file (allocated)
  • d/d - Directory (allocated)
  • r/r * - Regular file (deleted) - asterisk indicates deleted
  • d/d * - Directory (deleted)

Example Usage

Bash 
# List root directory contents
fls -o 2048 evidence.dd

# Example output:
r/r 4-128-3:    $AttrDef
r/r 8-128-2:    $BadClus
r/r 8-128-1:    $BadClus:$Bad
d/d 11-144-4:   Documents
d/d 25-144-4:   Users
r/r * 67-128-1: deleted_file.docx

# Recursively list all files with full paths
fls -o 2048 -r -p evidence.dd

# List only deleted files
fls -o 2048 -r -d evidence.dd

# List contents of specific directory (inode 25)
fls -o 2048 evidence.dd 25
💡 Pro Tip

The inode number (e.g., 67-128-1) is used with istat to view metadata and icat to extract file contents.

istat displays detailed metadata about a specific file using its inode (or MFT entry) number. This includes timestamps, size, permissions, and data block locations.

Syntax

Bash 
istat [options] -o offset image_file inode

Common Options

  • -o offset - Sector offset to file system
  • -f fstype - Force file system type
  • -B - Display sizes in bytes
  • -s seconds - Display times in seconds since epoch

Modified, Accessed, Changed (MFT), Born (Created) - These timestamps are critical evidence for establishing file activity timelines.

icat extracts the contents of a file using its inode number. This is essential for recovering deleted files or extracting files without mounting the image.

Syntax

Bash 
icat [options] -o offset image_file inode > output_file

Common Options

  • -o offset - Sector offset to file system
  • -f fstype - Force file system type
  • -r - Recover deleted file (use slack space)
  • -s - Display slack space only

Example Usage

Bash 
# Extract file to current directory
icat -o 2048 evidence.dd 67 > recovered_document.docx

# Extract deleted file (with recovery mode)
icat -o 2048 -r evidence.dd 67 > recovered_deleted.docx

# Calculate hash of extracted file
icat -o 2048 evidence.dd 67 | md5sum
icat -o 2048 evidence.dd 67 | sha256sum

# Extract and immediately view text file
icat -o 2048 evidence.dd 42 | less

# Extract file and check file type
icat -o 2048 evidence.dd 67 > unknown_file
file unknown_file
⚠️ Important

Always hash extracted files immediately to document their integrity. Use redirection (>) to save output—without it, binary content displays in the terminal.

Forensic Analysis Workflow

The following workflow demonstrates the typical sequence of TSK commands used during a file system forensic examination.

Identify Partitions

Use mmls to examine the disk image and identify all partitions. Note the starting sector offset for each partition.

Bash 
mmls evidence.dd

Examine File System

Use fsstat with the partition offset to understand the file system structure, type, and configuration.

Bash 
fsstat -o 2048 evidence.dd

List Files and Directories

Use fls to browse the file system, identify files of interest, and locate deleted files.

Bash 
# List all files recursively
fls -o 2048 -r -p evidence.dd

# Find deleted files only
fls -o 2048 -r -d evidence.dd

Examine File Metadata

Use istat to view detailed metadata for files of interest, including timestamps and data locations.

Bash 
istat -o 2048 evidence.dd 67

Extract Evidence Files

Use icat to extract files for further analysis. Hash all extracted files to document integrity.

Bash 
# Extract file
icat -o 2048 evidence.dd 67 > recovered_file.docx

# Generate hash
sha256sum recovered_file.docx > recovered_file.docx.sha256

Practical Examples

📘 Scenario Context

These examples use a hypothetical intellectual property theft investigation where a suspect's workstation containing an NTFS partition has been imaged.

Example 1: Complete Analysis of a Disk Image

Bash - Complete Workflow 
# Step 1: Identify partitions
mmls suspect_workstation.dd

# Output shows NTFS partition starting at sector 2048

# Step 2: Get file system details
fsstat -o 2048 suspect_workstation.dd

# Step 3: List files in the Projects folder (assume inode 150)
fls -o 2048 suspect_workstation.dd 150

# Step 4: Find all deleted files
fls -o 2048 -r -d -p suspect_workstation.dd > deleted_files.txt

# Step 5: Examine a suspicious deleted file
istat -o 2048 suspect_workstation.dd 67

# Step 6: Recover the deleted file
icat -o 2048 -r suspect_workstation.dd 67 > recovered_project.docx
sha256sum recovered_project.docx

Example 2: Generate Timeline Data

Bash - Timeline Generation 
# Generate body file for timeline (mactime format)
fls -o 2048 -r -m "C:" suspect_workstation.dd > timeline_body.txt

# Create timeline with mactime (part of TSK)
mactime -b timeline_body.txt -d > timeline.csv

# Filter timeline for specific date range
mactime -b timeline_body.txt -d 2026-01-01..2026-01-31 > january_timeline.csv

Example 3: Batch File Extraction

Bash - Batch Recovery Script 
#!/bin/bash
# Script to extract all deleted files from a partition

IMAGE="suspect_workstation.dd"
OFFSET=2048
OUTPUT_DIR="./recovered_files"

mkdir -p "$OUTPUT_DIR"

# Get list of deleted files with inodes
fls -o $OFFSET -r -d "$IMAGE" | while read line; do
    # Extract inode number (first field before colon)
    inode=$(echo "$line" | grep -oP '\d+-\d+-\d+' | head -1)
    # Extract filename
    filename=$(echo "$line" | awk -F: '{print $2}' | xargs)
    
    if [ ! -z "$inode" ]; then
        echo "Recovering: $filename (inode: $inode)"
        icat -o $OFFSET -r "$IMAGE" "$inode" > "$OUTPUT_DIR/${inode}_${filename}"
    fi
done

# Hash all recovered files
cd "$OUTPUT_DIR"
sha256sum * > ../recovered_files_hashes.txt
