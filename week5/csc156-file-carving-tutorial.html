Overview

File carving and MFT analysis are essential techniques in digital forensics for recovering deleted files and examining file system metadata. This guide covers three powerful tools that complement The Sleuth Kit in your forensic toolkit.

📘 Learning Connection

These tools support CSLO 4: Apply file system analysis techniques to examine storage media and recover digital evidence. As referenced in NIST SP 800-86, analysts should have a forensic toolkit containing various tools that provide the ability to perform quick reviews of data as well as in-depth analysis.

🔪

Scalpel

High-performance file carver

  • Header/footer based carving
  • Highly configurable signatures
  • Fast processing speed
  • Minimal dependencies
📷

PhotoRec

Multi-format file recovery

  • 500+ file format support
  • Cross-platform compatibility
  • Interactive and command-line modes
  • Read-only operation
📊

MFTECmd

NTFS MFT parser

  • Complete MFT parsing
  • CSV/JSON output formats
  • $I30 and $J parsing
  • Timeline-friendly output

When to Use Each Tool

Tool selection guide based on forensic scenario
Scenario Recommended Tool Rationale
Recover specific file types with known signatures Scalpel Configurable signatures, fast targeted recovery
Broad recovery of multiple file types PhotoRec Extensive format support, minimal configuration
NTFS file system metadata analysis MFTECmd Complete MFT parsing, timestamp analysis
Unallocated space carving Scalpel / PhotoRec Both excel at carving from raw data
Timeline creation from NTFS MFTECmd CSV output integrates with timeline tools

Scalpel

Scalpel is a fast, filesystem-independent file carving tool originally based on Foremost. It reads a database of header and footer definitions and extracts matching files from a disk image or raw device. Scalpel is noted for its speed and configurability.

💡 Key Advantage

Scalpel's strength lies in its highly configurable signature file, allowing forensic examiners to define custom file types and fine-tune carving parameters for specific investigations.

Installation

Ubuntu/Debian Installation

Bash 
# Update package list
sudo apt update

# Install Scalpel
sudo apt install scalpel

# Verify installation
scalpel -V

# Locate configuration file
ls /etc/scalpel/scalpel.conf

Fedora/RHEL Installation

Bash 
# Install via DNF
sudo dnf install scalpel

# Verify installation
scalpel -V

Building from Source

Bash 
# Install dependencies
sudo apt install build-essential libtre-dev

# Clone repository
git clone https://github.com/sleuthkit/scalpel.git
cd scalpel

# Build and install
./configure
make
sudo make install

# Copy default configuration
sudo mkdir -p /etc/scalpel
sudo cp scalpel.conf /etc/scalpel/

SIFT Workstation

Scalpel comes pre-installed on the SIFT Workstation. Verify installation:

Bash 
# Check Scalpel version
scalpel -V

# Configuration file location on SIFT
ls /etc/scalpel/scalpel.conf

Configuration

Scalpel uses a configuration file (scalpel.conf) to define file signatures. By default, all signatures are commented out—you must uncomment the file types you want to carve.

Each line in the configuration file defines a file type with the following format:

Configuration Format 
# extension  case_sensitive  max_size  header                    footer
#
# extension:      File extension for carved files
# case_sensitive: y/n for header/footer matching
# max_size:       Maximum file size to carve
# header:         Hex or ASCII signature at file start
# footer:         Hex or ASCII signature at file end (optional)

Example Configuration Entries

scalpel.conf 
# JPEG images (standard JFIF)
jpg     y       200000000       \xff\xd8\xff\xe0\x00\x10        \xff\xd9

# JPEG images (EXIF format - digital cameras)
jpg     y       200000000       \xff\xd8\xff\xe1                \xff\xd9

# PNG images
png     y       20000000        \x89\x50\x4e\x47\x0d\x0a\x1a\x0a        \x49\x45\x4e\x44\xae\x42\x60\x82

# PDF documents
pdf     y       50000000        %PDF                            %EOF\x0d
pdf     y       50000000        %PDF                            %EOF\x0a

# Microsoft Office (DOCX, XLSX, PPTX - ZIP-based)
docx    y       50000000        \x50\x4b\x03\x04                \x50\x4b\x05\x06

# ZIP archives
zip     y       100000000       \x50\x4b\x03\x04                \x50\x4b\x05\x06

# Windows executables
exe     y       50000000        \x4d\x5a

# SQLite databases
sqlite  y       50000000        \x53\x51\x4c\x69\x74\x65\x20\x66\x6f\x72\x6d\x61\x74
⚠️ Important

Only uncomment the file types you need. Enabling too many signatures increases processing time and may generate false positives.

Basic Usage

Bash 
# Basic syntax
scalpel [options] image_file

# Common options:
#   -c config_file   Specify configuration file
#   -o output_dir    Specify output directory (required)
#   -b               Carve files from unallocated space only
#   -q clustersize   Carve only cluster-aligned files
#   -v               Verbose mode
#   -n               Don't add extensions to carved files
#   -p               Perform preview/dry run

Example Commands

Bash 
# Basic carving with default config
scalpel -o ./carved_files evidence.dd

# Use custom configuration file
scalpel -c ./my_scalpel.conf -o ./carved_files evidence.dd

# Verbose output for monitoring progress
scalpel -v -o ./carved_files evidence.dd

# Preview mode (don't actually carve)
scalpel -p -o ./preview evidence.dd

# Carve from raw device (requires root)
sudo scalpel -o ./carved_files /dev/sdb

Output Structure

Scalpel creates a directory structure organizing carved files by type:

Output Directory Structure 
carved_files/
├── audit.txt           # Log of carving operations
├── jpg-0-0/            # JPEG files (first signature)
│   ├── 00000000.jpg
│   ├── 00000001.jpg
│   └── 00000002.jpg
├── jpg-1-0/            # JPEG files (second signature)
│   └── 00000000.jpg
├── pdf-2-0/            # PDF files
│   ├── 00000000.pdf
│   └── 00000001.pdf
└── docx-3-0/           # DOCX files
    └── 00000000.docx
💡 Audit File

Always review audit.txt—it contains the offset where each file was found, which is crucial for documenting evidence location.

PhotoRec

PhotoRec is a powerful, open-source file recovery tool designed to recover lost files including video, documents, and archives from hard disks, CD-ROMs, and memory cards. Despite its name, PhotoRec recovers far more than just photos—it supports over 500 file formats.

📘 Course Reference

As noted in the Week 5 course materials, PhotoRec provides "multi-format recovery" with "wide format support," making it ideal for comprehensive file carving operations.

Installation

Ubuntu/Debian Installation

PhotoRec is part of the TestDisk package:

Bash 
# Install TestDisk (includes PhotoRec)
sudo apt update
sudo apt install testdisk

# Verify installation
photorec --version

# Alternative: Latest version from source
wget https://www.cgsecurity.org/testdisk-7.2.linux26-x86_64.tar.bz2
tar -xjf testdisk-7.2.linux26-x86_64.tar.bz2
cd testdisk-7.2
./photorec_static

Windows Installation

  1. Download from cgsecurity.org/wiki/TestDisk_Download
  2. Extract the ZIP file to a folder (e.g., C:\Tools\testdisk)
  3. Run photorec_win.exe from the extracted folder
PowerShell 
# Run PhotoRec from command line
cd C:\Tools\testdisk
.\photorec_win.exe

# Or add to PATH and run from anywhere
$env:PATH += ";C:\Tools\testdisk"
photorec_win.exe
💡 Tip

Run as Administrator when accessing physical drives or disk images in protected locations.

macOS Installation

Bash 
# Install via Homebrew
brew install testdisk

# Verify installation
photorec --version

# Run PhotoRec
photorec

Interactive Mode

PhotoRec features an interactive text-based interface for guided file recovery:

Select Media

Launch PhotoRec and select the disk image or device to analyze:

Bash 
# Launch with disk image
photorec evidence.dd

# Or launch and select from menu
photorec

Select Partition

Choose the partition to recover files from, or select "No partition" to scan the entire image.

Choose File System Type

Select the appropriate file system:

  • ext2/ext3/ext4 - Linux file systems
  • Other - FAT, NTFS, HFS+, and others

Select Search Space

  • Free - Search only unallocated space (faster)
  • Whole - Search entire partition (more thorough)

Choose Output Directory

Navigate to the destination folder and press C to confirm.

Command-Line Mode

PhotoRec also supports non-interactive command-line operation:

Bash 
# Basic syntax
photorec /log /d output_directory image_file

# Common options:
#   /log             Create photorec.log file
#   /d directory     Specify output directory
#   /cmd             Command-line mode (non-interactive)

# Example: Recover from disk image
photorec /log /d ./recovered evidence.dd

# Command-line mode with specific options
photorec /cmd evidence.dd partition_type,search,options

File Type Selection

│   ├── f0000001.jpg
│   ├── f0000002.pdf

Bash - Post-Recovery Validation 
# Count recovered files by type
find ./recovered -name "*.jpg" | wc -l
find ./recovered -name "*.pdf" | wc -l

# Validate file integrity
file ./recovered/recup_dir.1/f0000000.jpg

# Generate hashes for documentation
find ./recovered -type f -exec sha256sum {} \; > recovered_hashes.txt

# Check for corrupted images (requires ImageMagick)
find ./recovered -name "*.jpg" -exec identify {} \; 2>&1 | grep -i error

MFTECmd

MFTECmd is a powerful command-line tool developed by Eric Zimmerman for parsing NTFS Master File Table ($MFT) records. It extracts comprehensive file system metadata and outputs it in formats suitable for timeline analysis and forensic examination.

📘 Course Reference

MFT analysis is covered in Module 5.6.3 of the course materials. MFTECmd enables you to "Perform MFT analysis using specialized tools" and examine NTFS file system metadata including $STANDARD_INFORMATION and $FILE_NAME timestamps.

Installation

Windows Installation

MFTECmd is part of Eric Zimmerman's forensic tools collection:

Option 1: Download Individual Tool
  1. Visit ericzimmerman.github.io
  2. Download MFTECmd.zip
  3. Extract to C:\Tools\ZimmermanTools\
Option 2: Use Get-ZimmermanTools Script
PowerShell (Administrator) 
# Download the installer script
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/EricZimmerman/Get-ZimmermanTools/master/Get-ZimmermanTools.ps1" -OutFile "Get-ZimmermanTools.ps1"

# Run the script to download all tools
.\Get-ZimmermanTools.ps1 -Dest C:\Tools\ZimmermanTools

# Verify installation
C:\Tools\ZimmermanTools\MFTECmd.exe --version
Add to System PATH
PowerShell 
# Add to PATH for current session
$env:PATH += ";C:\Tools\ZimmermanTools"

# Verify
MFTECmd.exe --version

MFTECmd requires .NET 6 or later. Download from dotnet.microsoft.com if needed.

Extracting the MFT

Before using MFTECmd, you need to extract the $MFT file from the disk image:

Bash - Using icat from The Sleuth Kit 
# Find the partition offset
mmls evidence.dd

# Extract $MFT (inode 0 on NTFS)
icat -o 2048 evidence.dd 0 > extracted_mft.bin

# Verify extraction
file extracted_mft.bin
ls -lh extracted_mft.bin
PowerShell - Using FTK Imager or Raw Copy 
# If you have direct access to a mounted NTFS volume
# (Run as Administrator)
# Use raw copy tool or FTK Imager to extract:
# - $MFT
# - $MFTMirr
# - $LogFile
# - $UsnJrnl:$J

Basic Usage

Command Line 
# Basic syntax
MFTECmd.exe -f <MFT_file> --csv <output_directory>

# Common options:
#   -f, --file        Path to $MFT file
#   --csv             Output directory for CSV files
#   --csvf            Output filename for CSV
#   --json            Output directory for JSON files
#   --body            Output directory for bodyfile (timeline)
#   --bodyf           Bodyfile filename
#   --bdl             Drive letter for bodyfile paths
#   --blf             Use long format for bodyfile
#   --fls             Output fls-style format
#   --de              Debug mode (show errors)
#   --vss             Process Volume Shadow Copies

Example Commands

PowerShell / Bash 
# Parse MFT and output to CSV
MFTECmd.exe -f extracted_mft.bin --csv ./output --csvf mft_analysis.csv

# Generate bodyfile for timeline analysis
MFTECmd.exe -f extracted_mft.bin --body ./output --bodyf mft_timeline.body --bdl C:

# Output to JSON format
MFTECmd.exe -f extracted_mft.bin --json ./output

# Process with verbose error output
MFTECmd.exe -f extracted_mft.bin --csv ./output --de

# Linux with dotnet
dotnet MFTECmd.dll -f extracted_mft.bin --csv ./output

Output Fields

MFTECmd CSV output field descriptions
Field Description Forensic Value
EntryNumber MFT entry number Unique file identifier
SequenceNumber Reuse counter for entry Detect entry reuse
ParentEntryNumber Parent directory MFT entry Reconstruct directory structure
FileName File or directory name Identify files
Extension File extension File type identification
FileSize Logical file size Size analysis
Created0x10 $STANDARD_INFORMATION created File creation time
LastModified0x10 $SI last modified Content modification
LastRecordChange0x10 $SI MFT record change Metadata changes
LastAccess0x10 $SI last access File access time
Created0x30 $FILE_NAME created Original creation time
LastModified0x30 $FN last modified Timestamp comparison
InUse Entry allocation status Identify deleted files
IsDirectory Directory flag File vs folder
HasAds Alternate Data Streams present Hidden data detection
SiFlags $SI attribute flags Hidden, system, archive
🔑 Timestamp Analysis

Comparing 0x10 ($STANDARD_INFORMATION) and 0x30 ($FILE_NAME) timestamps can reveal timestamp manipulation—a key anti-forensics detection technique.

Timeline Integration

Bash - Creating Timeline with mactime 
# Generate bodyfile from MFT
MFTECmd.exe -f extracted_mft.bin --body ./output --bodyf mft.body --bdl C:

# Convert to timeline using mactime (from Sleuth Kit)
mactime -b ./output/mft.body -d > mft_timeline.csv

# Filter by date range
mactime -b ./output/mft.body -d 2026-01-01..2026-01-31 > january_activity.csv

# Combine with fls bodyfile for comprehensive timeline
cat fls.body mft.body > combined.body
mactime -b combined.body -d > full_timeline.csv

💡 Best Practice

Use these tools in combination: MFTECmd for NTFS metadata analysis and timeline creation, PhotoRec for broad file recovery, and Scalpel for targeted carving of specific file types with custom signatures.

Integrated Forensic Workflow

This workflow demonstrates how to use Scalpel, PhotoRec, and MFTECmd together with The Sleuth Kit for comprehensive file system analysis.

Initial Image Analysis

Use The Sleuth Kit to identify partitions and file systems:

Bash 
# Identify partitions
mmls evidence.dd

# Get file system details
fsstat -o 2048 evidence.dd

Extract MFT for NTFS Analysis

If NTFS, extract and parse the MFT:

Bash 
# Extract $MFT using icat
icat -o 2048 evidence.dd 0 > mft.bin

# Parse with MFTECmd
MFTECmd.exe -f mft.bin --csv ./analysis --body ./analysis --bdl C:

Identify Deleted Files

Use fls and MFTECmd output to identify deleted files:

Bash 
# List deleted files with fls
fls -o 2048 -r -d evidence.dd > deleted_files.txt

# Filter MFT CSV for deleted entries
# (InUse = FALSE)

Targeted File Carving with Scalpel

Carve specific file types from unallocated space:

Bash 
# Edit scalpel.conf to enable needed file types
# Then run carving
scalpel -c ./custom_scalpel.conf -o ./carved evidence.dd

Broad Recovery with PhotoRec

Perform comprehensive file recovery:

Bash 
# Run PhotoRec for broad recovery
photorec /log /d ./photorec_output evidence.dd

Create Unified Timeline

Combine all timeline sources:

Bash 
# Generate fls bodyfile
fls -o 2048 -r -m "C:" evidence.dd > fls.body

# Combine with MFT bodyfile
cat fls.body ./analysis/mft.body > combined.body

# Create timeline
mactime -b combined.body -d > full_timeline.csv

Validate and Document

Hash all recovered files and document findings:

Bash 
# Hash carved files
find ./carved -type f -exec sha256sum {} \; > carved_hashes.txt

# Hash PhotoRec output
find ./photorec_output -type f -exec sha256sum {} \; > photorec_hashes.txt

# Validate file types
find ./carved -type f -exec file {} \; > file_types.txt

Command Generator

Interactive Command Builder
scalpel -c /etc/scalpel/scalpel.conf -o ./output evidence.dd
