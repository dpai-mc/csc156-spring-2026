Overview

File carving and MFT analysis are essential techniques in digital forensics for recovering deleted files and examining file system metadata. This guide covers three powerful tools that complement The Sleuth Kit in your forensic toolkit.

📘 Learning Connection

These tools support CSLO 4: Apply file system analysis techniques to examine storage media and recover digital evidence. As stated in NIST SP 800-86, analysts should have a forensic toolkit containing various tools that provide the ability to perform quick reviews of data as well as in-depth analysis. The toolkit should allow its applications to be run quickly and efficiently from removable media or a forensic workstation.

Scalpel

🐧
Linux Only

Scalpel is designed for Linux systems. For Windows file carving, use PhotoRec or the Autopsy integrated carving module.

Scalpel is a fast, filesystem-independent file carving tool originally based on Foremost. It reads a database of header and footer definitions and extracts matching files from a disk image or raw device. As noted in the course materials, Scalpel's strengths are its speed and configurability.

💡 Key Advantage

Scalpel's strength lies in its highly configurable signature file, allowing forensic examiners to define custom file types and fine-tune carving parameters for specific investigations.

Installation

Ubuntu/Debian Installation

Bash 
# Update package list
sudo apt update

# Install Scalpel
sudo apt install scalpel

# Verify installation
scalpel -V

# Locate configuration file
ls /etc/scalpel/scalpel.conf

Fedora/RHEL Installation

Bash 
# Install via DNF
sudo dnf install scalpel

# Verify installation

# Configuration file location
ls /etc/scalpel/scalpel.conf

Building from Source

Bash

sudo apt install build-essential libtre-dev

# Clone repository

cd scalpel

# Build and install
./configure
make
sudo make install

# Copy default configuration
sudo mkdir -p /etc/scalpel
sudo cp scalpel.conf /etc/scalpel/

SIFT Workstation

Scalpel comes pre-installed on the SIFT Workstation. Verify installation:

Bash

scalpel -V

# Configuration file location on SIFT
ls /etc/scalpel/scalpel.conf

Configuration

Scalpel uses a configuration file (scalpel.conf) to define file signatures. By default, all signatures are commented out—you must uncomment the file types you want to carve.

Each line in the configuration file defines a file type. As documented in the course materials, the configuration fields are:

  1. Extension to assign to carved files
  2. Case sensitivity (y/n) for header/footer matching
  3. Maximum file size to carve
  4. Header signature (hex or ASCII)
  5. Footer signature (optional)
Configuration Format 
# extension  case_sensitive  max_size  header                    footer
#
# extension:      File extension for carved files
# case_sensitive: y/n for header/footer matching
# max_size:       Maximum file size to carve
# header:         Hex or ASCII signature at file start
# footer:         Hex or ASCII signature at file end (optional)

Example Configuration (from Course Materials)

scalpel.conf 
# Scalpel configuration file

# JPEG images
jpg     y       200000000       \xff\xd8\xff    \xff\xd9

# PNG images
png     y       200000000       \x89PNG         \x49\x45\x4e\x44\xae\x42\x60\x82

# PDF documents
pdf     y       50000000        %PDF            %EOF

# ZIP archives (including DOCX, XLSX, PPTX)
zip     y       100000000       PK\x03\x04      \x50\x4b\x05\x06
⚠️ Important

Only uncomment the file types you need. Enabling too many signatures increases processing time and may generate false positives.

Basic Usage

Bash 
# Basic syntax
scalpel [options] image_file

# Common options:
#   -c config_file   Specify configuration file
#   -o output_dir    Specify output directory (required)
#   -b               Carve files from unallocated space only
#   -q clustersize   Carve only cluster-aligned files
#   -v               Verbose mode
#   -n               Don't add extensions to carved files
#   -p               Perform preview/dry run

Example Commands

Bash 
# Basic carving with default config
scalpel -o ./carved_files evidence.dd

# Use custom configuration file
scalpel -c ./my_scalpel.conf -o ./carved_files evidence.dd

# Verbose output for monitoring progress
scalpel -v -o ./carved_files evidence.dd

# Preview mode (don't actually carve)
scalpel -p -o ./preview evidence.dd

# Carve from raw device (requires root)
sudo scalpel -o ./carved_files /dev/sdb

Output Structure

Scalpel creates a directory structure organizing carved files by type:

Output Directory Structure 
carved_files/
├── audit.txt           # Log of carving operations
├── jpg-0-0/            # JPEG files (first signature)
│   ├── 00000000.jpg
│   ├── 00000001.jpg
│   └── 00000002.jpg
├── jpg-1-0/            # JPEG files (second signature)
│   └── 00000000.jpg
├── pdf-2-0/            # PDF files
│   ├── 00000000.pdf
│   └── 00000001.pdf
└── zip-3-0/            # ZIP/Office files
    └── 00000000.zip
💡 Audit File

Always review audit.txt—it contains the offset where each file was found, which is crucial for documenting evidence location. As noted in NIST SP 800-86, the exact commands and their output should be documented.

Carving Best Practices

From the course materials, follow these best practices when using Scalpel:

  1. Carve from forensic image, not original evidence
  2. Start with common file types (images, documents)
  3. Adjust maximum sizes based on case context
  4. Review results for false positives
  5. Document carving parameters used
  6. Validate recovered files (open, hash, examine)
  7. Consider fragmentation limitations
  8. Carve both allocated and unallocated space

PhotoRec

🌐
Cross-Platform

PhotoRec runs on Linux, Windows, and macOS. It provides wide format support and is ideal for multi-format recovery operations.

PhotoRec is a powerful, open-source file recovery tool designed to recover lost files including video, documents, and archives from hard disks, CD-ROMs, and memory cards. Despite its name, PhotoRec recovers far more than just photos—it supports over 500 file formats.

📘 Course Reference

As noted in the Week 5 course materials, PhotoRec provides "multi-format recovery" with "wide format support," making it ideal for comprehensive file carving operations when you need to recover many different file types.

Installation

cd testdisk-7.2

  2. Extract the ZIP file to a folder (e.g., C:\Tools\testdisk)

Interactive Mode

PhotoRec features an interactive text-based interface for guided file recovery. As noted in the course materials, PhotoRec supports both interactive mode and command-line operation.

Select Media

Launch PhotoRec and select the disk image or device to analyze:

Bash 
# Launch with disk image
photorec evidence.dd

# Or launch and select from menu
photorec

Select Partition

Choose the partition to recover files from, or select "No partition" to scan the entire image.

Choose File System Type

Select the appropriate file system:

  • ext2/ext3/ext4 - Linux file systems
  • Other - FAT, NTFS, HFS+, and others

Select Search Space

  • Free - Search only unallocated space (faster)
  • Whole - Search entire partition (more thorough)

Choose Output Directory

Navigate to the destination folder and press C to confirm.

Command-Line Mode

PhotoRec also supports non-interactive command-line operation:

Bash 
# Basic syntax
photorec /log /d output_directory image_file

# Common options:
#   /log             Create photorec.log file
#   /d directory     Specify output directory
#   /cmd             Command-line mode (non-interactive)

# Example: Recover from disk image
photorec /log /d ./recovered evidence.dd

# Command-line mode with specific options
photorec /cmd evidence.dd partition_type,search,options

MFTECmd

🪟
Windows Only

MFTECmd is built for Windows using .NET. Eric Zimmerman's tools are designed specifically for the Windows platform. For Linux-based MFT analysis, consider using analyzeMFT (Python) as an alternative.

MFTECmd is a powerful command-line tool developed by Eric Zimmerman for parsing NTFS Master File Table ($MFT) records. It extracts comprehensive file system metadata and outputs it in formats suitable for timeline analysis and forensic examination.

📘 Course Reference

MFT analysis is covered in Module 5.6.3 of the course materials. MFTECmd enables you to "Perform MFT analysis using specialized tools" and examine NTFS file system metadata including $STANDARD_INFORMATION and $FILE_NAME timestamps. As noted in the course resources, Eric Zimmerman's blog is recommended for NTFS tools.

Prerequisites: .NET 9 Runtime

⚠️ .NET Version Requirement

Eric Zimmerman's GitHub provides two versions of his tools: a legacy .NET 4 version and the supported .NET 9 version. For this course, we use the .NET 9 version which includes the latest features and security updates.

🔍 System Check: Verify .NET 9 Installation

Before installing MFTECmd, verify that .NET 9 runtime is installed on your Windows system:

PowerShell - Check Installed .NET Runtimes 
# List all installed .NET runtimes
dotnet --list-runtimes

# Expected output should include a line similar to:
# Microsoft.NETCore.App 9.x.x [C:\Program Files\dotnet\shared\Microsoft.NETCore.App]

# If you see version 9.x.x, you're ready to proceed
# If not, follow the installation steps below
If .NET 9 is NOT installed, follow the installation instructions in the next section.
If .NET 9 IS installed, skip to "Installing MFTECmd" below.

Extracting the MFT

Before using MFTECmd, you need to extract the $MFT file from the disk image. Use The Sleuth Kit on your Linux forensic workstation:

Bash (Linux) - Using icat from The Sleuth Kit 
# Find the partition offset
mmls evidence.dd

# Extract $MFT (inode 0 on NTFS)
icat -o 2048 evidence.dd 0 > extracted_mft.bin

# Verify extraction
file extracted_mft.bin
ls -lh extracted_mft.bin

# Transfer to Windows workstation for MFTECmd analysis
# (via shared folder, USB, or network transfer)
PowerShell (Windows) - Using FTK Imager 
# If using FTK Imager on Windows:
# 1. Open FTK Imager
# 2. File > Add Evidence Item > Image File
# 3. Navigate to the NTFS partition
# 4. Expand [root] folder
# 5. Right-click $MFT > Export Files
# 6. Save to your analysis folder

# Key files to extract for comprehensive analysis:
# - $MFT (Master File Table)
# - $MFTMirr (MFT backup)
# - $LogFile (Transaction log)
# - $UsnJrnl:$J (Change journal)

Basic Usage

PowerShell 
# Basic syntax
MFTECmd.exe -f <MFT_file> --csv <output_directory>

# Common options:
#   -f, --file        Path to $MFT file
#   --csv             Output directory for CSV files
#   --csvf            Output filename for CSV
#   --json            Output directory for JSON files
#   --body            Output directory for bodyfile (timeline)
#   --bodyf           Bodyfile filename
#   --bdl             Drive letter for bodyfile paths
#   --blf             Use long format for bodyfile
#   --fls             Output fls-style format
#   --de              Debug mode (show errors)
#   --vss             Process Volume Shadow Copies

Example Commands

PowerShell 
# Parse MFT and output to CSV (most common usage)
MFTECmd.exe -f C:\Evidence\extracted_mft.bin --csv C:\Analysis --csvf mft_analysis.csv

# Generate bodyfile for timeline analysis
MFTECmd.exe -f C:\Evidence\extracted_mft.bin --body C:\Analysis --bodyf mft_timeline.body --bdl C:

# Output to JSON format
MFTECmd.exe -f C:\Evidence\extracted_mft.bin --json C:\Analysis

# Process with verbose error output
MFTECmd.exe -f C:\Evidence\extracted_mft.bin --csv C:\Analysis --de

# Generate both CSV and bodyfile
MFTECmd.exe -f C:\Evidence\extracted_mft.bin --csv C:\Analysis --csvf mft.csv --body C:\Analysis --bodyf mft.body --bdl C:

Output Fields

As covered in Module 5.6.3, key fields to analyze include entry and sequence numbers, parent references, SI vs. FN timestamps, file sizes, and attribute types.

Field Description Forensic Value
EntryNumber MFT entry number Unique file identifier
SequenceNumber Reuse counter for entry Detect entry reuse
ParentEntryNumber Parent directory MFT entry Reconstruct directory structure
FileName File or directory name Identify files

🔑 Timestamp Analysis (SI vs. FN)

As covered in Lab Activity 5, comparing 0x10 ($STANDARD_INFORMATION) and 0x30 ($FILE_NAME) timestamps can reveal timestamp manipulation—a key anti-forensics detection technique. The $SI timestamps are easily modified by user-level tools, while $FN timestamps require kernel-level access to change.

Timeline Integration

PowerShell/Bash - Creating Timeline with mactime 
# On Windows: Generate bodyfile from MFT
MFTECmd.exe -f extracted_mft.bin --body C:\Analysis --bodyf mft.body --bdl C:

# Transfer bodyfile to Linux workstation, then:
# Convert to timeline using mactime (from Sleuth Kit)
mactime -b mft.body -d > mft_timeline.csv

# Filter by date range
mactime -b mft.body -d 2026-01-01..2026-01-31 > january_activity.csv

# Combine with fls bodyfile for comprehensive timeline
cat fls.body mft.body > combined.body
mactime -b combined.body -d > full_timeline.csv

💡 Best Practice

Use these tools in combination across your forensic workstations: On Linux/SIFT, use Scalpel for targeted carving and PhotoRec for broad recovery. On Windows, use MFTECmd for NTFS metadata analysis and timeline creation. PhotoRec works on both platforms for consistent file recovery workflows.

Integrated Forensic Workflow

This workflow demonstrates how to use Scalpel, PhotoRec, and MFTECmd together with The Sleuth Kit for comprehensive file system analysis. Note the platform requirements for each step.

Initial Image Analysis Linux

Bash

# Get file system details
fsstat -o 2048 evidence.dd

Extract MFT for NTFS Analysis Linux

Bash 
# Extract $MFT using icat
icat -o 2048 evidence.dd 0 > mft.bin

file mft.bin
ls -lh mft.bin

# Transfer to Windows workstation for MFTECmd analysis

Parse MFT with MFTECmd Windows

Analyze MFT metadata on Windows:

PowerShell

MFTECmd.exe -f mft.bin --csv C:\Analysis --body C:\Analysis --bdl C:

# Open CSV in Timeline Explorer for analysis
TimelineExplorer.exe C:\Analysis\*_MFTECmd_Output.csv

Identify Deleted Files Linux

Use fls to identify deleted files:

Bash 
# List deleted files with fls

# Review deleted files
cat deleted_files.txt

Targeted File Carving with Scalpel Linux

Carve specific file types from unallocated space:

Bash

# Run carving

# Review audit log

Perform comprehensive file recovery:

Bash / PowerShell 
# Run PhotoRec for broad recovery
photorec /log /d ./photorec_output evidence.dd

# Or use interactive mode
photorec evidence.dd

Create Unified Timeline Linux

Combine all timeline sources:

Bash 
# Generate fls bodyfile
fls -o 2048 -r -m "C:" evidence.dd > fls.body

# Combine with MFT bodyfile (transferred from Windows)
cat fls.body mft.body > combined.body

# Create timeline
mactime -b combined.body -d > full_timeline.csv
