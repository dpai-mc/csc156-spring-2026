1. Introduction & Objectives

In Week 7 you explored the Windows Registry—artifacts that record configuration and user behavior. This week shifts focus to system-generated artifacts: evidence the operating system creates automatically, often without user awareness. Windows Event Logs are the most important of these. Every logon, failed password attempt, privilege escalation, service start, and policy change is recorded here, making Event Logs a primary evidence source in nearly every Windows investigation.

Within a forensic workflow, Event Log analysis typically occurs after imaging and hash verification (Week 6) and alongside Registry analysis (Week 7). Findings from logs are later correlated with execution artifacts, LNK files, and NTFS journals (Modules 8.2–8.5) to build a unified timeline.

After completing this module you will be able to:
  • Describe the .evtx file format, its XML-based structure, and the three main log channels.
  • Locate Event Log files on a Windows system and explain their forensic significance.
  • Identify critical Security Event IDs related to logon, account management, privilege use, and service activity.
  • Filter and create custom views in Windows Event Viewer.
  • Parse .evtx files with EvtxECmd, export to CSV, and analyze results in Timeline Explorer.
  • Interpret Event Log entries to answer investigative questions about user activity.

2. Event Log Architecture

2.1 The .evtx Format

Since Windows Vista / Server 2008, Event Logs are stored in the EVTX (Event Log XML) binary format. Each .evtx file is a binary container whose individual event records are serialized as Binary XML (BinXML). When rendered by a tool, each record produces a structured XML document containing fields such as EventID, TimeCreated, Computer, Channel, and EventData key-value pairs.

2.2 Storage Location

All default log files reside in:

C:\Windows\System32\winevt\Logs\

Filenames follow the pattern ChannelName%4SubChannel.evtx. The three primary files are System.evtx, Security.evtx, and Application.evtx.

4. Tool Walk-Through

4a. Windows Event Viewer — Filtering & Custom Views

Step 1 — Open Event Viewer

Press Win + R, type eventvwr.msc, and press Enter. Expand Windows Logs in the left pane to see System, Security, and Application.

Step 2 — Filter the Current Log

Right-click Security → Filter Current Log. In the dialog, enter 4624 in the Event IDs field. Optionally set a date/time range. Click OK. Only successful logon events are now displayed.

Step 3 — Create a Custom View

From the Action menu choose Create Custom View. Select By log → Security. In the Event IDs field, enter 4624,4625,4672 (comma-separated). Name the view Logon Investigation and click OK. The custom view now appears under Custom Views and persists across sessions.

Step 4 — Export for External Analysis

Right-click the filtered view → Save All Events As… → choose .evtx format. This exported file can be parsed with EvtxECmd on your forensic workstation.

4b. EvtxECmd & Timeline Explorer

EvtxECmd (by Eric Zimmerman) parses .evtx files from the command line and outputs structured CSV ideal for sorting, filtering, and timeline analysis.

Step 1 — Parse a Single Log File

EvtxECmd.exe -f "C:\Cases\Evidence01\Security.evtx" --csv "C:\Cases\Output" --csvf Security_parsed.csv

-f specifies the input file. --csv sets the output directory. --csvf names the output file.

Step 2 — Parse an Entire Logs Directory

EvtxECmd.exe -d "C:\Cases\Evidence01\winevt\Logs" --csv "C:\Cases\Output" --csvf AllLogs_parsed.csv

-d processes every .evtx file in the directory into a single CSV.

Step 3 — Open in Timeline Explorer

Launch Timeline Explorer (EZTools). Open the CSV produced in Step 1 or 2. Use the column filters to narrow results—for example, filter the EventId column to 4624 and the PayloadData1 column (which contains the target username) to a suspect account name.

Step 4 — Sort and Search

Click the TimeCreated column header to sort chronologically. Use Ctrl + F to search for specific values such as a workstation name or IP address across all payload fields.

5. Mini-Scenario: Unauthorized Logon Investigation

Scenario Background

On 2026-02-18, the IT Security team at Cascade Financial Group received an alert that a dormant service account, svc_backup, authenticated to the domain controller CFG-DC01 outside of its approved maintenance window (Saturday 02:00–04:00 UTC). Your task: examine the Security Event Log excerpts below and answer the guided questions.

Guided Questions

  1. Which account was targeted, and was the logon ultimately successful?
    The account svc_backup was targeted. After two failed attempts (entries 1 & 2, Event ID 4625), a successful logon occurred at 14:33:14 UTC (entry 3, Event ID 4624).
  2. What logon type was used, and what does it indicate?
    Logon Type 10 (RemoteInteractive) indicates a Remote Desktop (RDP) session was used.
  3. From which workstation and IP address did the logon originate?
    Source workstation: WKS-MKT04; Source IP: 10.20.5.118. This workstation should be investigated next.
  4. What elevated privileges were obtained?
    Event ID 4672 (entry 4) shows SeBackupPrivilege and SeRestorePrivilege were assigned—these allow reading/writing any file regardless of ACLs.
  5. What suspicious post-logon activity occurred?
    A new account helpdesk$ was created (4720) and deleted roughly five minutes later (4726)—consistent with a backdoor account used temporarily. The attacker then disabled Logon/Logoff auditing (4719) to cover tracks.
  6. What investigative steps would you recommend next?
    Examine WKS-MKT04 for compromise; check Prefetch/AmCache on CFG-DC01 for tools executed during the session; review $UsnJrnl for file-system changes between 14:33 and 14:42 UTC; preserve all evidence before audit logging is re-enabled.

6. Knowledge Check

Answer each question, then click your choice to see immediate feedback. These are ungraded self-assessment items.

1. A forensic examiner finds multiple Event ID 4625 entries followed by a single Event ID 4624 for the same account within two minutes. What does this pattern most likely indicate?

✓ Correct. Multiple failed logons (4625) immediately followed by a success (4624) is a classic indicator of password guessing or brute-force activity.
✗ Incorrect. The pattern of repeated 4625 (failed logon) entries followed by a 4624 (successful logon) for the same account strongly suggests a brute-force or password-guessing attack. The correct answer is B.

2. Which Event ID indicates that an account received elevated (administrator-equivalent) privileges upon logon?

✓ Correct. Event ID 4672 is logged when special privileges (such as SeDebugPrivilege or SeBackupPrivilege) are assigned to a new logon session.
✗ Incorrect. Event ID 4672 records special privilege assignment at logon. 4624 is a successful logon (without privilege detail), 4720 is account creation, and 7036 is a service state change. The correct answer is B.

3. Where are default Windows Event Log files stored on disk?

✓ Correct. The default storage path for all .evtx files is C:\Windows\System32\winevt\Logs\.
✗ Incorrect. Windows stores Event Log .evtx files in C:\Windows\System32\winevt\Logs\. The correct answer is B.

4. An investigator discovers Event ID 4719 in the Security log. Why is this finding significant?

✓ Correct. Event ID 4719 records a change to the system audit policy. Attackers may disable auditing to prevent subsequent activity from being logged—a critical anti-forensic indicator.
✗ Incorrect. Event ID 4719 means the audit policy was changed. This is significant because an attacker may disable auditing to hide their tracks. The correct answer is C.

5. Which EvtxECmd flag would you use to parse every .evtx file in a directory at once?

✓ Correct. The -d flag tells EvtxECmd to process an entire directory of .evtx files, combining results into a single output.
✗ Incorrect. Use -d to specify a directory for batch processing. The -f flag is for a single file. The correct answer is B.

7. Key Terms & Next Steps

EVTX
The modern Windows Event Log file format (introduced in Vista/Server 2008). Uses a binary container with XML-structured event records; file extension .evtx.
Event ID
A numeric identifier assigned to each type of event within a Windows Event Log channel (e.g., 4624 = successful logon).
Channel
A named category that groups related events. The three primary channels are System, Security, and Application.
Logon Type
A numeric code within Event IDs 4624/4625 that indicates how a user authenticated (e.g., Type 2 = Interactive, Type 10 = RDP).
XPath Query
An XML query language used within Event Viewer (and wevtutil) to filter events by field values such as Event ID, time range, or account name.
EvtxECmd
An open-source command-line tool (Eric Zimmerman suite) for parsing .evtx files and exporting structured CSV output for analysis in Timeline Explorer.
