🎯 Module Overview

📋 What You'll Learn

This module introduces the foundational concepts of forensic imaging—the cornerstone of digital forensic investigations. Understanding the difference between forensic imaging and standard file copying is essential for maintaining evidence integrity and ensuring your findings are admissible in legal proceedings.

Forensic imaging is your first hands-on interaction with evidence, and doing it correctly is critical. A single mistake during acquisition can compromise an entire investigation.

✓ Learning Objectives

By the end of this module, you will be able to:

  • Define forensic imaging and explain its purpose in digital investigations
  • Distinguish between forensic imaging and logical copying
  • Identify the three main types of forensic images (physical, logical, targeted)
  • Explain why bit-for-bit copies are essential for evidence integrity
  • Describe the basic workflow for creating a forensic image

💡 Why This Matters

Every forensic examination begins with acquisition. If evidence is not acquired properly, all subsequent analysis may be challenged or excluded. This module establishes the foundation for everything you'll learn in Weeks 7-17 about analyzing digital evidence.

📚 Key Definitions

Forensic Image

A forensic image is a bit-for-bit (sector-by-sector) copy of a storage device that captures every bit of data, including deleted files, file slack, unallocated space, and system areas. It is an exact duplicate of the original media at the binary level.

Bit-for-Bit Copy (Bit-Stream Copy)

A copying method that duplicates every single bit from the source media, regardless of whether that bit contains active file data, deleted data, or appears to be "empty." This preserves all potential evidence, including data that the operating system cannot see.

Logical Copy

A file-level copy that only captures files and folders visible to the operating system. It does not include deleted files (unless in the Recycle Bin), file slack, unallocated space, or hidden system areas.

Evidence Integrity

The assurance that digital evidence has not been altered, modified, or tampered with from the time of acquisition through analysis and presentation. Integrity is verified through cryptographic hash values.

Forensically Sound

A process or method that ensures digital evidence is collected, preserved, and analyzed in a manner that maintains its integrity and admissibility. A forensically sound acquisition does not modify the original evidence in any way.

Source Media (Evidence Drive)

The original storage device containing potential evidence. This device must never be modified during the forensic process.

Destination Media (Target Drive)

The storage device where the forensic image will be stored. Must have sufficient capacity and should be forensically wiped before use.

Write Blocker

A hardware or software tool that prevents any write operations to the source media, ensuring the original evidence remains unaltered. (Covered in detail in Module 6.2)

Hash Value

A fixed-size alphanumeric string generated by a cryptographic hash function (like MD5 or SHA-256) that serves as a unique "fingerprint" of data. Used to verify image integrity. (Covered in detail in Module 6.5)

Slack Space

The unused space in a file's last allocated cluster. May contain remnants of previously deleted files.

Unallocated Space

Disk space not currently assigned to any file. Often contains recoverable deleted data.

⚖️ Forensic Imaging vs. Logical Copying

Understanding the fundamental difference between forensic imaging and standard file copying is critical for digital forensic examiners. While both methods transfer data from one location to another, they capture vastly different amounts of information.

🔬 Forensic Imaging

  • Captures: Every sector of the storage device
  • Includes: Active files, deleted files, file slack, unallocated space, hidden partitions, boot records
  • Method: Bit-for-bit sector copy
  • Verification: Hash values verify exact duplication
  • Legal Status: Admissible as evidence
  • Purpose: Complete evidence preservation

📁 Logical Copying

  • Captures: Only files visible to the OS
  • Includes: Active files and folders only
  • Method: File system-level copy
  • Verification: May or may not include hashing
  • Legal Status: May be challenged in court
  • Purpose: General file backup

⚠️ Critical Understanding

When you perform a logical copy (like dragging files in Windows Explorer), you miss approximately 75% or more of potential evidence. Deleted files, file fragments, browser history remnants, and other critical artifacts exist in unallocated space and slack space—areas that logical copies completely ignore.

Real-World Example: Why This Matters

Scenario: An employee is suspected of stealing proprietary documents before leaving the company. They deleted the files and emptied the Recycle Bin.

  • Logical Copy Result: Shows no evidence of the documents—they appear to be gone
  • Forensic Image Result: Recovers the deleted documents from unallocated space, along with timestamps showing when they were accessed, copied to USB, and deleted

The forensic image provides the evidence needed; the logical copy would have resulted in a failed investigation.

💾 Types of Forensic Images

Forensic examiners can create different types of images depending on the investigation requirements, legal constraints, and available time. Understanding when to use each type is essential for effective evidence collection.

Physical Image (Full Disk Image)

A physical image is a complete bit-for-bit copy of the entire storage device, from the first sector to the last. This is the gold standard for forensic acquisition.

What It Captures:

  • All partitions (including hidden and deleted partitions)
  • Master Boot Record (MBR) or GUID Partition Table (GPT)
  • All file systems on the device
  • All allocated and unallocated space
  • File slack space
  • Bad sectors (marked as such)

When to Use:

  • Criminal investigations requiring complete evidence preservation
  • When deleted file recovery is anticipated
  • When hidden partitions may exist
  • When you have sufficient time and storage capacity
  • When legal requirements mandate complete acquisition

Advantages:

  • Most complete evidence capture
  • Highest court admissibility
  • Allows for any type of analysis

Disadvantages:

  • Requires storage equal to device capacity
  • Takes longest to acquire
  • May capture irrelevant data

ℹ️ Storage Consideration

A physical image of a 1TB drive requires at least 1TB of storage space (more if using uncompressed raw format). Always ensure your destination media has sufficient capacity before beginning acquisition.

Logical Image (Volume Image)

A logical image captures a specific partition or volume rather than the entire physical device. It includes all data within that partition, including unallocated space within the partition boundaries.

What It Captures:

  • All files and folders in the selected partition
  • Deleted files within the partition
  • Unallocated space within the partition
  • File slack within the partition
  • File system metadata (MFT for NTFS, etc.)

What It Does NOT Capture:

  • Other partitions on the device
  • Space between partitions
  • MBR/GPT partition tables
  • Hidden or deleted partitions

When to Use:

  • When only specific volumes are relevant
  • When time constraints exist
  • When storage capacity is limited
  • Civil litigation with defined scope
  • When legal authority is limited to specific data

⚠️ Important Limitation

Logical images miss evidence that may exist outside partition boundaries or in hidden partitions. If a suspect created a hidden partition to store illicit files, a logical image of the visible partition would not capture this evidence.

Targeted Image (Selective Acquisition)

A targeted image captures only specific files, folders, or data types relevant to the investigation. This is the most limited form of forensic acquisition.

What It Captures:

  • Specifically selected files and folders
  • Specific file types (e.g., all .docx files)
  • Files from specific date ranges
  • Specific user profiles or directories

What It Does NOT Capture:

  • Deleted files (unless specifically targeted)
  • Unallocated space
  • File slack
  • Files outside selection criteria
  • System artifacts and metadata

When to Use:

  • eDiscovery with narrow scope
  • When legal authority is very limited
  • Preliminary triage before full acquisition
  • When specific files are known targets
  • Extremely time-sensitive situations

⚠️ Significant Limitations

Targeted acquisition is not recommended for criminal investigations or situations where comprehensive evidence is needed. It may miss critical evidence and can be challenged as incomplete. Always document why targeted acquisition was chosen over more complete methods.

💡 Decision Framework

When deciding which image type to create, ask yourself:

  1. What is the legal scope? Search warrants and legal authority may dictate acquisition limits.
  2. What evidence might exist? If deleted files are likely relevant, physical imaging is necessary.
  3. What are the time constraints? Physical imaging takes longer but provides complete evidence.
  4. What storage is available? Ensure you have capacity for your chosen image type.

When in doubt, choose the most complete option your circumstances allow.

🔄 The Forensic Imaging Process

Creating a forensic image follows a structured process designed to maintain evidence integrity. While specific tools and commands vary (covered in Modules 6.3 and 6.4), the fundamental workflow remains consistent.

1

Preparation

Before touching the evidence, prepare your environment:

  • Document the evidence (photos, serial numbers, condition)
  • Verify your forensic workstation is ready
  • Ensure destination media is forensically wiped and has sufficient capacity
  • Prepare write blocking hardware or software
  • Have chain of custody forms ready
2

Write Protection

Connect the source media through a write blocker:

  • Hardware write blocker: Connect between evidence drive and workstation
  • Software write blocker: Enable before connecting drive
  • Verify write protection is active before proceeding
  • Document the write blocking method used

Detailed in Module 6.2: Write Blockers and Forensic Duplicators

3

Pre-Image Hash

Calculate hash values of the source media:

  • Generate MD5 and/or SHA-1/SHA-256 hash of source
  • Document hash values in your notes
  • This establishes the baseline for integrity verification

Detailed in Module 6.5: Hash Verification

4

Image Acquisition

Create the forensic image using appropriate tools:

  • Select the appropriate image format (raw, E01, AFF)
  • Configure compression and segmentation options
  • Begin the imaging process
  • Monitor for errors during acquisition
  • Document start time, end time, and any issues

Detailed in Modules 6.3 and 6.4: Acquisition Tools

5

Post-Image Verification

Verify the image integrity:

  • Calculate hash of the created image
  • Compare to pre-image hash—they must match exactly
  • If hashes don't match, investigate and re-image if necessary
  • Document verification results
6

Documentation

Complete all required documentation:

  • Update chain of custody
  • Record all hash values
  • Document tools used and their versions
  • Note any errors or anomalies
  • Store original evidence securely

Detailed in Module 6.7: Documentation Best Practices

ℹ️ Time Expectations

Forensic imaging takes significantly longer than logical copying. As a general rule:

  • USB 2.0: ~30-60 MB/s = ~4-8 hours for 1TB
  • USB 3.0: ~100-150 MB/s = ~2-3 hours for 1TB
  • SATA (direct): ~150-200 MB/s = ~1.5-2 hours for 1TB
  • NVMe: ~500+ MB/s = ~30-45 minutes for 1TB

Always plan for imaging to take longer than expected. Hardware issues, bad sectors, and verification add time.

📋 Module Summary

✅ Key Takeaways

Core Concepts

  • Forensic imaging creates a bit-for-bit copy of storage media, capturing all data including deleted files, slack space, and unallocated areas
  • Logical copying only captures files visible to the operating system, missing critical evidence
  • Evidence integrity is maintained through write protection and hash verification

Image Types

  • Physical image: Complete device copy—gold standard for investigations
  • Logical image: Single partition copy—useful when scope is limited
  • Targeted image: Selected files only—limited use cases, may miss evidence

Process Steps

  1. Preparation and documentation
  2. Write protection implementation
  3. Pre-image hash calculation
  4. Image acquisition
  5. Post-image hash verification
  6. Complete documentation

Remember

The acquisition phase is your ONE chance to capture evidence correctly. Once you begin analysis on a working copy, you cannot go back and capture data you missed. When in doubt, always choose the most complete acquisition method available.
