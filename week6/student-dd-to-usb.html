🔎 Lab Exercise — Forensic Disk Imaging

Write a Forensic .dd Image
to a USB Drive

You’ll use Win32DiskImager to write one of the Lab 5 disk images directly onto a USB drive — making the USB behave exactly like the original evidence disk. Then you’ll use FTK Imager to verify your work with hash values.

⏰ ~45–60 minutes 🏢 Windows 11 (host or VM) 💾 2 GB+ USB drive required 💪 Beginner-friendly

In Lab 5 you worked with .dd files loaded directly in FTK Imager as files on your computer. This lab takes it one step further — you’ll write one of those same disk images onto a USB drive so that FTK Imager sees it as a real physical disk, just like an actual piece of evidence.

🔎

Why Does This Matter?

In a real investigation, you receive a physical drive — not a file. By writing the .dd image to a USB, you’re practicing exactly what an investigator does: taking custody of physical media, creating a forensic copy of it, and verifying the copy with hash values. The USB becomes the evidence drive.

What happens under the hood

What you doWhat Windows sees on USBWhat FTK Imager sees
Copy the .dd file normally (drag & drop) A normal USB drive with a file called evidence.dd on it A USB drive. The .dd file is just a document sitting on it.
Write the .dd with Win32DiskImager (this lab) Windows can’t read the USB at all — shows a “Format?” warning A physical disk with the original filesystem inside — just like a real evidence drive ✓
⚠

Your USB drive will be completely erased

Win32DiskImager overwrites every byte on the USB drive — all existing files on it will be permanently gone. Use a USB drive with nothing on it you need to keep. A 2 GB or larger drive works for the 1 GB Lab 5 images.

  • A USB drive, 2 GB or larger, that you don’t mind erasing completely. USB 3.0 is faster but USB 2.0 works fine.
  • Windows 11 — either your regular Windows computer or your VirtualBox Windows 11 VM (see Step 5 for VM setup)
  • Win32DiskImager — you’ll download and install this in Step 3
  • A Lab 5 .dd image file downloaded to your computer — get the link from your instructor (Step 4)
  • FTK Imager already installed — you’ll use it in Step 8 to verify the write
  • About 45–60 minutes (most of which is waiting for the write to finish)
ℹ

USB Drive Size vs. the Image File

The Lab 5 images are 1 GB (.dd files). Your USB drive needs to have at least 1 GB of actual storage capacity — meaning a “1 GB” labeled USB will likely not work due to the difference between manufacturer-stated GB and real usable space. A 2 GB or larger USB drive is safe.

  1. Go to the SourceForge download page

    Open a web browser and navigate to:

    URL
    https://sourceforge.net/projects/win32diskimager/files/latest/download

    The download should start automatically. If it doesn’t, click the green Download button. You’re looking for win32diskimager-1.0.0-install.exe (about 17 MB).

  2. Run the installer as Administrator

    Once the download finishes, find it in your Downloads folder. Right-click on win32diskimager-1.0.0-install.exe → select Run as administrator. Click Yes on the UAC prompt.

    ℹ

    You must run it as Administrator. Win32DiskImager needs elevated permissions to access physical disk drives. If you just double-click it, it may install but then fail to see your USB drive later.

  3. Complete the installer

    Click through the setup wizard: Accept the license → choose the install location (default is fine) → click Install → click Finish. The installer will create a Start Menu shortcut.

  4. Verify the install

    Open the Start Menu and search for Win32DiskImager. It should appear. Do not launch it yet — first connect your USB drive, then always launch it by right-clicking → Run as administrator.

    ⚠

    Always launch as Administrator

    Every time you use Win32DiskImager, launch it with Run as administrator. If you launch it normally, it won’t be able to write to physical drives and the Device dropdown will be empty or grayed out.

Instructor Note — Link Provided Separately

Your instructor has provided a link to the Lab 5 .dd image files in Canvas. Download one of the 1 GB .dd files from that link to a location you can easily find — your Desktop or a C:\ForensicLab\ folder works well.

  1. Download the file

    Click the link from your instructor and download the .dd file. These files are approximately 1 GB so the download may take a few minutes depending on your connection speed.

  2. Note where it saved

    Remember the full path to the file. You’ll need to browse to it inside Win32DiskImager. For example:

    Example path
    C:\Users\YourName\Downloads\lab5-evidence.dd
  —or—

  3. Verify the download with a hash (optional but good practice)

    Your instructor may have provided a known-good hash value for the file. If so, verify it now with PowerShell. Open PowerShell (no admin needed for this) and run:

    PowerShell
    Get-FileHash "C:\Users\YourName\Downloads\lab5-evidence.dd" -Algorithm SHA256

    Compare the output to the hash your instructor provided. They should match exactly. This confirms the file downloaded correctly and hasn’t been corrupted.

ℹ

Not using a VM? Skip to Step 6.

If you are running Windows 11 directly on your computer (not inside VirtualBox), you can skip this entire step and go straight to Step 6.

By default, VirtualBox does not automatically give your Windows 11 VM access to USB drives you plug into your physical computer. The USB drive will be visible to your host OS (Mac, Linux, or Windows host) but the VM won’t see it — and Win32DiskImager inside the VM won’t be able to use it. You need to enable USB passthrough.

⚠

Do this with the VM powered off

The USB settings shown below must be configured while your Windows 11 VM is shut down, not while it is running or saved/paused. Shut it down completely first.

Enable USB 3.0 Controller in VirtualBox

  1. Open VM Settings

    In the VirtualBox Manager window, select your Windows 11 VM from the list (make sure it says Powered Off). Click the Settings gear icon at the top.

  2. Go to the USB section

    In the Settings window, click USB in the left panel.

  3. Enable USB and select the controller

    Make sure the Enable USB Controller checkbox is ticked. Then select USB 3.0 (xHCI) Controller. If you only see USB 1.1 and USB 2.0, you may need to install the VirtualBox Extension Pack (see note below).

    ℹ

    VirtualBox Extension Pack

    USB 3.0 support requires the VirtualBox Extension Pack, which is a free separate download from virtualbox.org/wiki/Downloads. Download the Extension Pack that matches your VirtualBox version, double-click it to install, then return to this USB settings step. USB 2.0 will also work — it will just be slower.

  4. Add a USB Device Filter for your drive

    Click the + icon on the right side of the USB Devices list. Plug in your USB drive now if you haven’t already. Your USB drive should appear in the list. Click on it to add it as a filter. This tells VirtualBox to automatically give the VM control of this specific USB device when it’s plugged in.

    Click OK to save the settings.

  5. Start the VM and confirm the USB drive appears

    Start your Windows 11 VM. Once it boots, open File Explorer inside the VM. Your USB drive should appear as a drive letter (like D: or E:).

    If it doesn’t appear automatically, try: in the VirtualBox menu bar at the top of the VM window, go to Devices → USB and click on your USB device name to manually attach it.

    ✓

    Once your USB drive shows up in File Explorer inside the VM, you’re ready. Continue to Step 6.

🛑

Troubleshooting: USB drive not appearing in the VM

  • VM host is Mac or Linux: You may need to add your user account to the vboxusers group. On Mac/Linux, run: sudo usermod -aG vboxusers $USER then log out and back in.
  • USB controlled by host OS: On a Windows host, right-click the USB icon in the system tray (bottom-right) and “Eject” the drive from the host before the VM can claim it.
  • Extension Pack version mismatch: The Extension Pack version must exactly match your VirtualBox version number. Check both in VirtualBox → Help → About.
  • Still not working? Ask your instructor — some VirtualBox configurations on lab computers may need additional setup.
🛑

Plug in your USB drive before launching Win32DiskImager

Connect your USB drive first, then launch the application. If you open Win32DiskImager before plugging in the USB, it may not appear in the Device list. If that happens, plug in the USB drive and click the refresh icon inside the app.

  1. Launch Win32DiskImager as Administrator

    Open the Start Menu → search for Win32DiskImager → right-click it → Run as administrator. Click Yes on the UAC prompt. The application window opens.

  2. Select your .dd file — change the file type filter first

    Click the small folder icon next to the Image File field. A standard Windows file browser opens.

    Win32DiskImager 1.0.0
    −
    □
    ✕

    ⚠ Important: The file browser defaults to showing only *.img files. Your .dd file will be invisible until you change this!

    In the file browser, find the file type dropdown (bottom-right, usually showing Disk Images (*.img *.IMG)) and change it to *.* (All files). Your .dd file will then appear.
    Device:
    [ G:\ ▼ ]

    After changing the filter, navigate to where you saved the .dd file, select it, and click Open.

    The Image File box should now show the full path to your .dd file.

  3. Confirm the correct Device is selected

    Look at the Device dropdown on the right side. It shows drive letters for removable devices — it should show your USB drive’s letter (like [G:\] or [E:\]).

    🛑

    Double-check this is your USB drive, not another drive

    Win32DiskImager only shows removable drives in this list, which reduces the risk — but verify it matches what you expect. Open File Explorer, find your USB drive, note its drive letter, and confirm it matches what Win32DiskImager shows. The data on the selected drive will be permanently destroyed.

  4. Click “Write”

    Click the Write button. Win32DiskImager will show a confirmation dialog:

    Confirm overwrite

    ⚠ Are you sure you want to write to this device?

    This will overwrite all data on the selected device. This operation cannot be undone.

    Confirm you have the right drive selected, then click Yes. The write begins immediately.

  5. Wait for the write to complete

    A progress bar shows the write progress. Expected times for a 1 GB image:

    USB TypeTypical Speed~Time for 1 GB
    USB 2.0~15–25 MB/s1–2 minutes
    USB 3.0 / 3.1~60–120 MB/sUnder 1 minute

    Don’t unplug the USB drive while the progress bar is moving. Wait until you see the “Write Successful” message.

As soon as the write finishes (and sometimes during the write), Windows will pop up a dialog that looks alarming:

Microsoft Windows
💾
You need to format the disk in Drive G: before you can use it.
Do you want to format it?
🛑

Click “Cancel” — Do NOT click “Format disk”

Clicking “Format disk” would erase the image you just wrote and turn the USB back into a blank drive. Your work would be gone. Always click Cancel.

🔎

Why does Windows show this?

The USB drive now contains a forensic disk image with its own filesystem (from the original imaged device). Windows doesn’t recognize this filesystem format and assumes the drive is blank or damaged. This is correct behavior — it means the write worked. FTK Imager, unlike Windows Explorer, knows how to read it properly.

In a real forensics job, seeing this dialog when you connect evidence media is actually a good sign — it means the drive hasn’t been tampered with or re-formatted.

After clicking Cancel, Win32DiskImager should display: “Write Successful.” You can now close Win32DiskImager.

Win32DiskImager said “Write Successful” but let’s confirm it actually worked by opening the USB in FTK Imager the same way you would open real evidence media.

  1. Open FTK Imager

    Launch FTK Imager. You don’t need to run it as Administrator just to browse evidence, but you will need Administrator access if you want to image the USB (which is what you’ll do next lab).

  2. Add your USB as a Physical Drive evidence item

    Go to File → Add Evidence Item…

    In the dialog that opens, select Physical Drive and click Next.

    In the drive list, find your USB drive. It will appear with its size (approximately 1 GB, matching the image). Click Finish.

    ℹ

    Select Physical Drive, not “Image File.” You’re treating the USB as a physical piece of evidence, just like a real hard drive.

  3. Expand the evidence tree

    In the Evidence Tree panel (top-left of FTK Imager), expand your USB drive entry. You should see:

    • A partition (e.g., Partition 1)
    • A filesystem label and folder structure
    • Files and folders from the original disk image
    ✓

    If you can see a filesystem structure with folders and files, the write was successful! The USB is presenting itself as the original evidence disk. Proceed to Step 9 to record your hash values.

    ⚠

    If the tree shows only “Unpartitioned Space” or nothing

    This usually means the write was interrupted or the wrong drive was selected. Remove the USB drive, repeat Step 6 carefully, and check that the device letter was correct.

Hash verification is the heart of forensic integrity. You need to confirm that what’s on the USB drive is identical to the original .dd file. You’ll compute hashes two ways and compare them.

Part A: Hash the original .dd file

Open PowerShell (no admin needed) and run these two commands. Replace the path with wherever your .dd file is saved:

PowerShell — Hash the source .dd file
Get-FileHash "C:\Users\YourName\Downloads\lab5-evidence.dd" -Algorithm MD5
Get-FileHash "C:\Users\YourName\Downloads\lab5-evidence.dd" -Algorithm SHA256

Record both values below. The output shows a Hash column — copy the long alphanumeric string.

Source .dd File Hashes

MD5:
SHA-256:

Part B: Hash the USB drive using FTK Imager

With your USB drive still loaded in FTK Imager as a Physical Drive:

  1. Right-click the USB physical drive

    In the Evidence Tree, right-click directly on the top-level entry for your USB physical drive (not a partition or folder inside it — the top-level item).

  2. Select “Verify Drive/Image…”

    FTK Imager will read every sector of the USB drive and compute its MD5 and SHA-256 hashes. For a 1 GB drive this takes about 1–3 minutes. A progress dialog shows the status.

  3. Record the results

    When complete, FTK Imager shows the hash values for the drive. Record them here:

USB Physical Drive Hashes (from FTK Imager Verify)

MD5:
SHA-256:

Part C: Compare the hashes

The MD5 from Part A should exactly match the MD5 from Part B. Same for SHA-256. Even one character difference means something went wrong.

✓

🎉 If hashes match: the write is verified!

The USB drive is a bit-for-bit identical copy of the original .dd file. This is exactly what a forensic investigator confirms before proceeding with any analysis. Your USB is now a verified evidence drive.

🛑

If hashes don’t match:

  • Did Windows format the USB when you weren’t looking? Check by browsing it in FTK Imager — if it looks empty or like a blank FAT drive, it was formatted. Repeat Step 6.
  • Did you hash the right .dd file in Part A? Make sure the path points to the same file you used in Win32DiskImager.
  • Was the write interrupted? If the progress bar didn’t fully complete, repeat Step 6.
