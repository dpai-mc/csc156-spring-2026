Week 6: Data Acquisition and Preservation

Forensic Imaging, Write Protection, and Hash Verification

📅 Duration: 5 Days
📚 Lecture: 2 Hours
🔬 Lab: 3 Hours
⏰ Total Time: ~12-14 Hours

Week 6 represents a critical transition point in the course—moving from understanding how data is stored (Weeks 4-5) to how to properly collect that data for forensic analysis. This week covers the essential techniques for creating forensically sound copies of digital evidence, including forensic imaging, write protection, and cryptographic hash verification. These skills are fundamental to maintaining evidence integrity and ensuring admissibility in legal proceedings.

You will learn to use industry-standard tools including dd, dcfldd, FTK Imager, and Guymager to create forensic images. Understanding the difference between forensic imaging and logical copying, and knowing when to apply each method, is crucial for any digital forensics professional.

By the end of Week 6, students will be able to:

  1. Differentiate between forensic imaging and logical copying, explaining when each method is appropriate
  2. Explain the purpose and operation of hardware and software write blockers
  3. Create forensically sound disk images using industry-standard tools (dd, FTK Imager, Guymager)
  4. Calculate and verify data integrity using cryptographic hash algorithms (MD5, SHA-1, SHA-256)
  5. Document the acquisition process in accordance with evidence handling procedures
  6. Select appropriate acquisition methods based on evidence type, time constraints, and legal requirements
  7. Troubleshoot common acquisition problems and implement solutions
  8. Explain the forensic implications of acquiring data from SSDs versus HDDs

Lab Activity 6: Forensic Imaging and Hash Verification

20 Points

Objective: Demonstrate proficiency in creating forensically sound disk images using multiple tools and verifying data integrity through cryptographic hashing.

📁 Part 1: Imaging with dd/dcfldd (Linux)

  • Connect USB drive via write blocker
  • Identify device using lsblk or fdisk -l
  • Create raw image with appropriate parameters
  • Calculate MD5 and SHA-256 hashes
  • Document process and verify hash match

💻 Part 2: Imaging with FTK Imager (Windows)

  • Connect USB drive via write blocker
  • Create E01 format image with compression
  • Enter appropriate case information
  • Verify image after creation
  • Compare hashes with Part 1 results

✅ Part 3: Image Verification & Conversion

  • Mount E01 image and verify accessibility
  • Convert between image formats
  • Verify integrity after conversion
  • Complete lab documentation
