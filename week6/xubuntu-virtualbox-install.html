Installing Xubuntu Minimal in VirtualBox

Platform: Windows 11 Host · VirtualBox 7.x  |  OS: Xubuntu 24.04 LTS (Minimal)  |  Approximate Time: 25–30 min

This tutorial walks you through creating a Xubuntu Minimal virtual machine inside VirtualBox on your Windows 11 computer. Xubuntu uses the lightweight XFCE desktop environment, which is faster and uses fewer resources than standard Ubuntu — making it ideal for running inside a VM. You will also install VirtualBox Guest Additions for better display support and USB passthrough, which is required to complete later lab exercises.

Why Xubuntu Minimal?

Standard Ubuntu ships with a full desktop experience (GNOME) and bundled apps, which consumes significant RAM and disk space inside a VM. Xubuntu Minimal strips this down to a functional XFCE desktop with fewer pre-installed apps:

ISOApprox. SizeDesktop
Ubuntu 24.04 Desktop~6.5 GBGNOME (full)
Xubuntu 24.04 Minimal~3 GBXFCE (lightweight)

Create a New VM

Open VirtualBox Manager and click New to start the wizard. Enter the following settings:

SettingValue
Namexubuntu-26-spring (or similar)
ISO ImageBrowse to your downloaded Xubuntu ISO
Type / VersionLinux / Ubuntu (64-bit)
Unattended InstallDo NOT enable — leave unchecked
Base Memory (RAM)4096 MB (4 GB) recommended; 2 GB minimum
Processors (CPUs)2 (increase if your host has extra cores)
Virtual Hard Disk25 GB
ℹ️

2 GB RAM is the minimum that works. If your computer has 16 GB or more, setting the VM to 4 GB gives a noticeably smoother experience.

Configure Display Settings

After the VM is created, open Settings → Display before starting it:

SettingValue
Video Memory128 MB
Graphics ControllerVMSVGA (preferred; try VBoxSVGA if issues)
3D AccelerationEnable (improves display performance)

Configure Other Settings

Still in Settings, make these additional adjustments:

CategoryAction
AudioDisable — not needed for this lab
NetworkLeave as NAT (default) — keep enabled
USBEnable USB; select USB 2.0 or 3.0 to match your drive
USB Device FiltersClick Add empty filter — configure after install
Shared FoldersOptional: add a host folder for easy file transfer between host and VM
💡

Shared Folders let you drag files between your Windows desktop and the VM without USB. Add a folder from your Windows host and it will always be available inside the VM — useful for copying files back and forth.

Boot and Launch the Installer

Start your VM. The boot menu will appear. Select Try or Install Xubuntu and press Enter. If you wait a few seconds, it will auto-select this option.

ℹ️

The installation process is identical to standard Ubuntu — the only difference is the XFCE desktop environment and the "X" in the name. Every wizard step is the same.

Walk Through the Installer Screens

Follow each screen in order:

  1. Language — Select English, click Next.
  2. Keyboard Layout — English (US), click Next.
  3. Internet Connection — Select Use wired connection. Even though your Windows host may use Wi-Fi, VirtualBox presents a virtual wired NIC to the VM.
  4. Installation Type — Choose Interactive installation.
  5. Application Selection — Keep Minimal. Do not add NVIDIA drivers or media codecs.
  6. Disk Setup — Choose Erase disk and install. This only erases the virtual 25 GB disk — your host computer is not affected.
  7. Encryption — Leave encryption OFF and click Next.
  8. User Account — Enter your desired username and a password. Check Require password to log in.
  9. Time Zone — Select your region (e.g., America/Los_Angeles for Pacific Time).
  10. Review Summary — Confirm partitions (sda1 and sda2 will be created), then click Install.
✅

Installation takes approximately 6–10 minutes depending on your computer's speed. You can monitor progress by right-clicking the VM in VirtualBox Manager and opening Resource Usage — if the CPU/Disk graphs show activity, it's still working.

⚠️

If the graphs are completely flat for more than 5 minutes and the installer hasn't progressed, the VM may be frozen. Try Save State → restart it. If that fails, power off and reinstall.

Shut Down After Installation

When installation completes, instead of clicking Restart, use the XFCE menu to shut the VM down cleanly:

  1. Click the small blue XFCE menu icon in the top-left corner of the desktop.
  2. Select Shut Down.
  3. Wait for the VM to fully power off.

The installer ISO will auto-eject itself. You can verify by going to Settings → Storage — the optical drive should show empty. If not, manually remove it before booting.

Start the VM and Open a Terminal

Power the VM back on and log in with your credentials. Once at the XFCE desktop, right-click anywhere on the desktop and select Open Terminal Here.

💡

If the terminal font is too small: go to Edit → Preferences → Appearance and change the font size to 14. Close and reopen the terminal if the change doesn't apply immediately.

Update the Package List

Before installing anything, update the package database to ensure you get the latest versions:

Terminal
sudo apt update

Enter your password when prompted. You will see a list of Ubuntu package repositories — this confirms Xubuntu is Ubuntu under the hood with XFCE on top.

What Are Guest Additions?

VirtualBox Guest Additions are drivers and utilities installed inside the VM that enable:

  • Dynamic screen resolution scaling
  • Smoother mouse integration (no more clicking to capture/release)
  • USB device passthrough (required for later labs)
  • Shared folders and clipboard between host and VM

Step 5a — Install Required Build Tools

Guest Additions compile kernel modules during installation. Install the build dependencies first:

Terminal
sudo apt install build-essential dkms linux-headers-$(uname -r) -y
⚠️

The package name is build-essential (no trailing "s"). Typing build-essentials will produce an "unable to locate package" error. The -y flag auto-confirms the installation prompt.

Step 5b — Insert the Guest Additions CD Image

In the VirtualBox window menu bar, go to:

Devices → Insert Guest Additions CD Image…

Wait a moment for the virtual CD to mount. A file manager window may open automatically showing the disc contents.

Step 5c — Navigate to the CD and Run the Installer

Use the terminal to navigate to the mounted Guest Additions disc and run the installer:

Step 1 — Go to /media
cd /media
Step 2 — List contents (find your username directory)
ls
Step 3 — Enter your username's directory (replace "yourusername")
cd yourusername
Step 4 — List contents (find the VBox_GAs folder)
ls
Step 5 — Enter the Guest Additions directory (version may differ)
cd VBox_GAs_7.2.6
Step 6 — List contents to confirm VBoxLinuxAdditions.run is present
ls
Step 7 — Run the installer
sudo ./VBoxLinuxAdditions.run
ℹ️

Linux is case-sensitive. The filename is VBoxLinuxAdditions.run with capital V, B, L, and A. The ./ prefix tells the shell to execute the file directly. You need sudo for the required system permissions.

✅

If it worked, you will see output like "VirtualBox Guest Additions installer" followed by module build messages. The installer exits automatically when complete.

Step 5d — Reboot the VM

Reboot to fully load the new Guest Additions kernel modules:

Terminal
sudo reboot

After reboot, your VM should have improved mouse integration and the screen resolution options under View → Virtual Screen should be enabled (they were greyed out before).

Set Up a USB Device Filter

USB passthrough allows your VM to directly access a USB drive plugged into your Windows host. This is required for later lab exercises.

  1. Power off the VM completely.
  2. In VirtualBox Manager, select your VM and open Settings → USB.
  3. Ensure USB is enabled and the controller matches your drive: USB 2.0 for older drives, USB 3.0 for newer drives.
  4. Click the Add Empty Filter icon (the plug with a "+" icon) on the right side. This creates a blank filter that will capture your USB drive when you plug it in later.
  5. Click OK to save.
💡

You do not need to specify the drive right now. The empty filter allows you to assign a specific USB device later. When you plug in your USB drive and start the VM, you can attach it via Devices → USB in the menu bar.

Verify USB Access Inside the VM

After starting the VM with a USB drive plugged in:

Navigate to /media and find your USB drive
cd /media/yourusername
ls

You should see your USB drive listed (e.g., NTFS or the drive label). You can then apply Linux commands to the files on that drive.

✅

If the drive appears in the file manager and in /media/yourusername/, USB passthrough is working correctly.

Scaling and Display Modes

VirtualBox offers several modes for managing the VM window on your Windows desktop:

ModeHow to ActivateDescription
Normal Window Default VM in a standard window. Adjust window size manually.
Scaled Mode View → Scaled Mode Stretches VM display to fill window. Doesn't change VM resolution but makes it larger/smaller visually.
Full Screen Host + F VM takes over the entire screen. Exit with Host + Home.
ℹ️

On Windows, the Host key is the Right Ctrl key by default. Use Host + Home to access the VirtualBox menu when in full screen or scaled mode.

Eject the Guest Additions CD

After installation and reboot, eject the Guest Additions disc to keep things clean:

  1. Open the File Manager from the XFCE menu.
  2. Find the VBox_GAs disc in the left sidebar.
  3. Click the eject icon next to it.

Alternatively, in Settings → Storage, select the optical drive and click the disc icon to remove the image.
