🎯 Module Overview

📋 What You'll Learn

While command-line tools (Module 6.3) offer power and flexibility, graphical user interface (GUI) tools provide an accessible, user-friendly approach to forensic imaging. This module covers the two most widely used GUI imaging tools:

  • FTK Imager - The industry-standard Windows-based imaging tool
  • Guymager - A fast, open-source Linux imaging tool

These tools simplify the acquisition process while providing comprehensive logging, built-in hash verification, and support for industry-standard forensic image formats.

✓ Learning Objectives

By the end of this module, you will be able to:

  • Explain the advantages and use cases for GUI imaging tools
  • Identify and compare forensic image formats (Raw, E01, AFF)
  • Use FTK Imager to create forensic images on Windows systems
  • Use Guymager for efficient Linux-based imaging
  • Configure case information and metadata during acquisition
  • Select the appropriate tool based on investigation requirements
🎓 CSLO 3: Perform data collection using forensic technology methods in accordance with evidence handling procedures

💡 GUI vs. Command-Line: When to Use Each

GUI tools are ideal when:

  • You need visual confirmation of settings before imaging
  • Case documentation and metadata entry are important
  • You're working with less experienced team members
  • You want integrated hash verification and logging

Command-line tools (Module 6.3) are better when:

  • Working in headless or minimal environments
  • Automating imaging through scripts
  • Maximum control over parameters is needed
  • GUI tools aren't available

📁 Forensic Image Formats

Before diving into specific tools, it's essential to understand the forensic image formats they support. The format you choose affects compression, metadata storage, compatibility, and file size.

Raw (dd)
Raw Disk Image
.dd, .raw, .img, .bin
Characteristics:
  • Exact bit-for-bit copy with no container
  • No compression (image = source size)
  • No built-in metadata storage
  • Universal compatibility
  • Hash stored in separate file
Best For:

Maximum compatibility, simple workflows, when storage space isn't limited

E01 (EnCase)
Expert Witness Format
.E01, .E02, .E03...
Characteristics:
  • Industry standard forensic format
  • Built-in compression (reduces size 30-50%)
  • Embedded case metadata
  • Integrated hash values (MD5, SHA-1)
  • Automatic file splitting
  • CRC verification per block
Best For:

Most forensic investigations, court cases, when documentation is critical

AFF
Advanced Forensic Format
.aff, .afd, .afm
Characteristics:
  • Open-source format
  • Compression support
  • Extensible metadata
  • Encryption capability
  • Less widely supported than E01
Best For:

Open-source workflows, when encryption is needed, research environments

ℹ️ E01 Format Recommendation

For most forensic investigations, E01 format is recommended because:

  • Compression reduces storage requirements by 30-50%
  • Case information is embedded in the image file
  • Hash values are stored with the image for verification
  • Widely supported by forensic analysis tools
  • Automatic splitting handles large images

Format Comparison

Comparison of forensic image formats
Feature Raw (dd) E01 AFF
Compression No Yes (zlib) Yes (multiple)
Embedded Metadata No Yes Yes
Embedded Hash No Yes (MD5, SHA-1) Yes
Auto-Split No (manual) Yes Yes
Encryption No No Yes
Tool Support Universal Very Wide Moderate
Court Acceptance High Very High Moderate
Open Standard Yes Proprietary* Yes

*E01 format is proprietary but well-documented and widely implemented in open-source tools via libewf.

🖥️ FTK Imager (Windows)

📀

FTK Imager

AccessData / Exterro

Free Windows GUI

FTK Imager

A free, Windows-based forensic imaging tool developed by AccessData (now Exterro). It is widely used in law enforcement and corporate investigations for creating forensic images, previewing evidence, and mounting forensic images.

Key Features

📁 Multiple Image Formats

Supports Raw (dd), E01, AFF, and SMART formats for maximum flexibility

🔐 Built-in Hash Calculation

Calculates MD5, SHA-1, and SHA-256 hashes during acquisition

👁️ Drive Preview

Preview drive contents without creating a full image (triage capability)

💾 Memory Acquisition

Capture live system RAM for volatile data analysis

📂 Image Mounting

Mount forensic images as read-only drives for analysis

📋 Comprehensive Logging

Detailed acquisition logs for documentation and court

FTK Imager Workflow

1

Select Source

Choose the evidence source type from the File menu: File → Create Disk Image

💿 Physical Drive
📁 Logical Drive
🖼️ Image File
📂 Folder Contents

Physical Drive: Entire disk including all partitions and unallocated space
Logical Drive: Single partition/volume only
Image File: Convert between image formats
Folder Contents: Logical acquisition of specific folders

2

Select Destination

Configure the output settings for your forensic image:

  • Image Format: E01 (recommended), Raw, AFF, or SMART
  • Destination Path: Where to save the image file
  • Image Filename: Base name for the image (without extension)
  • Fragment Size: Size for split files (e.g., 2GB for DVD, 4GB for FAT32)
3

Enter Case Information

Document case details that will be embedded in the E01 image:

📋 Case Number
🏷️ Evidence Number
📝 Description
👤 Examiner Name
📄 Notes

ℹ️ Documentation Tip

Always fill in case information completely. This metadata is embedded in E01 files and provides crucial documentation for chain of custody.

4

Configure Options

Set acquisition parameters:

  • Compression: 0 (none) to 9 (maximum) - typically use 6 for good balance
  • Fragment Size: 0 for single file, or specify size (e.g., 2147483648 for 2GB)
  • Verify after creation: Always enable to confirm image integrity
5

Acquire and Verify

Start the imaging process and monitor progress:

  • Watch the progress bar and transfer speed
  • Note any errors reported during acquisition
  • Wait for automatic verification to complete
  • Review the verification results
  • Save the acquisition log

✓ Verification Complete

FTK Imager displays "Image Verification Results" showing computed vs. reported hashes. Both must match for a successful acquisition.

FTK Imager Additional Capabilities

FTK Imager can preview drive contents without creating a full image—useful for triage:

  1. Select File → Add Evidence Item
  2. Choose Physical Drive or Logical Drive
  3. Browse the file system in the Evidence Tree
  4. Export specific files if needed

⚠️ Write Protection Required

Even for preview, always use a write blocker. FTK Imager opens drives read-only, but the operating system may still attempt writes.

FTK Imager can capture live system memory for volatile data analysis:

  1. Select File → Capture Memory
  2. Choose destination path and filename
  3. Optionally include pagefile
  4. Click "Capture Memory"

Memory captures include:

  • Running processes
  • Network connections
  • Encryption keys in memory
  • Recently accessed data
  • Malware artifacts

💡 When to Capture Memory

Capture memory BEFORE shutting down a live system. Once powered off, RAM contents are lost. Memory acquisition is covered in detail in Week 12.

FTK Imager can mount forensic images as virtual drives for analysis:

  1. Select File → Image Mounting
  2. Browse to the image file (E01, Raw, etc.)
  3. Choose mount type:
    • Physical & Logical: Mount as both physical and logical drive
    • Physical Only: Mount as physical disk
    • Logical Only: Mount individual partitions
  4. Select mount method:
    • Block Device / Read Only: Recommended for forensics
    • File System / Read Only: For file-level access
  5. Click "Mount"

The image appears as a drive letter in Windows Explorer, allowing you to browse and analyze using standard tools.

Extract specific files from a forensic image:

  1. Add the image as an evidence item
  2. Navigate to the desired file in the Evidence Tree
  3. Right-click and select Export Files
  4. Choose destination folder

This is useful for extracting specific artifacts without processing the entire image.

FTK Imager Advantages and Limitations

✅ Advantages

  • Free and widely available
  • User-friendly graphical interface
  • Comprehensive logging and documentation
  • Preview capability reduces unnecessary imaging
  • Memory acquisition included
  • Image mounting functionality
  • Multiple format support
  • Industry-standard acceptance

❌ Limitations

  • Windows only (no Linux/macOS version)
  • Slower than some alternatives (single-threaded)
  • Limited automation/scripting capability
  • No command-line interface
  • Cannot image to network locations directly
  • Requires GUI environment

🐧 Guymager (Linux)

🔧

Guymager

Open Source Forensic Imager

Free Open Source Linux GUI

Guymager

An open-source forensic imaging tool for Linux that features multi-threaded acquisition for high-speed imaging. It is commonly included in forensic Linux distributions like CAINE, SIFT, and Paladin.

Key Features

⚡ Multi-Threaded Acquisition

Optimized for speed with parallel processing—significantly faster than single-threaded tools

📁 Format Support

Creates Raw (dd), E01 (Expert Witness), and AFF images

🔐 Built-in Verification

Automatic hash calculation and verification (MD5, SHA-1, SHA-256)

📋 Clone Capability

Direct disk-to-disk cloning for creating physical duplicates

📊 Detailed Logging

Comprehensive info files with acquisition details

💻 Low Resource Usage

Efficient operation even on older hardware

Guymager Workflow

1

Launch Guymager

Guymager requires root privileges to access raw devices:

sudo guymager

The main window displays all detected storage devices with details including:

  • Device path (e.g., /dev/sdb)
  • Model and serial number
  • Size
  • State (Idle, Acquisition, etc.)
2

Select Source Device

Identify your evidence drive in the device list. Verify by checking:

  • Device size matches expected
  • Serial number matches documentation
  • Model name is correct

⚠️ Device Verification

Always verify you've selected the correct device. Imaging the wrong drive could overwrite critical data or miss evidence entirely.

3

Start Acquisition

Right-click on the source device and select "Acquire image"

The acquisition dialog appears with configuration options.

4

Configure Acquisition Settings

Set the following options in the acquisition dialog:

File Format:
  • Linux dd raw: Uncompressed raw image
  • Expert Witness Format (E01): Compressed with metadata (recommended)
  • Advanced Forensic Format (AFF): Open-source format
Destination:
  • Image directory: Path to save the image
  • Image filename: Base name (without extension)
Case Information (for E01):
  • Case number
  • Evidence number
  • Examiner
  • Description
  • Notes
Hash Calculation:
  • ☑️ Calculate MD5
  • ☑️ Calculate SHA-1
  • ☑️ Calculate SHA-256
  • ☑️ Verify image after acquisition
Split Size (E01):

Default is 2GB segments. Adjust based on destination file system.

5

Monitor and Complete

Click "Start" to begin acquisition. Monitor progress in the main window:

  • State: Shows "Acquisition" during imaging
  • Progress: Percentage complete
  • Speed: Current transfer rate
  • Remaining: Estimated time to completion

Upon completion, state changes to "Finished - Verified & ok" if successful.

6

Review Results

Guymager creates an info file (.info) alongside the image containing:

  • Acquisition start and end times
  • Source device details
  • Hash values (MD5, SHA-1, SHA-256)
  • Verification results
  • Any errors encountered

Save this info file as part of your case documentation.

Guymager Speed Advantage

⚡ Multi-Threading Performance

Guymager's multi-threaded architecture provides significant speed improvements over single-threaded tools:

  • Reading: Dedicated thread for reading source data
  • Hashing: Separate threads for hash calculation
  • Compression: Parallel compression (for E01)
  • Writing: Dedicated thread for writing output

Typical Performance: Guymager can be 2-3x faster than FTK Imager for the same hardware, especially with E01 compression enabled.

Guymager Advantages and Limitations

✅ Advantages

  • Free and open-source
  • Very fast (multi-threaded)
  • Excellent for Linux forensic workstations
  • Low resource requirements
  • Included in forensic distributions (CAINE, SIFT)
  • Disk-to-disk cloning capability
  • Detailed info file logging

❌ Limitations

  • Linux only (no Windows version)
  • No preview capability
  • No memory acquisition
  • Cannot mount images
  • Less intuitive interface than FTK Imager
  • Requires root privileges

Tool Selection Decision Guide

Recommended: FTK Imager

If your forensic workstation runs Windows, FTK Imager is the clear choice:

  • Native Windows application with full feature set
  • Preview capability for triage before imaging
  • Memory acquisition for live systems
  • Image mounting for analysis
  • Industry-standard acceptance

Alternative: Consider using command-line tools (dd via WSL or Cygwin) for scripted workflows.

Recommended: Guymager

For Linux forensic workstations (CAINE, SIFT, Ubuntu):

  • Native Linux application with excellent performance
  • Multi-threaded for fast acquisition
  • Pre-installed in forensic distributions
  • Disk-to-disk cloning capability
  • Low resource requirements

Alternative: Use dcfldd or dc3dd (Module 6.3) for command-line workflows or scripting.

Recommended: Guymager (Linux)

When imaging speed is the primary concern:

  • Guymager's multi-threading provides 2-3x speed improvement
  • Consider booting a Linux forensic distribution if normally using Windows
  • Use SATA or NVMe connections instead of USB when possible
  • Ensure destination drive is fast (SSD preferred)

Speed Comparison (approximate for 500GB drive):

  • FTK Imager (USB 3.0): ~90 minutes
  • Guymager (USB 3.0): ~45 minutes
  • Guymager (SATA direct): ~30 minutes

💡 Best Practice: Know Multiple Tools

Professional forensic examiners should be proficient with multiple imaging tools:

  • Primary tool: Your preferred tool for routine imaging (FTK Imager or Guymager)
  • Backup tool: Alternative for when primary isn't available
  • Command-line: dd/dcfldd/dc3dd for scripts and minimal environments

Different situations may require different tools—flexibility is key.

📋 Module Summary

✅ Key Takeaways

Forensic Image Formats

  • E01: Recommended for most investigations—compression, metadata, embedded hashes
  • Raw (dd): Universal compatibility, no compression or metadata
  • AFF: Open-source with encryption, less widely supported

FTK Imager (Windows)

  • Free, industry-standard Windows imaging tool
  • Features: Preview, memory acquisition, image mounting
  • 5-step workflow: Source → Destination → Case Info → Options → Acquire
  • Limitation: Single-threaded (slower than Guymager)

Guymager (Linux)

  • Free, open-source Linux imaging tool
  • Multi-threaded for fast acquisition (2-3x faster)
  • Included in forensic distributions (CAINE, SIFT)
  • Limitation: No preview, memory acquisition, or mounting

Tool Selection

  • Windows environment → FTK Imager
  • Linux environment → Guymager
  • Speed priority → Guymager (boot Linux if needed)
  • Preview/triage needed → FTK Imager

Best Practice Summary

  • Always use E01 format unless specific requirements dictate otherwise
  • Complete all case information fields for documentation
  • Enable verification to confirm image integrity
  • Save acquisition logs as part of case documentation
  • Know multiple tools for flexibility in different situations
