Lab Overview

This lab demonstrates proficiency in creating forensically sound disk images using multiple tools and verifying data integrity through cryptographic hashing. You will work with both command-line tools (dd/dcfldd) on Linux and GUI-based tools (FTK Imager) on Windows to understand the strengths of each approach.

⚠️ Critical Reminder

Data acquisition is the "make or break" phase of digital forensics. An improperly acquired image can render an entire investigation useless. Always use write blocking and verify your hashes!

Learning Objectives

  • Create forensically sound disk images using command-line tools (dd/dcfldd)
  • Create forensically sound disk images using GUI tools (FTK Imager)
  • Calculate and verify MD5 and SHA-256 hashes for data integrity
  • Compare raw (dd) and E01 (Expert Witness) image formats
  • Document the acquisition process following forensic standards

Part 1: Imaging with dd/dcfldd (Linux)

The dd command is a fundamental Unix utility for low-level copying of data. In forensics, we use it (or its enhanced version dcfldd) to create bit-for-bit copies of storage media.

1 Connect and Identify the Evidence Drive

▼
  1. Connect the provided USB drive through a write blocker (hardware preferred) or enable software write blocking
  2. Open a terminal and identify the device using lsblk
  3. Verify the device details with fdisk -l
  4. Document the device identifier (e.g., /dev/sdb)
# List all block devices
lsblk

# Example output:
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0 256.0G  0 disk 
├─sda1   8:1    0   512M  0 part /boot/efi
└─sda2   8:2    0 255.5G  0 part /
sdb      8:16   1   8.0G  0 disk        ← Evidence drive
└─sdb1   8:17   1   8.0G  0 part 

# Get detailed information
sudo fdisk -l /dev/sdb

Terminal commands to identify connected storage devices

⛔ Warning

Triple-check your device identifier before imaging! Using the wrong device (like your system drive) can result in catastrophic data loss.

2 Create a Raw Image with dd

▼

Now we'll create a bit-for-bit copy of the evidence drive using the dd command with appropriate forensic parameters.

# Create forensic image with dd
sudo dd if=/dev/sdb of=/cases/lab6/evidence.raw bs=4096 conv=noerror,sync status=progress

Basic dd command for forensic imaging

Command Parameters Explained

Parameter Meaning Forensic Purpose
if=/dev/sdb Input file (source) The evidence drive to image
of=evidence.raw Output file (destination) The forensic image file
bs=4096 Block size (4KB) Optimal for modern drives
conv=noerror Continue on read errors Captures as much data as possible
conv=sync Pad error blocks with zeros Maintains image size integrity
status=progress Show progress Monitor imaging progress
💡 Pro Tip: Using dcfldd

For forensic work, consider using dcfldd which adds hashing on-the-fly:

# Enhanced forensic imaging with dcfldd
sudo dcfldd if=/dev/sdb of=/cases/lab6/evidence.raw bs=4096 \
    hash=md5,sha256 hashwindow=1G hashlog=/cases/lab6/hash.log \
    conv=noerror,sync

dcfldd command with integrated hashing

3 Calculate Hash Values

▼

Hash values serve as digital fingerprints that prove the integrity of our evidence. We calculate hashes of both the source device and the image file to verify they match.

# Calculate MD5 hash of source device
sudo md5sum /dev/sdb
# Example: d41d8cd98f00b204e9800998ecf8427e  /dev/sdb

# Calculate MD5 hash of image file
md5sum /cases/lab6/evidence.raw
# Example: d41d8cd98f00b204e9800998ecf8427e  evidence.raw

# Calculate SHA-256 hash of source device
sudo sha256sum /dev/sdb

# Calculate SHA-256 hash of image file
sha256sum /cases/lab6/evidence.raw

Commands to calculate MD5 and SHA-256 hash values

Part 2: Imaging with FTK Imager (Windows)

FTK Imager is a free, GUI-based forensic imaging tool from AccessData. It creates images in the E01 (Expert Witness Format) which includes built-in compression and hash verification.

4 Create E01 Image with FTK Imager

▼

Follow these steps to create an E01 format image using FTK Imager:

  1. Connect the same USB drive via write blocker to Windows workstation
  2. Launch FTK Imager as Administrator
  3. Select File → Create Disk Image
  4. Choose Physical Drive as the source type
  5. Select the evidence drive from the dropdown list
  6. Click Add to specify image destination
  7. Select E01 as the image type

Evidence Item Information

Enter the following case information when prompted:

Image Settings

  • Image Fragment Size: 0 (no fragmentation) or 2GB for large images
  • Compression: 6 (balanced)
  • Check "Verify images after they are created"
  • Check "Create directory listings"

Part 3: Image Verification and Format Comparison

6 Compare Image Formats

▼

Compare the raw (dd) and E01 (FTK Imager) formats you created:

📁 Raw Format (dd)

  • Exact bit-for-bit copy
  • No compression
  • No built-in metadata
  • Universal compatibility
  • File size = source size

📦 E01 Format (FTK)

  • Expert Witness Format
  • Built-in compression
  • Embedded case metadata
  • Integrated hash verification
  • File size often smaller

File Size Comparison

# Check raw image size
ls -lh /cases/lab6/evidence.raw

# Check E01 image size (on Windows or after copying)
# E01 files may be split: evidence.E01, evidence.E02, etc.
dir evidence.E01

Troubleshooting Common Issues

🔧 dd: Permission denied

Use sudo before the dd command. Raw device access requires root privileges.

🔧 Hash mismatch between source and image

Ensure write blocker is functioning. Re-image the drive. Check for drive errors using dmesg.

🔧 FTK Imager doesn't see the drive

Run FTK Imager as Administrator. Check Device Manager for drive detection. Try a different USB port.
