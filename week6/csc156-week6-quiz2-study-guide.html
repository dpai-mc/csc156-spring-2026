Flashcards

Q: What is the TRIM command and why does it matter forensically? Click to reveal
A: TRIM is an ATA command that tells an SSD which data blocks are no longer in use and can be wiped internally. Forensically, this is significant because the SSD may erase data before an examiner can acquire it, making recovery of deleted files much more difficult or impossible compared to traditional HDDs.
Q: What is slack space and why is it forensically significant? Click to reveal
A: Slack space is the unused space between the end of a file and the end of the cluster it occupies. It's forensically significant because it may contain remnants of previously deleted files, providing valuable evidence that wasn't intentionally preserved.
Q: What is LBA and how does it differ from CHS addressing? Click to reveal
A: LBA (Logical Block Addressing) assigns a linear address to each sector on a drive, starting from 0. CHS (Cylinder-Head-Sector) was the older method that referenced physical geometry. LBA is simpler, supports larger drives, and is used by all modern systems.

1 Practice Question

Which component of an SSD is responsible for managing data placement, wear leveling, and garbage collection?

  • A NAND flash chips
  • B SSD Controller
  • C SATA interface
  • D Cache memory

The SSD Controller is the "brain" of the SSD, managing all operations including data placement across NAND chips, wear leveling to extend drive life, garbage collection to reclaim space, and handling the TRIM command. Understanding this is crucial for forensic examiners because the controller's operations can affect data recoverability.

Flashcards

Q: What is the MFT and what forensic information does it contain? Click to reveal
A: The Master File Table (MFT) is a database in NTFS that contains metadata for every file and directory on the volume. Forensically valuable information includes: file names, timestamps (MACB), file sizes, security descriptors, and data locations. Even deleted files often retain MFT entries, making recovery possible.
Q: How does FAT32 mark a file as deleted? Click to reveal
A: FAT32 marks a file as deleted by replacing the first character of the filename in the directory entry with 0xE5 (σ). The FAT chain entries are marked as available but the actual data remains on disk until overwritten, enabling recovery if acted upon quickly.
Q: What is file carving and when is it used? Click to reveal
A: File carving is a technique for recovering files from unallocated space by searching for file signatures (headers and footers) rather than relying on file system metadata. It's used when file system structures are damaged, when files have been deleted and metadata overwritten, or when analyzing raw disk images.

2 Practice Question

In NTFS, which system file contains a record of all changes made to the file system for recovery purposes?

  • A $MFT
  • B $LogFile
  • C $Bitmap
  • D $Volume

$LogFile is the NTFS journal that records all transactions (changes) to the file system. This enables recovery after system crashes and is forensically valuable because it may contain records of file operations even after files are deleted. $MFT contains file metadata, $Bitmap tracks cluster allocation, and $Volume contains volume information.

Flashcards

Q: Why do forensic examiners calculate multiple hash values (e.g., both MD5 and SHA-256)? Click to reveal
A: Using multiple hash algorithms provides defense in depth. While MD5 is faster and widely used historically, it's cryptographically broken (collisions possible). SHA-256 is more secure. If both hashes match, it provides stronger evidence of integrity. Additionally, some legacy systems or databases may only have MD5 hashes for comparison.
Q: What is the difference between a forensic image and a logical copy? Click to reveal
A: A forensic image is a bit-for-bit copy of the entire storage device, including all sectors, unallocated space, and slack space. A logical copy only copies files visible to the operating system. Forensic images preserve deleted files and hidden data; logical copies do not.
Q: What does the conv=noerror,sync option do in a dd command? Click to reveal
A: conv=noerror tells dd to continue imaging even if read errors occur (important for damaged media). conv=sync pads error blocks with zeros to maintain proper alignment and image size. Together, they ensure the imaging process completes and produces a usable image even from problematic drives.

3 Practice Question

A forensic examiner needs to image a suspect's laptop hard drive. The drive is encrypted with BitLocker and the laptop is currently powered on with the user logged in. What is the BEST acquisition approach?

  • A Immediately shut down the laptop and remove the hard drive for imaging
  • B Capture memory first, then create a logical image of the decrypted volume
  • C Wait for the user to log out, then image the encrypted drive
  • D Only collect browser history and documents visible on the desktop

When dealing with an encrypted system that is currently unlocked, the best approach is to capture memory first (which may contain encryption keys) and then create a logical image of the decrypted volume. Shutting down would lock the encryption and potentially make data inaccessible. This is a live acquisition scenario where preserving volatile data is critical.

💡 Common Mistakes to Avoid
  • Confusing forensic imaging with logical copying
  • Forgetting that SSDs behave differently than HDDs for forensics
  • Mixing up NTFS system files ($MFT, $LogFile, $Bitmap)
  • Not understanding when to use hardware vs software write blockers
  • Forgetting the importance of hash verification before AND after imaging
