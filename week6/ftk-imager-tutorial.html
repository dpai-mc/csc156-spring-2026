FTK Imager by Exterro

Learn to download, install, and use one of digital forensics’ most essential free imaging tools — trusted by law enforcement, legal professionals, and cybersecurity practitioners worldwide.

⏰ Estimated time: 60–90 minutes
💻 Windows / macOS / Linux
💰 Free tool
✊ Beginner friendly

FTK Imager is a free forensic imaging and previewing tool developed by Exterro (formerly AccessData). It is one of the most widely used tools in digital forensics for creating forensically sound copies of digital evidence, previewing file systems, and generating hash values to verify integrity.

🕵

Forensic Significance

FTK Imager creates bit-for-bit copies of storage media, meaning every bit from the original device is duplicated exactly — including deleted files, slack space, and unallocated clusters. This makes it court-admissible evidence when proper procedures are followed.

Key Capabilities

💾

Disk Imaging

Create forensic images in Raw (dd), E01, AFF, and SMART formats

🔑

Hash Verification

Generate and verify MD5, SHA-1, and SHA-256 hash values automatically

🔍

Evidence Preview

Browse file systems, view deleted files, and preview content before imaging

🤖

Memory Capture

Acquire volatile memory (RAM) from live Windows systems for analysis

📄

Report Generation

Export case reports with hash values, file listings, and metadata

📡

Free & Portable

No license cost; portable version runs from USB without installation

Always Download from the Official Source

Only download FTK Imager from Exterro’s official website. Third-party downloads may be modified, infected, or outdated. Using an unverified tool on evidence media could compromise the integrity of your investigation.

Exterro acquired AccessData (original developer of FTK and FTK Imager) and now distributes FTK Imager as a free standalone tool. A free registration is required to download.

Requirements

Windows 10 / 11 (64-bit), Administrator privileges, 4 GB RAM minimum (8 GB recommended), 500 MB free disk space.

  1. Run the Installer

    Right-click FTK_Imager_4.7.3.81.exe and select “Run as administrator”. This is required for proper driver installation. Accept the UAC prompt.

  2. Accept the License Agreement

    Read and accept the End User License Agreement (EULA). FTK Imager is free for educational and professional use. Click Next.

  3. Choose Installation Directory

    The default path is recommended:

    Path
    C:\Program Files\AccessData\FTK Imager\

    Click Install. Installation typically takes 30–60 seconds.

  4. Complete Installation & Launch

    When installation completes, click Finish. You can optionally check “Launch FTK Imager” to open it immediately. A shortcut will be placed on your desktop and in the Start menu.

  5. Verify Installation

    PowerShell
    Get-Command "C:\Program Files\AccessData\FTK Imager\FTK Imager.exe"

    Confirm the version number appears in Help → About FTK Imager.

⚠

ARM Mac (Apple Silicon) Note

FTK Imager for macOS runs via Rosetta 2 on M1/M2/M3 chips. Functionality is limited compared to the Windows version. For full features, use Windows (via Boot Camp or a VM).

  1. Open the DMG File

    Double-click FTK_Imager_macOS_4.7.3.81.dmg. Drag the FTK Imager icon into your Applications folder.

  2. Allow Gatekeeper

    On first launch, macOS may block the app. Go to System Preferences → Security & Privacy → General and click “Open Anyway” next to the FTK Imager warning.

    Terminal (alternative)
    xattr -d com.apple.quarantine /Applications/FTK\ Imager.app

  3. Grant Disk Access

    Go to System Preferences → Security & Privacy → Privacy → Full Disk Access. Click the lock icon, then add FTK Imager. This is required for reading disk devices.

Forensic Principle: Write Protection

Always attach a write blockerA hardware or software device that prevents any data from being written to an evidence drive, preserving its integrity. before connecting evidence media to your workstation. FTK Imager includes software write-blocking, but a hardware write blocker is preferred for chain of custody.

Imaging Workflow

  1. Launch Create Disk Image

    Go to File → Create Disk Image… or click the imaging toolbar icon. The source selection dialog appears.

  2. Select the Source

    Choose the type of source media:

    • Physical Drive — Image an entire physical disk (recommended for most cases)
    • Logical Drive — Image a specific partition/volume
    • Image File — Re-image from an existing forensic image
    • Contents of a Folder — Logical acquisition of a directory
    • Fernico Device / Multiple CDs/DVDs — Optical media

  3. Configure Image Destination & Format

    Click Add… to set the output destination. Select your image format:

    Create Image

    Select Image Type:

    Image Destination: E:\Cases\Case-2025-001\Evidence\
    Image Filename: evidence_001
    Segment Size (MB): 2047
    FormatExtensionBest ForCompression
    Raw (dd).dd / .rawMaximum compatibility, Linux toolsNone
    E01.E01Court cases, EnCase compatibilityYes
    AFF.affOpen format, metadata embeddingYes
    SMART.s01AccessData/FTK ecosystemYes
    ✓

    Recommended: E01 Format for Court Cases

    E01 is the industry standard. It supports compression (saving storage space), embedded hash values, and case metadata — all in a single container file. It is widely accepted by prosecutors and courts.

  4. Enter Case Information

    For E01 images, you will be prompted to enter case metadata embedded directly in the image file:

    • Case Number: Your case identifier (e.g., 2025-FE-001)
    • Evidence Number: Evidence item identifier (e.g., E001)
    • Unique Description: Brief description (e.g., Suspect laptop HDD - 500GB)
    • Examiner: Your full name and badge/employee number
    • Notes: Additional context (date, location seized, etc.)
    🕵

    This metadata is embedded in the E01 file and cannot be changed after creation without invalidating the hash. Fill it in accurately before clicking Start.

  5. Monitor Progress and Completion

    FTK Imager displays a progress dialog showing:

    • Elapsed and estimated time remaining
    • Data written and percentage complete
    • Current imaging speed (MB/s)
    • Bad sectors encountered (document any bad sectors in your case notes)

    Imaging a 500 GB drive typically takes 1–3 hours depending on USB speed and drive health. Do not disconnect the drive or interrupt the process.

Hash verificationThe process of computing a cryptographic hash of the original and copy, then comparing them to confirm they are identical. is the cornerstone of forensic integrity. After imaging completes, FTK Imager automatically computes hash values of both the source and the image, then compares them.

🔓

MD5

128-bit hash. Fast, widely accepted. Collision vulnerabilities exist but irrelevant for forensic integrity checks.

🔒

SHA-1

160-bit hash. More robust than MD5. Being phased out of some contexts but still accepted in court.

🔐

SHA-256

256-bit hash. Current gold standard. Recommended for all new cases. Computationally intensive but definitive.

Once you have a forensic image, you can load it into FTK Imager for examination without modifying the original. This is how you begin the analysis phase of an investigation.

  1. Add Evidence Item

    Go to File → Add Evidence Item… Select the source type:

    • Image File — Load an existing .E01, .dd, or .aff image
    • Physical Drive — Directly browse a write-blocked drive
    • Logical Drive — Browse a specific volume/partition
    • Contents of a Folder — Browse a specific directory

  2. Browse the Evidence Tree

    Once loaded, expand the evidence tree to see the full file system. Key areas to examine:

    • [root] — Top-level drive/partition
    • [unallocated space] — Areas where deleted files reside
    • [orphan files] — Files with lost parent directory references
    • $Recycle.Bin — Recycle Bin contents (Windows)
    • Windows\System32 — Operating system files
    • Users\[name]\AppData — User application data (browser history, etc.)

  3. View File Properties & Hashes

    Click any file to view its properties in the bottom-right panel. Right-click → Export File Hash List to generate a full hash manifest of all files.

    ℹ

    Files highlighted in red in the file list are deleted files that have been recovered from the file system metadata (MFT entries). These are still forensically accessible even after deletion.

  4. Export Files of Interest

    Right-click any file or folder → Export Files… Choose your destination. Exported files should always go into a dedicated working copy folder, never back to the evidence drive.

    Recommended Folder Structure
    Cases/
└── Case-2025-001/
    ├── Evidence/          ← Original images (read-only)
    ├── WorkingCopy/       ← Exported items for analysis
    ├── Reports/           ← FTK Imager reports
    └── Notes/             ← Investigator case notes

Generating the Verification Report

  1. Go to File → Report

    After loading or creating an image, this option becomes available.

  2. Select Output Location

    Save to your Cases/[CaseNum]/Reports/ folder. The report is a plain text file compatible with any word processor.

  3. Review Report Contents

    The report includes: case information, tool version, source device details, image format, segment sizes, hash values for source and image, verification result, date/time stamp, and examiner name. Print and sign a physical copy for your chain of custody documentation.

The ftkimager command-line tool (available on Linux and Windows) allows scripted, automated acquisitions — useful for large-scale investigations or when a GUI is not practical.

Syntax

Syntax
ftkimager [source] [destination] [options]

Common Examples

Create a raw (dd) image from a physical drive:

Bash
sudo ftkimager /dev/sdb /cases/case001/evidence_001 \
  --case-number "2025-FE-001" \
  --evidence-number "E001" \
  --description "Suspect USB drive - 32GB" \
  --examiner "Jane Smith" \
  --compress

Create an E01 image:

Bash
sudo ftkimager /dev/sdb /cases/case001/evidence_001 \
  --e01 \
  --case-number "2025-FE-001" \
  --evidence-number "E001" \
  --compress \
  --verify

Compute file hash only (no imaging):

Bash
ftkimager /dev/sdb --hash-logfile /cases/case001/hashes.txt

Windows — Image a physical drive:

cmd (Run as Administrator)
"C:\Program Files\AccessData\FTK Imager\ftkimager.exe" ^
  \\.\PhysicalDrive1 ^
  E:\Cases\Case-2025-001\Evidence\evidence_001 ^
  --e01 --compress --verify ^
  --case-number "2025-FE-001" ^
  --evidence-number "E001" ^
  --examiner "John Smith"

Key CLI Options

OptionDescription
--e01Output in E01 (Expert Witness) format
--compressEnable compression (E01 / SMART only)
--verifyVerify image after creation (compares hashes)
--frag [size]Set segment size in MB (default: 1500)
--case-numberEmbed case number in E01 metadata
--evidence-numberEmbed evidence number in E01 metadata
--examinerEmbed examiner name in E01 metadata
--descriptionEmbed evidence description in E01 metadata
--hash-logfileWrite hash values to a log file
--list-drivesList all available drives on the system
