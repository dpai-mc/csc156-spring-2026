Phase 4: Examination

üî¨

What is Examination?

Examination is processing forensic images to extract relevant artifacts and data, making evidence accessible for analysis. Think of it as "mining" the data‚Äîyou're extracting ore (artifacts) from the mountain (forensic image), but you haven't yet refined it into gold (conclusions).

Critical Distinction: Examination vs. Analysis

Examination

"What?"

Focus: Technical extraction

Activities:

  • Running forensic tools
  • Recovering deleted files
  • Extracting artifacts
  • Creating file listings
  • Generating timelines

Output:

  • Extracted data
  • File listings
  • Artifact collections
  • Tool reports

Question Answered:
"What data exists?"

Skill Emphasis:

  • Tool proficiency
  • Technical knowledge
  • Systematic processing

Analysis

"So what?"

Focus: Interpretation

Activities:

  • Reconstructing events
  • Testing hypotheses
  • Drawing conclusions
  • Determining significance
  • Establishing attribution

Output:

  • Timeline narrative
  • Findings summary
  • Conclusions
  • Recommendations

Question Answered:
"What does it mean?"

Skill Emphasis:

  • Critical thinking
  • Investigative reasoning
  • Pattern recognition

Examination = "Finding all the puzzle pieces and sorting them by color, edge pieces vs. interior"

Analysis = "Assembling the puzzle to see the complete picture and understanding what it depicts"

Examination = "Archaeological dig‚Äîcarefully excavating artifacts from the site"

Analysis = "Interpreting the artifacts‚Äîwhat do they tell us about the civilization?"

Why This Distinction Matters:

  • Different skill sets required (technical vs. analytical)
  • Different tools used (forensic software vs. brain + spreadsheet)
  • Different outputs (data vs. conclusions)
  • Both are essential‚Äîcan't analyze what you haven't examined
  • Examination can be partially automated; analysis requires human judgment

Key Examination Techniques

üìÇ File System Analysis

‚ñº

What It Is: Examining file system structures to understand how data is organized, identify artifacts, and recover information from deleted files and unallocated space.

File Metadata (MAC Times)

  • Modified: When file contents last changed
  • Accessed: When file was last opened or read
  • Created: When file was created on this file system
  • (Windows also tracks Birth/Created time in $MFT)
File: confidential_project.docx
Created: 2025-01-10 09:30:00 UTC
Modified: 2025-01-15 14:22:00 UTC
Accessed: 2026-01-28 10:15:00 UTC
Size: 2,458,624 bytes (2.4 MB)
Owner: DOMAIN\jsmith
MD5: d41d8cd98f00b204e9800998ecf8427e

üí° Created Date (Jan 10): File first appeared on this system on January 10, 2025. This is when it was created OR when it was copied from another location.

üí° Modified Date (Jan 15): File contents were changed on January 15‚Äîfive days after creation. Someone edited this document.

üí° Accessed Date (Today): File was opened or read TODAY (Jan 28, 2026). This recent access suggests the file is relevant to current investigation.

File System Structures

Windows NTFS:

  • $MFT (Master File Table): Database of all files on volume, includes deleted file entries
  • $LogFile: Transaction log showing file system changes
  • $UsnJrnl: Update Sequence Number Journal tracking file modifications
  • Alternate Data Streams (ADS): Hidden data attached to files

Linux ext3/ext4:

  • Inodes: File metadata structures
  • Journal: File system transaction log
  • Superblock: File system metadata and configuration

üìÖ Timeline Creation

‚ñº

What It Is: Creating chronological sequences of events from file system metadata, log files, and other timestamped artifacts to reconstruct what happened and when.

Why Timelines Matter

  • Show sequence of events (cause and effect)
  • Identify gaps or anomalies in activity
  • Correlate activity across multiple sources
  • Establish timeframes for incidents
  • Support or refute alibis and claims

üí° Super Timeline: A comprehensive timeline combining ALL timestamped artifacts from a system‚Äîfile system, registry, logs, browser history, etc. Tools like log2timeline/plaso can create these automatically.

Timeline Analysis Workflow

  1. Extract timestamps from all available sources
  2. Normalize timestamps to common timezone (usually UTC)
  3. Combine into single timeline sorted chronologically
  4. Filter for relevant timeframes (incident window)
  5. Look for patterns and anomalies

‚ö†Ô∏è Timestamp Limitations: Timestamps can be manipulated (timestomping). Always corroborate with other evidence.

üîç Keyword Searching

‚ñº

What It Is: Searching forensic images for specific text strings, patterns, or regular expressions to locate relevant evidence quickly without manually reviewing every file.

When to Use Keyword Searching

  • Looking for specific terms related to investigation (project names, code words)
  • Finding communication about specific topics
  • Locating credit card numbers, SSNs, or other PII
  • Identifying malware indicators (command and control domains)
  • Finding evidence of specific activities

Search Strategy

Start Broad, Then Narrow:

  1. Begin with obvious keywords from case details
  2. Review initial results for context
  3. Identify additional keywords from results
  4. Refine searches with more specific terms
  5. Use Boolean operators (AND, OR, NOT) for precision

‚ö†Ô∏è False Positives: Keywords will find unrelated matches. Always verify context.

#Ô∏è‚É£ Hash Analysis

‚ñº

What It Is: Using cryptographic hash values (digital fingerprints) to identify known files, detect changes, and filter out irrelevant data.

Hash Applications

1. Known File Filtering (NSRL)

The National Software Reference Library maintains hash sets of billions of known files. Examiners can exclude matches to focus on unique/suspicious files.

2. Known Bad File Identification

Hash sets of malware and contraband allow automatic flagging:

  • VirusTotal - malware hashes
  • Project VIC - contraband identification
  • Custom organizational hash sets

3. Data Deduplication

Identify duplicate files across systems. If 1000 copies of same file exist, you only need to examine one.

4. Change Detection

Compare file hashes before/after to detect modifications.

Hash filtering is like: Security checking everyone entering a building. Most have valid IDs (NSRL) and wave through. A few are on the watch list (malware) and detained. Everyone else gets closer scrutiny.

‚ôªÔ∏è Data Recovery & Carving

‚ñº

What It Is: Recovering deleted files and file fragments from unallocated space using file carving techniques.

How File Deletion Works

When you "delete" a file:

  1. File system marks the space as "available for reuse"
  2. Directory entry removed or marked deleted
  3. File contents remain intact until overwritten

üí° Key Insight: Deletion is lazy. File systems don't actually erase data immediately‚Äîthey just mark the space as available. This is why file recovery is possible.

File Carving

File carving recovers files from raw data by identifying file signatures (magic numbers):

File Type Header Signature
JPEG Image FF D8 FF
PNG Image 89 50 4E 47
PDF Document 25 50 44 46 (%PDF)
ZIP Archive 50 4B 03 04

‚ö†Ô∏è Limitations: Fragmented files are difficult to carve. Overwritten data cannot be recovered. Encrypted files appear as random data.

Phase 5: Analysis

üß†

What is Analysis?

Analysis is interpreting examination results to answer investigative questions, draw conclusions, and determine the significance of evidence. While examination asks "What data exists?", analysis asks "What does it mean?"

From Data to Meaning

Analysis transforms raw artifacts into actionable intelligence through:

  • Pattern Recognition: Identifying relationships and trends
  • Timeline Reconstruction: Building narrative from events
  • Hypothesis Testing: Confirming or refuting theories
  • Correlation: Connecting evidence across sources
  • Attribution: Determining who did what

Analytical Thinking Framework

The "5 W's + H" of Forensic Analysis:

  • Who: Who performed the action? (Attribution)
  • What: What happened? What was affected? (Incident scope)
  • When: When did it occur? Timeline? (Temporal analysis)
  • Where: Where did it happen? Which systems? (Location)
  • Why: What was the motive? (Intent analysis)
  • How: What methods were used? (TTPs)

Analysis is like detective work: You have clues (examination results), but you need to figure out what they mean, how they fit together, and what story they tell.

Phase 6: Presentation

üìä

What is Presentation?

Presentation is communicating forensic findings to stakeholders in a clear, accurate, and persuasive manner‚Äîwhether through written reports, oral testimony, or visual presentations. Even perfect examination and brilliant analysis are worthless if you can't explain what you found.

Why Presentation Matters

  • Technical findings must be accessible to non-technical audiences
  • Evidence must be presented persuasively to support decisions
  • Complex investigations must be distilled into clear, actionable conclusions
  • Your credibility as an examiner is judged by communication
  • Documentation creates permanent record for legal proceedings

Presentation Formats

Written Reports

Components:

  • Executive Summary
  • Case Background
  • Evidence Inventory
  • Methodology
  • Findings & Analysis
  • Conclusions
  • Technical Appendices

Key Attributes:

  • Permanent record
  • Detailed and comprehensive
  • Supports legal discovery
  • Multiple audience levels

Oral Testimony

Contexts:

  • Courtroom testimony
  • Depositions
  • Arbitration hearings
  • Executive briefings
  • Board presentations

Key Attributes:

  • Interactive (Q&A)
  • Requires credibility
  • Simplified language
  • Visual aids helpful
  • Cross-examination ready

Effective Forensic Communication

Principle 1: Know Your Audience

Audience What They Need Communication Approach
Executive Leadership Business impact, risk, decisions High-level summary, financial impact, minimal technical detail
Legal Counsel Evidence strength, legal implications Chain of custody, methodology, confidence levels
Technical Peers Methodology, tools, detailed findings Technical depth, tool output, hash values
Jury Simple explanations, credibility Analogies, avoid jargon, visual aids

Principle 2: Structure for Clarity

üí° The Inverted Pyramid: Start with conclusions, then provide supporting details.

  1. Executive Summary: Bottom line up front
  2. Key Findings: Most important evidence
  3. Supporting Analysis: How you reached conclusions
  4. Methodology: How you examined evidence
  5. Technical Details: Appendices

Principle 3: Be Objective and Defensible

Maintaining Objectivity:

  • State facts, not opinions (unless expert opinion requested)
  • Present evidence, let others draw legal/business conclusions
  • Acknowledge limitations and alternative explanations
  • Use confidence levels appropriately
  • Separate observation from interpretation

üí° Defensible Statement: "Based on file timestamps and browser history, the user accessed the customer database at 2:14 PM and created an Excel file at 2:18 PM. This sequence is consistent with data exfiltration, though other explanations are possible."

‚ö†Ô∏è Not Defensible: "The user definitely stole customer data because they're guilty."

Problem: Conclusive language, assumes intent, not supported by technical evidence alone.
