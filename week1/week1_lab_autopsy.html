Part 1: Installing Autopsy (30 minutes)

‚öôÔ∏è Installation Steps

  1. Download Autopsy Navigate to https://www.sleuthkit.org/autopsy/ and download the latest version for your operating system (Windows or Linux).
  2. Run the Installer For Windows: Run the .exe installer with administrator privileges. For Linux: Follow the installation instructions for your distribution. Accept the default installation path unless you have specific requirements.
  3. Verify Installation Launch Autopsy from the Start Menu (Windows) or applications menu (Linux). You should see the Autopsy welcome screen.
  4. Document Version Information In your lab notebook, record:
    • Autopsy version number
    • Installation date and time
    • Operating system and version
    • Computer name/workstation ID
üí° Pro Tip

Always document tool versions in forensic work. This information is critical for court admissibility and reproducibility of results. Include this in all case reports.

Part 2: Creating Your First Case (20 minutes)

üìÅ Case Creation Process

  1. Launch Autopsy Open Autopsy and you'll see the welcome screen with options to create a new case or open an existing one.
  2. Start New Case Click "New Case" and fill in the case information form:
    Case Name: Lab1_FirstName_LastName Base Directory: C:\ForensicCases\ (or your preferred location) Case Number: CSC156-W1-L1 Examiner Name: [Your Full Name] Organization: Mendocino College Phone: [Your Phone] Email: [Your Email] Notes: Week 1 Lab - Introduction to Autopsy
  3. Configure Case Settings On the next screen, choose "Single-user case" (recommended for this lab). Review and click "Finish."
  4. Add Data Source Autopsy will prompt you to add a data source:
    • Select "Disk Image or VM File"
    • Browse to your downloaded forensic image file (e.g., downloaded .dd, .E01, or .001 file)
    • Click "Next"
  5. Configure Ingest Modules For this introductory lab, we'll keep all default ingest modules enabled. These include:
    • File Type Identification
    • EXIF Parser
    • Hash Lookup
    • Keyword Search
    • Recent Activity
    Click "Next" and then "Finish" to begin ingestion.
  6. Wait for Ingestion A small forensic image (under 200MB) should complete ingestion in 2-10 minutes depending on size. Watch the progress bar in the lower-right corner. You can begin exploring the interface while ingestion continues.
‚è±Ô∏è Time Expectation

Small forensic images (under 200MB) are intentionally chosen to ensure quick processing. Typical ingestion time is 2-10 minutes, allowing you to focus on analysis rather than waiting. Larger images from some sources may take longer.

Part 3: Navigating the Autopsy Interface (30 minutes)

üó∫Ô∏è Interface Components

The Autopsy interface has five main areas you need to understand:

Component Location Purpose
Tree Viewer Left panel Navigate data sources, file system, and results
Result Viewer Upper right Display contents of selected items (table/thumbnail view)
Content Viewer Lower right View file contents, metadata, hex, strings
Keyword Search Upper toolbar Search for text strings across the image
Status Bar Bottom Shows ingestion progress and background tasks
  1. Explore the Tree Viewer In the left panel, expand the data source tree:
    • Expand "Data Sources"
    • Expand the USB image
    • Explore the file system hierarchy
    • Notice the volume and partition information
  2. Browse Files and Folders Navigate through the directory structure. Find and note:
    • Number of allocated files
    • Number of deleted files (if any)
    • Types of files present
    • Any interesting folder names
  3. Examine File Metadata Select any file in the Result Viewer. In the Content Viewer (bottom right), switch between tabs:
    • File Metadata: Timestamps, size, permissions
    • Hex: Raw hexadecimal view
    • Strings: Extracted text strings
    • Application: Rendered view (for viewable files)
  4. Explore Results In the Tree Viewer, expand "Results":
    • Extracted Content (emails, documents, etc.)
    • Web History (browser artifacts)
    • Recent Documents
    • Hash Set Hits
üîç Investigation Tip

The "Results" section in Autopsy automatically categorizes important artifacts. This is where you'll find critical evidence like web history, emails, and recently accessed documents without manually searching the entire file system.

Part 4: Hands-On Analysis Exercise (60 minutes)

üîé Investigation Scenario

üìã Case Background

You've been provided with a USB drive image from a hypothetical investigation. Your task is to analyze the drive and answer the investigative questions below. Document all findings in your lab notebook with supporting screenshots.

‚ùì Investigative Questions

Complete the following analysis tasks and document your findings:

  1. File System Analysis
    • What file system is used on this USB drive?
    • What is the total capacity of the drive?
    • How many allocated (active) files are present?
    • Are there any deleted files? If so, how many?
    Hint: Look at the volume information in the tree viewer.
  2. File Type Identification
    • What types of files are present? (documents, images, executables, etc.)
    • Navigate to "Results" ‚Üí "File Types" to see a categorized breakdown
    • Are there any files with mismatched extensions? (e.g., a .jpg that's actually a .txt)
    Hint: Autopsy's File Type Identification module detects files by content, not just extension.
  3. Timeline Analysis
    • Find the oldest file on the drive (by creation date)
    • Find the most recently modified file
    • What is the date range of activity on this drive?
    Hint: You can sort files by clicking column headers in the Result Viewer.
  4. Keyword Search
    • Use the keyword search feature (top toolbar) to search for: "password"
    • How many hits did you get?
    • In which files does this keyword appear?
    • Try searching for other keywords: "confidential", "secret", your name
    Hint: Keyword search looks through file contents and metadata.
  5. Web Artifacts
    • Navigate to "Results" ‚Üí "Web History"
    • Are there any browser artifacts present?
    • If so, what websites were visited?
    • What browser was used?
  6. Interesting Files
    • Check "Results" ‚Üí "Interesting Items"
    • What has Autopsy flagged as potentially interesting?
    • Why might these items be relevant to an investigation?
  7. File Content Examination
    • Select any text document
    • View it in the "Application" tab of the Content Viewer
    • Switch to "Hex" view - can you identify the file header?
    • Switch to "Strings" view - what readable text appears?
  8. Metadata Analysis
    • Select any image file (if present)
    • View the EXIF data in the Content Viewer
    • What metadata is available? (camera make, GPS coordinates, etc.)
    • Why is this metadata important in investigations?
üì∏ Documentation Requirement

For each question answered, capture a screenshot showing the evidence. Use Autopsy's built-in screenshot tool or your OS screenshot utility. Annotate screenshots to highlight relevant findings.

Part 5: Generating Case Reports (20 minutes)

üìÑ Creating Your First Forensic Report

  1. Access Report Generation In Autopsy, go to: Tools ‚Üí Generate Report
  2. Select Report Type Choose "HTML Report" for this lab (easiest to view and share). Click "Next."
  3. Configure Report Settings
    • Select which data to include (all categories for comprehensive report)
    • Choose output location
    • Add your name as the report author
  4. Generate and Review Click "Finish" and wait for report generation. Open the HTML report in your browser and review:
    • Case summary information
    • Data source details
    • File listings and results
    • Tagged items (if any)
‚ÑπÔ∏è Report Purpose

Forensic reports document your investigation process and findings. They must be comprehensive, accurate, and reproducible. In real cases, these reports may be submitted as legal evidence.

Post-Lab Questions

üí≠ Reflection and Analysis

Answer the following questions in your lab report:

  1. What is the purpose of creating a case in Autopsy before analyzing evidence?

    Think about documentation, organization, and legal requirements.

  2. Why is it important to use ingest modules when processing a disk image?

    Consider what might be missed if you only manually browsed files.

  3. How does Autopsy identify files differently than just looking at file extensions?

    Think about file headers and magic numbers.

  4. What advantages does Autopsy offer compared to manually browsing a mounted drive in Windows Explorer or Finder?

    Consider deleted files, metadata, timelines, and artifact extraction.

  5. Why is it critical to document the version of forensic tools used in an investigation?

    Think about reproducibility and legal defensibility.
