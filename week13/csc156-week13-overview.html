Week 13: Malware Forensics

From examining artifacts left behind — to examining the malicious tool itself.

30 Points This Week  |  Lab 12

Welcome to Week 13

For the past several weeks you have been reconstructing what happened on a system — parsing registry hives, reading event logs, recovering deleted files, analyzing memory dumps, and tracing network traffic. This week the analytical lens shifts: instead of studying the traces an attacker left behind, you will study the weapon itself.

Malware forensics asks a different set of questions. What does this executable do? How does it persist? What network infrastructure does it contact? How can we detect it on every other machine in the enterprise? Answering those questions requires a blend of the skills you have already built — and a new, disciplined methodology for safely examining hostile code.

Key shift: Weeks 7–12 focused on artifacts produced by activity. Week 13 focuses on analyzing the artifact that is the threat — the malware sample itself.

What You Will Be Able to Do

By the end of this week you should be comfortable with the following:

  • Classify a piece of malware by type and behavior — virus, worm, trojan, ransomware, RAT, rootkit, or fileless — and explain what each type means for your investigation.
  • Perform static analysis on a suspicious file: extract strings, examine PE headers, review import tables, and look up file hashes for known threats.
  • Set up an isolated sandbox environment and conduct dynamic analysis, observing what the malware actually does when it runs — file changes, registry writes, process creation, and network calls.
  • Extract and document Indicators of Compromise (IOCs) in a structured format that could be shared with an incident-response team.
  • Recognize common anti-analysis techniques — packers, obfuscation, anti-VM checks, and anti-debugging — and understand how analysts detect and work around them.
  • Connect your malware findings back to evidence from earlier forensic domains: memory dumps, packet captures, Windows and Linux system artifacts.

Building on What You Already Know

Week 13 does not start from scratch. You already have a strong foundation — this week applies it in a new context.

Week 11 – Memory Forensics

  • You used Volatility’s malfind plugin to detect injected code and suspicious processes in RAM. This week you take the next step: once a suspicious file is identified, how do you analyze it?
  • Volatility’s netscan and cmdline output will help you correlate memory-resident artifacts with the malware sample’s behavior.

Week 12 – Network Forensics

  • You identified C2 beaconing, data exfiltration patterns, and suspicious DNS queries in PCAPs. This week you connect those network indicators to the malware that generated them.
  • The same Wireshark skills from Module 12.2 apply directly when you monitor sandbox network traffic during dynamic analysis.

Weeks 7–9 – Windows & Linux Forensics

  • Registry Run keys, scheduled tasks, cron jobs, systemd units, SUID binaries — you already know these as persistence artifacts. Now you will watch malware create them in real time during dynamic analysis.
  • PE header analysis builds on the file-header and magic-byte concepts from Week 5, and hash verification reuses the integrity skills from Week 6.

Going Deeper

Malware is involved in the vast majority of security incidents that lead to forensic investigations. Ransomware, remote-access trojans, and info-stealers are not abstract threats — they are the artifacts you will encounter on compromised hard drives, in memory dumps, and in packet captures throughout your career.

Organizations need analysts who can go beyond running an antivirus scan. They need people who can determine what a piece of malware does, how it persists, what data it accessed, and how to detect it on every other endpoint. That is exactly the skill set you are building this week.

This week’s methodology — triage, static analysis, dynamic analysis, IOC extraction — is the same workflow used by analysts at incident-response firms, government agencies, and corporate security teams worldwide. It also directly supports the EC-Council CHFI and GIAC GCFA certification objectives covered in this course.
