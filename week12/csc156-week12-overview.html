Welcome to Week 12

Welcome to the final major evidence domain before we move into specialized forensics topics. For the past several weeks you have been investigating what happens on a system — Windows artifacts, Linux logs, and volatile memory. This week you shift your investigative lens to what happens between systems: the network traffic, server logs, and protocol-level indicators that reveal how attackers get in, what they do, and what data they take with them.

Week 12 integrates packet capture analysis, web server log investigation, firewall and IDS log interpretation, and introductory email header and DNS forensics into a single, fast-paced week. By the end, you will be able to correlate host-based evidence from earlier weeks with network-based evidence — completing the host-to-network investigation loop.

⚠️ High-Stakes Week: 60 Points This week carries two graded assessments worth a combined 60 points — Lab 11 (30 pts) and Quiz 3 (30 pts). Quiz 3 is cumulative, covering Windows Forensics (Weeks 7–8), Linux Forensics (Week 9), Memory Forensics (Week 11), and this week's Network Forensics. Plan your time carefully and start early.

Week 12 Learning Objectives

By the end of this week you should be able to:

  1. Capture and analyze network packets with Wireshark — apply display filters, follow TCP streams, and extract transferred objects from PCAP files. (Core network forensics skill)
  2. Distinguish normal from suspicious network traffic — recognize reconnaissance scans, command-and-control beaconing, and data exfiltration indicators in packet captures. (Core network forensics skill)
  3. Analyze web server logs for evidence of compromise — parse Apache, Nginx, and IIS access and error logs to identify attack signatures. (Core network forensics skill)
  4. Recognize common web attack indicators in log data — including SQL injection, directory traversal, cross-site scripting (XSS), and automated scanner signatures. (Core network forensics skill)
  5. Extract forensic evidence from network and application logs — including firewall logs, IDS/IPS alerts, and DNS query logs. (Core network forensics skill)
  6. Interpret email headers and DNS records at an introductory level — trace an email's routing path and identify forensically relevant DNS record types. (Foundation for Week 16 deep dive)
  7. Demonstrate readiness for Quiz 3 — synthesize and review concepts from Windows Forensics, Linux Forensics, Memory Forensics, and Network Forensics. (Cumulative assessment preparation)
ℹ️ CSLO Alignment This week primarily supports CSLO 1 (Describe the fundamental concepts of incident response and forensic investigation) and CSLO 2 (Conduct thorough examinations of computer hard disk drives, and other electronic data storage media to recover information and electronic data).

Key Concepts and Themes

By the end of Week 12, you should be able to explain each of these ideas clearly:

  • PCAP as evidence — A packet capture file is forensic evidence. Hash it, document the chain of custody, and never modify the original.
  • Wireshark display filters — The examiner's query language for isolating relevant traffic from thousands of packets.
  • TCP stream reconstruction — Reassembling full conversations (HTTP sessions, FTP transfers, cleartext credentials) from individual packets.
  • Web attack signatures in logs — Recognizing SQL injection, directory traversal, XSS, and scanner fingerprints in Apache, Nginx, and IIS logs.
  • Suspicious traffic patterns — Port scanning, C2 beaconing, DNS tunneling, and data exfiltration indicators in network captures.
  • Cross-source correlation — No single evidence source tells the whole story. Combining host artifacts (memory, event logs) with network evidence (PCAPs, firewall logs) produces a stronger investigation.
  • Email header tracing — Reading Received headers bottom-to-top to identify an email's true origin (expanded in Week 16).
