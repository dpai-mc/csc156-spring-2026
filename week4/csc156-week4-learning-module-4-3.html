üéØ Module Overview & Learning Objectives

üí°
Why Storage Interfaces Matter in Digital Forensics

Storage interfaces are the communication pathways between storage devices and the rest of the computer system. Understanding these interfaces is essential because they directly impact acquisition speed, write-blocking strategies, and tool compatibility. The interface you're working with determines which write blockers you can use, how fast you can image evidence, and what challenges you'll face during acquisition.

Learning Objectives

By the end of this module, you will be able to:

  1. Identify common storage interfaces including SATA, NVMe, and USB, and explain their technical characteristics
  2. Explain how different interfaces affect forensic acquisition including speed limitations, write-blocking requirements, and tool compatibility
  3. Describe RAID configurations and their forensic implications, including documentation requirements and reconstruction challenges
  4. Select appropriate write blockers based on interface type and investigation requirements
  5. Apply interface knowledge to make informed decisions about evidence collection strategies
  6. Document RAID configurations properly to prevent data loss during evidence collection
üîç
Connection to CSLO 2 and CSLO 3

This module supports CSLO 2: Conduct thorough examinations of computer hard disk drives and other electronic data storage media and CSLO 3: Perform data collection using forensic technology methods in accordance with evidence handling procedures. You cannot properly collect evidence without understanding how storage devices connect to systems and how to prevent writes during acquisition.

Critical Concept

The interface determines your acquisition strategy. A SATA HDD and an NVMe SSD require different write blockers, have vastly different acquisition speeds, and present different challenges. Understanding interfaces is not just academic‚Äîit's practical knowledge you'll use in every investigation.

üíø SATA (Serial ATA) Interface

What is SATA?

Serial ATA (SATA) is the most common interface for connecting internal storage devices (HDDs and SSDs) in desktop computers and laptops. Introduced in 2003 to replace the older parallel ATA (PATA/IDE) interface, SATA uses serial communication for faster, more reliable data transfer with simpler cabling.

Throughput: ~150 MB/s
Common Use: Legacy systems

  • Compatibility: All write blockers support SATA I
  • Prevalence: Rare in modern systems but found in older evidence

  • Compatibility: Excellent write blocker support

  • Note: SATA III is backward compatible with SATA I/II

SATA Physical Characteristics

  • Cable Type: 7-pin data cable (thin, flexible, red or black typically)
  • Power Connector: 15-pin SATA power connector
  • Cable Length: Maximum 1 meter (3.3 feet)
  • Hot-Swappable: Yes, when AHCI mode is enabled in BIOS
üí°
AHCI (Advanced Host Controller Interface)

AHCI is a mode that enables advanced SATA features including hot-swapping (connecting/disconnecting drives without shutdown) and Native Command Queuing (NCQ) for better performance. Most modern systems use AHCI mode by default.

Forensic Note: AHCI mode is beneficial for forensics as it enables easier drive connection/disconnection during examination.

Forensic Advantages of SATA

‚úÖ
Why SATA is "Forensically Friendly"
  • ‚úÖ Universal Write Blocker Support: Every hardware write blocker supports SATA
  • ‚úÖ Predictable Performance: Known speeds enable accurate acquisition time estimates
  • ‚úÖ Simple Connection: Standard cables and connectors, no adapters needed
  • ‚úÖ Mature Technology: Well-understood behavior, extensive tool support
  • ‚úÖ Hot-Swap Capability: Easy to connect/disconnect during examination
  • ‚úÖ Backward Compatible: SATA III ports work with SATA I/II drives

SATA Acquisition Time Estimates

Based on SATA III maximum throughput (~600 MB/s) in ideal conditions:

  • 250 GB drive: ~7-10 minutes
  • 500 GB drive: ~15-20 minutes
  • 1 TB drive: ~30-35 minutes
  • 2 TB drive: ~60-70 minutes
  • 4 TB drive: ~2-2.5 hours
‚ö†Ô∏è
Real-World Performance Note

These are theoretical maximums. Actual acquisition speeds depend on:

  • Drive health and condition
  • Source drive speed (HDD vs. SSD)
  • Destination drive speed
  • Write blocker overhead
  • USB-to-SATA adapter limitations (if used)
  • Bad sectors requiring retries

Expect 50-80% of theoretical maximum in practice (e.g., 300-480 MB/s for SATA III).

‚ö° NVMe (Non-Volatile Memory Express) Interface

What is NVMe?

NVMe (Non-Volatile Memory Express) is a modern storage interface protocol designed specifically for SSDs, utilizing PCIe (Peripheral Component Interconnect Express) lanes for extremely high-speed data transfer. NVMe represents a fundamental departure from SATA, bypassing legacy interfaces to communicate directly with the CPU through PCIe.

üí°
Why NVMe Exists

SATA was designed for mechanical hard drives with millisecond access times. SSDs have microsecond access times‚ÄîSATA became a bottleneck. NVMe was purpose-built for flash storage, enabling SSDs to reach their full performance potential.

  • Acquisition Speed: 5-6x faster than SATA III
  • Write Blockers: Limited but growing support
  • Prevalence: Common in laptops and desktops from 2016+

  • Prevalence: Increasingly common in new systems

  • Write Blockers: Minimal support, emerging technology
  • Prevalence: Rare currently, will increase

NVMe Form Factors

üìê M.2 Form Factor ‚ñº

M.2 is the most common form factor for NVMe SSDs, especially in laptops and compact desktops.

Physical Characteristics:
  • Size: Small stick-like format (22mm wide, various lengths: 2230, 2242, 2260, 2280, 22110)
  • Most Common: M.2 2280 (22mm x 80mm)
  • Connection: Inserts directly into M.2 slot on motherboard
  • Keying: M-key or B+M-key notches prevent incorrect insertion
Forensic Considerations:
  • Removal: Usually requires removing a small screw
  • Adapter Required: M.2 to USB or M.2 to SATA adapters needed for forensic workstation
  • Write Blocking: Software write blockers often necessary (hardware blockers rare)
  • Documentation: Photograph M.2 slot location before removal
‚ö†Ô∏è
M.2 Confusion Alert

Not all M.2 drives are NVMe! M.2 is a form factor, not an interface. M.2 slots can support:

  • M.2 NVMe: Uses PCIe lanes (very fast)
  • M.2 SATA: Uses SATA protocol (same speed as 2.5" SATA SSD)

Check drive specifications to confirm interface type!

üîå U.2 Form Factor ‚ñº

U.2 (formerly SFF-8639) is an enterprise-focused form factor that looks similar to a 2.5" SATA drive but uses NVMe protocol.

Physical Characteristics:
  • Size: 2.5" drive form factor (same as laptop SATA drives)
  • Connection: Special U.2 cable and connector
  • Hot-Swappable: Yes, designed for server environments
Forensic Considerations:
  • Prevalence: Rare in consumer devices, more common in enterprise servers
  • Adapter Required: U.2 to M.2 or U.2 to PCIe adapters available
  • Write Blocking: Similar challenges to M.2 NVMe
üé¥ PCIe Add-in Card ‚ñº

PCIe Add-in Card NVMe drives plug directly into PCIe expansion slots on the motherboard.

Physical Characteristics:
  • Form: Full-height or half-height PCIe card
  • Slot: Typically uses x4 or x8 PCIe slot
  • Cooling: Often includes heatsink
Forensic Considerations:
  • Removal: Unscrew card from case bracket
  • Connection: May require PCIe to M.2 adapter for forensic workstation
  • Prevalence: Less common, typically in workstations or gaming PCs

NVMe Forensic Challenges

‚ö†Ô∏è
Critical NVMe Challenges for Forensic Examiners
  • ‚ùå Limited Write Blocker Support: Few hardware write blockers support NVMe (as of 2026)
  • ‚ùå Adapter Complications: May require M.2 to USB or M.2 to SATA adapters
  • ‚ö†Ô∏è Software Write Blocking: Often must rely on software write blockers (less trusted legally)
  • ‚ö†Ô∏è Power State Management: NVMe drives have complex power states that can affect acquisition
  • ‚ö†Ô∏è Extremely Fast TRIM: NVMe SSDs execute TRIM commands even faster than SATA SSDs
  • ‚ö†Ô∏è Driver Dependencies: May require specific NVMe drivers on forensic workstation

NVMe Acquisition Strategies

Given the challenges, forensic examiners use these strategies for NVMe drives:

  1. Software Write Blockers: Use trusted software write blocking solutions (FTK Imager, Arsenal Image Mounter)
  2. Specialized Hardware: Invest in NVMe-capable hardware write blockers (expensive but increasingly necessary)
  3. Live Acquisition: In some cases, image the drive while system is running (if encrypted or time-critical)
  4. Document Thoroughly: Explain write-blocking method used and justify approach in reports
  5. Verify Integrity: Multiple hash verifications to ensure no modifications occurred
üîç
Case Example: NVMe Acquisition Challenge

In a 2024 corporate espionage investigation, the suspect's laptop contained a Samsung 980 Pro NVMe SSD (1TB, PCIe 4.0). The forensic lab's hardware write blockers only supported SATA.

Solution: Examiner used FTK Imager's software write blocking with verification. The report explicitly documented the write-blocking method, explained why hardware blocking wasn't available, and provided multiple hash verifications to demonstrate integrity.

Outcome: Evidence was admitted. The examiner's thorough documentation and transparency about limitations enhanced credibility rather than undermining it.

üîå USB (Universal Serial Bus) Interface

What is USB?

Universal Serial Bus (USB) is the most ubiquitous interface for connecting external devices, including storage devices, to computers. USB is found in virtually every investigation because of its universal adoption for flash drives, external hard drives, smartphones, and countless other devices.

USB naming has become confusing over the years. USB 3.0, USB 3.1 Gen 1, and USB 3.2 Gen 1 are all the same thing (5 Gbps). Always check actual speed specifications rather than relying on version numbers alone.

USB Connector Types

üîå Common USB Connectors ‚ñº

  • Description: Rectangular connector, most common on computers
  • Usage: Flash drives, external drives, keyboards, mice
  • Color Coding: Black (USB 2.0), Blue (USB 3.0), Red/Yellow (always-on power)
USB-B (Type-B)
  • Description: Square connector with beveled corners
  • Usage: Printers, scanners, external drive enclosures
  • Variants: Mini-B and Micro-B for smaller devices
USB-C (Type-C)
  • Description: Oval, reversible connector (can be inserted either way)
  • Usage: Modern laptops, smartphones, tablets, external SSDs

Micro-USB
  • Description: Small connector, asymmetric
  • Usage: Older Android phones, cameras, small electronics
  • Forensic Note: Still very common in evidence from 2015-2020 era devices

Forensic Significance of USB

üîç USB Connection Artifacts ‚ñº

Operating systems log extensive information about USB device connections, creating valuable forensic artifacts covered in detail in Weeks 7-8 (Windows Forensics).

Windows USB Artifacts:
  • Registry Keys: USBSTOR, USB, MountedDevices keys track all connected USB devices
  • Information Captured: Vendor ID, Product ID, Serial Number, Device Name, First/Last Connection Times
  • Setupapi.dev.log: Detailed log of device driver installations
  • LNK Files: Shortcuts created when files opened from USB devices
Forensic Value:
  • Proves a specific USB device was connected to the computer
  • Establishes timeline of when device was connected/disconnected
  • Identifies what files were accessed from the device
  • Can reveal data exfiltration to external storage
üîç
Case Example: USB Data Exfiltration

In a 2023 intellectual property theft case, the suspect denied ever using USB drives. Windows Registry analysis revealed:

  • A SanDisk 32GB USB drive (Serial: 123456789ABC) was connected 47 times over 3 months
  • Last connection was 2 days before suspect's resignation
  • LNK files showed proprietary CAD files were opened from the USB drive

Outcome: Confronted with this evidence, the suspect admitted to the theft. The USB drive was later recovered from suspect's home.

‚úÖ
Acquiring USB Storage Devices

  2. ‚úÖ Use Hardware Write Blocker: USB write blockers are readily available and affordable
  3. ‚úÖ Check Write Protection: Some USB drives have physical write-protect switches

üóÑÔ∏è RAID Configurations

What is RAID?

RAID (Redundant Array of Independent Disks) is a technology that combines multiple physical drives into a single logical unit to provide redundancy (data protection), improved performance, or both. RAID is common in servers, workstations, and NAS (Network Attached Storage) devices.

‚ö†Ô∏è
CRITICAL WARNING: RAID and Data Loss

Improperly handling RAID arrays can result in permanent, unrecoverable data loss. RAID configurations must be meticulously documented before disassembling any array. This is not optional‚Äîit's absolutely critical.

Never remove drives from a RAID array without first documenting:

  • RAID level (0, 1, 5, 6, 10, etc.)
  • Disk order and position
  • Stripe size and configuration
  • Controller settings
  • Physical cable connections (photograph!)

  1. üì∏ Photograph Everything: Physical drive arrangement, cable connections, labels
  2. üìù RAID Level: Access RAID controller BIOS to confirm (0, 1, 5, 6, 10, etc.)
  3. üî¢ Disk Order: Number each disk before removal (use labels or tape)

  5. üíæ Array Capacity: Document total array size and individual disk sizes
  6. üîß Controller Settings: Screenshot or write down all RAID controller configuration

üîç
Real Case: The 3-Day RAID Reconstruction

Investigators seized a file server with a 6-disk RAID 5 array. One disk had already failed (RAID 5 tolerates one failure). However, the technician who disassembled the server didn't document disk order or take photographs.

Problem: With 5 remaining disks, there were 5! = 120 possible disk orders. The forensic team had to attempt reconstruction with each permutation.

Result: After 3 days of processing, the correct disk order was identified and data recovered. This delay could have been avoided with 5 minutes of documentation.

Lesson: Documentation is not bureaucracy‚Äîit's essential risk management. Those 5 minutes of documentation would have saved 3 days of work and prevented potential data loss.

RAID Acquisition Strategies

üìã Step-by-Step RAID Acquisition ‚ñº
Phase 1: Documentation (Before Touching Anything)
  1. Access RAID controller BIOS or management interface
  2. Screenshot or photograph all configuration screens
  3. Document RAID level, disk order, stripe size, array capacity
  4. Photograph physical drive arrangement and cable connections
  5. Label each drive clearly (Disk 1, Disk 2, etc.)
  6. Note any failed or degraded disks
Phase 2: Individual Disk Imaging
  1. Remove one disk at a time (maintaining labels)
  2. Connect via hardware write blocker
  3. Create forensic image of individual disk
  4. Calculate and verify hash
  5. Repeat for all disks in array
Phase 3: Array Reconstruction (Forensic Environment)
  1. Use forensic RAID reconstruction tools (X-Ways, EnCase RAID, R-Studio)
  2. Input documented configuration (RAID level, stripe size, disk order)
  3. Reconstruct array from individual disk images
  4. Verify reconstruction succeeded (check file system integrity)
  5. Analyze reconstructed array
‚úÖ
Why Image Individual Disks?

Imaging individual disks before reconstruction provides multiple benefits:

  • Preserves original disks (can be stored as evidence)
  • Enables multiple reconstruction attempts without risk
  • Allows trying different configurations if documentation was incomplete
  • Creates verified copies for court presentation

RAID Reconstruction Tools

Several forensic tools can reconstruct RAID arrays from individual disk images:

  • X-Ways Forensics: Excellent RAID reconstruction capabilities, supports all common levels
  • EnCase: RAID reconstruction module available
  • R-Studio: Specialized in RAID recovery, user-friendly interface
  • mdadm (Linux): Command-line tool for software RAID reconstruction
  • Runtime RAID Reconstructor: Free tool for RAID analysis

üîí Write Blockers & Forensic Acquisition

What are Write Blockers?

Write blockers are devices or software that allow read-only access to storage media, preventing any write operations that could alter evidence. Write blocking is a fundamental requirement for forensically sound evidence acquisition.

üîç
Why Write Blocking is Non-Negotiable

When you connect a storage device to a computer, the operating system automatically:

  • Updates file access timestamps
  • Creates temporary files
  • Writes to log files
  • May trigger TRIM on SSDs
  • Modifies file system metadata

Any modification can be challenged in court as evidence tampering. Write blockers prevent all writes, ensuring evidence integrity.

Types of Write Blockers

üîß Hardware Write Blockers
Physical devices that sit between storage media and forensic workstation
‚úÖ Advantages:
  • OS-independent (works with any OS)
  • More trusted in legal proceedings
  • Cannot be bypassed by software
  • Often include additional features (hashing, imaging)
  • Widely accepted forensic standard
‚ùå Disadvantages:
  • Expensive ($200-$2000+ per unit)
  • Limited interface support (especially NVMe)
  • Requires physical device for each interface type
  • Can introduce slight performance overhead
üíª Software Write Blockers
Software-based write protection through OS-level filtering
‚úÖ Advantages:
  • Free or low cost
  • Support for newer interfaces (NVMe)
  • No additional hardware required
  • Can be updated for new technologies
  • Convenient for field work
‚ùå Disadvantages:
  • Less trusted in legal proceedings
  • OS-dependent (Windows, Linux specific)
  • Potential for software bugs
  • Can be bypassed if not properly configured
  • Requires validation and testing

Purchasing Recommendation

For a forensic lab starting out, prioritize:

  1. SATA/USB combo write blocker (covers 90% of cases)
  2. USB-only write blocker (portable, for field work)
  3. NVMe-capable device (as budget allows, increasingly necessary)

Software Write Blocker Options

üíª Software Write Blocking Solutions ‚ñº
Windows Software Write Blockers:
  • Arsenal Image Mounter: Free, mounts images as read-only volumes
  • FTK Imager: Includes write-blocking when previewing drives
  • Registry Modification: Can disable write access via registry (requires expertise)
Linux Software Write Blockers:
  • hdparm: Set drive to read-only mode
  • blockdev --setro: Set block device read-only
  • Mount with ro flag: Mount file systems read-only
  • Write-blocking distributions: SIFT, CAINE include built-in write blocking
Validation & Testing:

When using software write blockers, you MUST validate they work:

  1. Connect test drive with known hash
  2. Enable software write blocker
  3. Attempt to write to drive
  4. Verify write was blocked
  5. Re-hash drive to confirm no changes
  6. Document validation process
‚ö†Ô∏è
Legal Considerations

Some jurisdictions or courts may be skeptical of software write blockers. Be prepared to:

  • Explain why software write blocking was necessary (e.g., NVMe interface)
  • Demonstrate validation and testing procedures
  • Provide multiple hash verifications
  • Show that no modifications occurred
