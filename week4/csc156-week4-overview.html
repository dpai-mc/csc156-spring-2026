üíø Hard Disk Drives (HDD) Architecture ‚ñº

Physical Structure

  • Platters: Magnetic disks storing data
  • Read/Write Heads: Electromagnetic devices for data access
  • Actuator Arm: Positions heads over correct track
  • Spindle Motor: Rotates platters (5400-15000 RPM)

Data Organization

  • Tracks: Concentric circles on platter surface
  • Sectors: Smallest addressable unit (512 or 4096 bytes)
  • Clusters: Group of sectors (file system unit)
  • Cylinders: Same track across multiple platters

Forensic Advantages

  • ‚úÖ Complete data recovery possible from deleted files
  • ‚úÖ No automatic data destruction
  • ‚úÖ Predictable behavior with mature forensic tools
  • ‚úÖ Slack space contains remnants of previous data

Forensic Challenges

  • ‚ö†Ô∏è Physical damage (head crashes, motor failures)
  • ‚ö†Ô∏è Full-disk encryption (BitLocker, FileVault)
  • ‚ö†Ô∏è Large capacities increase acquisition time
‚ö° Solid State Drives (SSD) Architecture ‚ñº

Physical Structure

  • NAND Flash Memory: Electronic data storage (no moving parts)
  • Types: SLC (fastest), MLC, TLC, QLC (highest density)
  • Controller: Sophisticated processor managing operations
  • Over-Provisioning: Reserved space (7-28%) not visible to OS

Critical SSD Technologies

‚ö†Ô∏è TRIM Command:

OS notifies SSD which blocks are no longer in use after file deletion. SSD immediately erases TRIMmed blocks, making deleted file recovery impossible. Enabled by default on Windows 7+, macOS 10.6.8+, Linux 2.6.33+.

  • Wear Leveling: Distributes writes evenly across cells; data location changes unpredictably
  • Garbage Collection: Automatically erases deleted data in background
  • Hardware Encryption: Many SSDs include built-in AES encryption

Forensic Challenges

  • ‚ùå Deleted files often unrecoverable (TRIM erases within minutes)
  • ‚ùå Unpredictable data remnants due to wear leveling
  • ‚ùå Over-provisioned area inaccessible with standard tools
  • ‚ùå Hardware encryption keys stored in controller

SSD Best Practices

  1. ‚è±Ô∏è Immediate acquisition: Prioritize SSDs in evidence collection
  2. üîí Hardware write blocking: Prevent TRIM commands
  3. ‚ö° Power management: Consider keeping drive powered with write blocker
  4. üìä Expect limited recovery: Adjust expectations for deleted files
  5. üìù Document thoroughly: Record make, model, firmware, TRIM status
üîÑ Boot Processes & Firmware ‚ñº

BIOS vs. UEFI

BIOS (Basic Input/Output System)

  • Legacy firmware standard (since 1975)
  • 16-bit mode, limited to 2.2 TB disks
  • Uses MBR (Master Boot Record) partitioning
  • Boot Process: POST ‚Üí Load MBR ‚Üí Execute bootloader

UEFI (Unified Extensible Firmware Interface)

  • Modern firmware standard (2000s)
  • 32/64-bit mode, supports disks >2.2 TB
  • Uses GPT (GUID Partition Table)
  • Secure Boot capability prevents unauthorized bootloaders
  • Network boot support and graphical interface

Partition Schemes

MBR (Master Boot Record)

  • Located in first 512 bytes of disk
  • Supports up to 4 primary partitions
  • Maximum disk size: 2.2 TB
  • Structure: Boot code (446B) + Partition table (64B) + Signature (2B)

GPT (GUID Partition Table)

  • Part of UEFI standard
  • Supports up to 128 partitions
  • Maximum disk size: 9.4 ZB (zettabytes)
  • Includes backup partition table at end of disk
  • CRC32 checksums for integrity verification

Forensic Artifacts in Boot Process

  • Boot order configuration reveals investigation priorities
  • Hardware settings changes may indicate tampering
  • Firmware timestamps (if available)
  • Secure Boot status affects forensic tool booting
  • EFI System Partition contains bootloader artifacts
üîå Storage Interfaces & Media Types ‚ñº

SATA (Serial ATA)

  • Speeds: SATA I (1.5 Gbps), II (3 Gbps), III (6 Gbps)
  • Usage: Most common for desktop/laptop internal drives
  • Forensics: Well-supported by hardware write blockers

NVMe (Non-Volatile Memory Express)

  • Interface: PCIe lanes (much faster than SATA)
  • Speeds: Up to 7000+ MB/s vs. SATA's 600 MB/s
  • Form Factors: M.2, U.2, PCIe add-in cards
  • Forensics: Write blocking more challenging, fewer compatible blockers

USB (Universal Serial Bus)

  • Versions: USB 2.0 (480 Mbps), 3.x (5-40 Gbps)
  • Common Media: Flash drives, external drives, smartphones
  • Forensics: OS logs USB connections (covered in Weeks 7-8)

RAID Configurations

  • RAID 0: Striping (no redundancy) - all disks required
  • RAID 1: Mirroring - examine any disk
  • RAID 5: Striping with parity - can lose 1 disk
  • RAID 6: Double parity - can lose 2 disks
  • RAID 10: Mirroring + Striping combination
‚ö†Ô∏è RAID Warning: Improperly handling RAID systems can result in permanent data loss. Always document RAID configuration before disassembly.

üìä HDD vs. SSD Comparison

Characteristic HDD SSD
Data Persistence High - deleted data remains until overwritten Low - TRIM/garbage collection erase quickly
Recovery Success High for deleted files Limited after TRIM executes
Imaging Speed Moderate (mechanical limitations) Fast (no moving parts)
Forensic Complexity Lower Higher (wear leveling, TRIM)
Timeline Accuracy Reliable Can be affected by wear leveling
Write Blocking Well-supported More challenging (especially NVMe)
