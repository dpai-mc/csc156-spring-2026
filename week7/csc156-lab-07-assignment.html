Lab Activity 7: Windows Registry and User Artifact Analysis

Week 7 — Windows Forensics I: Registry and User Artifacts

📝 Type: Hands-on Lab with Written Deliverable 💯 Points: 20 ⏱ Estimated Time: 3 hours

📋 Lab Overview

In this lab, you will perform a forensic examination of a provided Windows forensic image. Using the skills and knowledge from Modules 7.1–7.4, you will extract Registry hive files, analyze system-level and user-level artifacts, and compile your findings into a preliminary user activity timeline.

This lab bridges the gap between examination (extracting artifacts) and analysis (interpreting what they mean). You will not only identify artifacts but also explain their forensic significance and correlate them into a coherent investigative narrative.

🎯 Lab Objectives

  • Extract Registry hive files (SAM, SYSTEM, SOFTWARE, NTUSER.DAT) from a forensic image
  • Determine the computer name, time zone, and OS version from system hives
  • Enumerate user accounts from the SAM hive and identify account metadata
  • Extract UserAssist entries to determine program execution history
  • Analyze RecentDocs for recently accessed files
  • Parse Jump Lists to identify application-file associations
  • Examine Recycle Bin artifacts for deleted file evidence
  • Compile a preliminary user activity timeline from all extracted artifacts
📋 CSLO Alignment

Primary CSLO: CSLO 2 — Conduct thorough examinations of computer hard disk drives, and other electronic data storage media to recover information and electronic data.

1 Extract Registry Hive Files 3 points

Open the provided forensic image in Autopsy or FTK Imager and extract the following Registry hive files to your working folder. For each file, also extract the associated transaction log files (.LOG1, .LOG2).

System Hive Files

C:\Windows\System32\config\SAM
C:\Windows\System32\config\SAM.LOG1
C:\Windows\System32\config\SAM.LOG2
C:\Windows\System32\config\SYSTEM
C:\Windows\System32\config\SYSTEM.LOG1
C:\Windows\System32\config\SYSTEM.LOG2
C:\Windows\System32\config\SOFTWARE
C:\Windows\System32\config\SOFTWARE.LOG1
C:\Windows\System32\config\SOFTWARE.LOG2

User Hive Files

C:\Users\<username>\NTUSER.DAT
C:\Users\<username>\NTUSER.DAT.LOG1
C:\Users\<username>\NTUSER.DAT.LOG2
💡 Tip — Identifying User Profiles

Browse the C:\Users\ directory in the forensic image to identify all user profile folders. Extract NTUSER.DAT from every user profile — not just the suspected user. Rename extracted files to avoid confusion (e.g., jsmith_NTUSER.DAT).

Deliverable for Task 1

2 System-Level Artifact Analysis 4 points

Using Registry Explorer and/or RegRipper, analyze the SYSTEM and SOFTWARE hive files to extract the following system identification data.

2a. Determine the Active ControlSet

HKLM\SYSTEM\Select
Value: Current (REG_DWORD) → identifies the active ControlSet number

Record which ControlSet is active (e.g., ControlSet001). All subsequent SYSTEM hive paths should reference this ControlSet.

2b. Extract Computer Name

HKLM\SYSTEM\ControlSet00X\Control\ComputerName\ComputerName
Value: ComputerName (REG_SZ)

2c. Extract Time Zone

HKLM\SYSTEM\ControlSet00X\Control\TimeZoneInformation
Values: TimeZoneKeyName (REG_SZ)
        Bias (REG_DWORD)
        ActiveTimeBias (REG_DWORD)
⚠️ Critical — Time Zone First

Record the time zone before analyzing any other timestamps. You will need it to convert all UTC timestamps to local time throughout the rest of this lab. Note whether Bias and ActiveTimeBias differ (indicating DST was active).

2d. Extract OS Version and Install Date

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion
Values: ProductName, DisplayVersion, CurrentBuild,
        InstallDate (Unix epoch), RegisteredOwner

2e. Extract Last Shutdown Time

HKLM\SYSTEM\ControlSet00X\Control\Windows
Value: ShutdownTime (REG_BINARY → 8-byte FILETIME, UTC)
rip.exe -r SYSTEM -p compname
rip.exe -r SYSTEM -p timezone
rip.exe -r SYSTEM -p shutdown
rip.exe -r SOFTWARE -p winver

Deliverable for Task 2

3 User Account Enumeration (SAM) 3 points

Analyze the SAM hive to enumerate all local user accounts and extract account metadata.

3a. Parse the SAM Hive

rip.exe -r SAM -p samparse

3b. For Each Account, Record

FieldDescription
UsernameAccount name
RIDRelative Identifier (500 = Admin, 501 = Guest, 1001+ = user-created)
Account CreatedWhen the account was created
Last LoginMost recent successful login
Login CountNumber of successful logins
Password Last SetWhen the password was last changed
Account FlagsEnabled/Disabled/Locked status

3c. Map SIDs to Profile Paths

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList\{SID}
Value: ProfileImagePath (REG_EXPAND_SZ)

This mapping connects each account to its NTUSER.DAT file location.

🔍 Analysis Prompt

As you enumerate accounts, look for anything unusual:

  • Recently created accounts with low login counts
  • Accounts with logins at unusual hours
  • Multiple active accounts where only one is expected
  • Accounts with Administrator privileges that seem unnecessary

Note any suspicious findings in your deliverable.

Deliverable for Task 3

4 User Activity Artifact Extraction 5 points

This is the most substantial task. Using the NTUSER.DAT file(s) identified in Task 3, extract the following user activity artifacts.

4a. UserAssist — Program Execution History

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist\{GUID}\Count

RegRipper: rip.exe -r NTUSER.DAT -p userassist

For each entry, record: decoded program path (ROT13 decoded), run count, last execution time (UTC and local), and focus time.

💡 Reminder — ROT13

UserAssist value names are ROT13-encoded. Registry Explorer and RegRipper decode them automatically. If you see paths starting with P:\ instead of C:\, the data has not been decoded yet.

4b. RecentDocs — Recently Accessed Files

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs

RegRipper: rip.exe -r NTUSER.DAT -p recentdocs

Document the most recently accessed files, organized by file extension. Note the key last-write timestamp for each extension subkey.

4c. Additional User Artifacts

Extract and document any relevant findings from the following artifacts (if present in the image):

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\TypedPaths

RegRipper: rip.exe -r NTUSER.DAT -p typedpaths

Record all typed paths. Note any UNC network paths (\\server\share) or removable drive paths (E:\, D:\) that indicate access to external resources.

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\WordWheelQuery

RegRipper: rip.exe -r NTUSER.DAT -p wordwheelquery

Record all search terms in MRU order (most recent first). Note any searches that suggest intent (e.g., searching for sensitive data, deletion methods, or tools).

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RunMRU

RegRipper: rip.exe -r NTUSER.DAT -p runmru

Record all commands in MRU order. Note any administrative tools, network paths, or suspicious commands.

HKCU\Software\Microsoft\Windows\CurrentVersion\Run
HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce

RegRipper: rip.exe -r NTUSER.DAT -p run

Record all autorun entries. Flag any programs running from unusual locations (Temp folders, Downloads, AppData) or with suspicious names mimicking system processes.

Deliverable for Task 4

5 Jump List Analysis Included in Task 4 (5 pts)

Extract the AutomaticDestinations Jump List files from the user profile and parse them using JLECmd.

5a. Locate Jump List Files

C:\Users\<username>\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations\

Export this entire directory from the forensic image to your working folder.

5b. Parse with JLECmd

JLECmd.exe -d "C:\Lab7_Evidence\AutomaticDestinations" --csv "C:\Lab7_Evidence\Output" --csvf jumplists.csv

5c. Analyze Results

Open the CSV output and identify:

  • Which applications accessed which files (AppID → application name)
  • Files accessed from removable volumes (volume type = Removable)
  • Files accessed from network shares (UNC paths)
  • Entry timestamps showing when files were accessed

Deliverable for Task 5

6 Recycle Bin Analysis Included in Task 4 (5 pts)

Examine the Recycle Bin for deleted file evidence.

6a. Locate Recycle Bin Files

C:\$Recycle.Bin\{User SID}\

Use the SID-to-username mapping from Task 3 to identify which user's Recycle Bin to examine. Export the user's Recycle Bin folder from the forensic image.

6b. Parse with RBCmd

RBCmd.exe -d "C:\Lab7_Evidence\Recycle_Bin_SID" --csv "C:\Lab7_Evidence\Output" --csvf recyclebin.csv

6c. Analyze Results

For each deleted file, record:

  • Original file path — Where was the file located before deletion?
  • File size — How large was the file?
  • Deletion timestamp — When was it deleted? (Convert to local time)
  • Forensic significance — Is the filename or location relevant to the investigation?

Deliverable for Task 6

7 Preliminary User Activity Timeline 3 points

Compile all timestamped findings from Tasks 2–6 into a single chronological timeline. This is where individual artifacts become a coherent forensic narrative.

7a. Timeline Format

Create a spreadsheet (or table in your worksheet) with the following columns:

ColumnDescriptionExample
Timestamp (Local)Event time converted to system's local time zoneFeb 28, 2026 7:15:22 AM PST
Timestamp (UTC)Original UTC timestamp2026-02-28 15:15:22 UTC
SourceWhich hive or artifact fileNTUSER.DAT (jsmith)
Artifact TypeWhich specific artifactUserAssist
DescriptionWhat happenedExecuted CCleaner64.exe (count: 5)
SignificanceWhy it mattersAnti-forensics tool execution

7b. Timeline Requirements

  • Include all timestamped artifacts from Tasks 2–6
  • Sort in chronological order (earliest to latest)
  • All timestamps must be normalized to the system's local time zone (identified in Task 2c)
  • Include the UTC timestamp alongside each local time for verification
  • Add a brief analysis paragraph (3–5 sentences) summarizing the overall narrative that emerges from the timeline
🔍 Analysis Guidance

As you build the timeline, look for:

  • Patterns: Clusters of activity at specific times
  • Sequences: Events that logically follow each other (search → access → delete)
  • Anomalies: Activity at unusual hours, rapid file access/deletion sequences
  • Correlations: Multiple artifacts supporting the same conclusion
  • Gaps: Periods of no activity that may be significant

🔧 Tools Quick Reference

Registry Explorer (Eric Zimmerman)
GUI tool for browsing offline Registry hive files. Displays key timestamps, decodes data types, supports transaction log replay and bookmarking.
Usage: File → Load Hive → select hive file
       (Place .LOG1/.LOG2 in same directory for automatic replay)
RegRipper (Harlan Carvey)
Plugin-based command-line tool for automated extraction of specific forensic artifacts from Registry hive files.
Single plugin:  rip.exe -r <hivefile> -p <plugin>
All plugins:    rip.exe -r <hivefile> -a
Save to file:   rip.exe -r <hivefile> -a > output.txt
JLECmd (Eric Zimmerman)
Parses AutomaticDestinations and CustomDestinations Jump List files to extract application-file relationships.
JLECmd.exe -d <jumplist_folder> --csv <output_dir> --csvf <filename>.csv
RBCmd (Eric Zimmerman)
Parses Recycle Bin $I metadata files to extract original file paths, sizes, and deletion timestamps.
RBCmd.exe -d <$Recycle.Bin_SID_folder> --csv <output_dir> --csvf <filename>.csv
Autopsy / FTK Imager
Forensic suites for opening forensic images, browsing file systems, and extracting individual files. Autopsy includes built-in Registry and Recent Activity ingest modules.
Autopsy: Create case → Add data source → Run ingest modules
FTK Imager: File → Add Evidence Item → Image File

Common Issues

NTUSER.DAT is marked with Hidden and System attributes. In Autopsy, it should be visible by default. In FTK Imager, ensure you are browsing the file system view (not just the directory listing). The file is located directly in C:\Users\<username>\.

This means the hive has uncommitted transactions. Place the .LOG1 and .LOG2 files in the same directory as the hive file. Registry Explorer will offer to replay the transaction logs when loading the hive. Click Yes to apply them.

CurrentControlSet is a volatile key that only exists on a running system. In offline analysis, navigate to the Select key and check the Current value to determine which ControlSet00X was active (e.g., if Current = 1, use ControlSet001).

The InstallDate value in the SOFTWARE hive is a Unix epoch timestamp (seconds since January 1, 1970 UTC). You can convert it using:

  • Online: epochconverter.com
  • Python: datetime.utcfromtimestamp(value)
  • PowerShell: [DateTimeOffset]::FromUnixTimeSeconds(value)
