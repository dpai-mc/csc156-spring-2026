🔄 The Transition: From Foundations to Applied OS Forensics

Windows remains the dominant desktop operating system in both enterprise and consumer settings, making it the single most common platform encountered in digital forensic investigations. This week, you shift from understanding how data is stored and acquired to understanding what that data means within the context of a Windows system. The Registry is the key that unlocks that understanding.

🎯 What You Will Learn

By the end of this week, you will be able to:

Core Competencies
  • Navigate the Windows Registry structure — root keys, hive files, keys, subkeys, values, and data types
  • Map logical Registry hives to their physical file locations on an NTFS volume
  • Extract and interpret system-level artifacts: computer name, time zone, OS version, USB device history, user accounts
  • Analyze user-level artifacts from NTUSER.DAT: program execution (UserAssist), file access (RecentDocs), searches (WordWheelQuery), typed paths, and commands (RunMRU)
  • Parse Jump Lists and Recycle Bin artifacts for application-file relationships and deletion evidence
  • Correlate multiple artifact sources into a unified investigative timeline
  • Document findings with proper forensic standards for legal defensibility

🔬 Lab Activity 7: Windows Registry and User Artifact Analysis

20 Points Hands-on Lab + Written Deliverable

Using a provided Windows forensic image, you will perform a complete Registry and user artifact examination. This lab bridges the gap between examination (extracting artifacts) and analysis (interpreting what they mean).

Lab Tasks

  1. Extract Registry hive files (SAM, SYSTEM, SOFTWARE, NTUSER.DAT) from the forensic image
  2. Determine the computer name, time zone, and OS version from system hives
  3. Enumerate user accounts from the SAM hive and identify account metadata
  4. Extract UserAssist entries to determine program execution history
  5. Analyze RecentDocs for recently accessed files
  6. Parse Jump Lists to identify application-file associations
  7. Examine Recycle Bin artifacts for deleted file evidence
  8. Compile a preliminary user activity timeline from all extracted artifacts

📖 Key Terms for This Week

These terms are essential vocabulary for Week 7. You will encounter all of them in the learning modules and lab.

Registry Hive
A logical group of keys, subkeys, and values backed by a single file on disk
Registry Key
A container object (like a folder) that holds subkeys and values
Registry Value
A named data entry within a key, consisting of name, type, and data
Root Key
One of five top-level keys: HKCR, HKCU, HKLM, HKU, HKCC
Last-Write Timestamp
A 64-bit FILETIME on each key recording when it was last modified (UTC)
Hive File
Physical file storing Registry data (e.g., SAM, SYSTEM, NTUSER.DAT)
Transaction Log
.LOG1/.LOG2 files recording pending changes before they are committed
ControlSet
A complete set of system configuration data; Windows maintains multiple for recovery
SAM
Security Account Manager — stores local user accounts, password hashes, and metadata
SID
Security Identifier — unique ID for each user, used as subkey name under HKU
RID
Relative Identifier — numeric account ID (500=Admin, 501=Guest, 1001+=user)
USBSTOR
Registry key recording USB mass storage device connection history
Bias / ActiveTimeBias
Time zone offset values; difference indicates DST status
ProfileList
SOFTWARE hive key mapping SIDs to user profile directory paths
NTUSER.DAT
Per-user Registry hive file; maps to HKCU when the user is logged in
UserAssist
Artifact recording GUI program execution — path, count, focus time, last run
ROT13
Simple letter substitution (rotate 13) used to obfuscate UserAssist paths
RecentDocs
Artifact recording recently opened files organized by extension
MRU List
Most Recently Used — ordered list with most recent item first
WordWheelQuery
Artifact recording search terms typed in Windows Explorer
Persistence Mechanism
Technique ensuring a program survives reboots (e.g., Run keys)
Jump List
Artifact tracking files accessed by specific applications with timestamps
AppID
16-character hex identifier in Jump List filenames mapping to applications
Recycle Bin ($I/$R)
$I = metadata (path, time); $R = file content; organized by user SID
Thumbcache
Database storing thumbnail images that persist after file deletion
Artifact Correlation
Combining evidence from multiple sources to build a complete narrative
Timeline Analysis
Collecting, normalizing, and analyzing timestamped events chronologically
Time Zone Normalization
Converting all timestamps to a common reference for accurate ordering
