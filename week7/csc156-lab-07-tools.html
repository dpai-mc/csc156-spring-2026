📝 Description

Registry Explorer is the gold standard for interactive, offline Registry hive analysis. It provides a graphical interface for browsing extracted hive files with features specifically designed for forensic examiners: key last-write timestamp display, automatic transaction log replay, data type decoding, bookmarking, and advanced search capabilities.

This is the primary tool you will use for manual browsing and verification of Registry artifacts throughout Modules 7.1–7.4 and Lab Activity 7.

📦 Download & Install
⬇ Eric Zimmerman Tools Page 📄 AboutDFIR Tool Page
  1. Navigate to ericzimmerman.github.io and locate Registry Explorer in the tool list
  2. Download the ZIP archive (the page provides direct download links)
  3. Extract the ZIP to a permanent location (e.g., C:\Tools\RegistryExplorer\)
  4. Run RegistryExplorer.exe — no installation required (portable application)
  5. On first run, you may need to install .NET 6 Desktop Runtime if prompted
💡 Tip — Zimmerman Tool Bundle

Eric Zimmerman provides a PowerShell script (Get-ZimmermanTools.ps1) that downloads all of his tools at once. This is the easiest way to get Registry Explorer, JLECmd, RBCmd, AppCompatCacheParser, and RECmd in a single step. The script is linked from his tools page.

🔧 Usage in This Course
  • Load offline hive files (SYSTEM, SAM, SOFTWARE, NTUSER.DAT) with automatic transaction log replay
  • Browse the key tree and view values with decoded data types
  • View key last-write timestamps for every key (critical for forensic analysis)
  • Decode UserAssist entries (automatic ROT13 decoding and binary structure parsing)
  • Bookmark important keys for reporting
  • Export selected data for inclusion in forensic reports

Basic Workflow

# Loading a hive file File → Load Hive → select extracted hive file # Place .LOG1 and .LOG2 files in the same directory # Registry Explorer will offer to replay transaction logs automatically # Navigating to an artifact Left panel: expand key tree → navigate to target path Right panel: view values, data types, and decoded content # Bookmarking a finding Right-click key → Add Bookmark → enter descriptive note

📝 Description

RegRipper is an open-source, plugin-based command-line tool for automated extraction of specific forensic artifacts from Windows Registry hive files. Each plugin targets a known forensic artifact and outputs human-readable results. RegRipper is the workhorse tool for fast, automated artifact extraction — complementing Registry Explorer's interactive browsing.

You will use RegRipper extensively in Modules 7.2–7.3 and Lab Activity 7 for extracting system identification data, user account information, program execution history, and more.

📦 Download & Install
⬇ GitHub: RegRipper 3.0 📦 SourceForge Mirror
  1. Navigate to the RegRipper 3.0 GitHub repository
  2. Click Code → Download ZIP (or clone the repository)
  3. Extract to a permanent location (e.g., C:\Tools\RegRipper\)
  4. The main executable is rip.exe — run from the command prompt or PowerShell
  5. No additional installation is required; plugins are included in the plugins\ directory
⚠️ Note — Perl Dependency

RegRipper is written in Perl. The standalone rip.exe includes a compiled Perl runtime and should work without installing Perl separately. If you encounter issues, ensure you are using the compiled executable from the official distribution, not the raw .pl scripts.

🔧 Usage in This Course

Command Syntax

# Run a single plugin against a hive file rip.exe -r <hive_file> -p <plugin_name> # Run all applicable plugins for the hive type rip.exe -r <hive_file> -a # Save output to a text file rip.exe -r <hive_file> -a > output_report.txt

📝 Description

JLECmd (Jump List Explorer Command Line) parses Windows Jump List files — both AutomaticDestinations and CustomDestinations — and exports the results to CSV for analysis. Jump Lists record which files were recently or frequently accessed by specific applications, providing the critical application-to-file relationship that other artifacts cannot.

You will use JLECmd in Module 7.4 and Lab Activity 7 (Task 5) to extract application-file associations, identify files accessed from removable or network volumes, and build timeline data.

📦 Download & Install
⬇ Eric Zimmerman Tools Page
  1. Download from the Eric Zimmerman tools page (or use the Get-ZimmermanTools.ps1 bundle script)
  2. Extract to your tools directory (e.g., C:\Tools\ZimmermanTools\)
  3. The executable is JLECmd.exe — run from command prompt or PowerShell
  4. Requires .NET 6 Runtime (same as Registry Explorer)
🔧 Usage in This Course
  • Parse AutomaticDestinations Jump List files from user profiles
  • Identify which applications opened which files (via AppID mapping)
  • Detect files accessed from removable drives (volume type = Removable)
  • Detect files accessed from network shares (UNC paths)
  • Extract entry timestamps for timeline correlation

Command Syntax

# Parse a directory of Jump List files and export to CSV JLECmd.exe -d "C:\Evidence\AutomaticDestinations" --csv "C:\Output" --csvf jumplists.csv # Parse a single Jump List file JLECmd.exe -f "C:\Evidence\5f7b5f1e01b83767.automaticDestinations-ms" --csv "C:\Output" # Quiet mode (suppress console output) JLECmd.exe -d "C:\Evidence\AutomaticDestinations" --csv "C:\Output" -q

Key Flags

FlagPurpose
-dDirectory containing Jump List files
-fSingle Jump List file to parse
--csvOutput directory for CSV results
--csvfOutput CSV filename
-qQuiet mode — suppress console output
💡 Jump List Source Location

Extract Jump List files from the forensic image at:
C:\Users\<username>\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations\

📚 Documentation

JLECmd ships with a PDF manual in the Zimmerman tools bundle. AppID-to-application mappings are available in the tool's AppIDs.txt resource file or on Eric Zimmerman's GitHub.

📝 Description

RBCmd (Recycle Bin Command Line) parses the $I metadata files found in the Windows Recycle Bin to extract information about deleted files: the original file path, file size, and deletion timestamp. Results are exported to CSV for analysis.

You will use RBCmd in Module 7.4 and Lab Activity 7 (Task 6) to identify what files were deleted, when, and by whom (using the SID-based Recycle Bin directory structure).

📦 Download & Install
⬇ Eric Zimmerman Tools Page
  1. Download from the Eric Zimmerman tools page (included in the tool bundle)
  2. Extract to your tools directory
  3. The executable is RBCmd.exe
  4. Requires .NET 6 Runtime
🔧 Usage in This Course
  • Parse $I metadata files from a user's Recycle Bin SID folder
  • Extract original file paths, sizes, and deletion timestamps
  • Identify forensically significant deleted files (sensitive documents, tools, evidence)
  • Correlate deletion timestamps with other activity in the timeline

Command Syntax

# Parse a user's Recycle Bin folder and export to CSV RBCmd.exe -d "C:\Evidence\$Recycle.Bin\S-1-5-21-...-1001" --csv "C:\Output" --csvf recyclebin.csv # Parse a single $I file RBCmd.exe -f "C:\Evidence\$IXQF2A1.xlsx"

Key Flags

FlagPurpose
-dDirectory containing $I files (the user's Recycle Bin SID folder)
-fSingle $I file to parse
--csvOutput directory for CSV results
--csvfOutput CSV filename
💡 Recycle Bin Source Location

Extract the user's Recycle Bin folder from the forensic image at:
C:\$Recycle.Bin\{User SID}\
Use the SID-to-username mapping from the SAM hive (Module 7.2) to identify which user's folder to examine.

📝 Description

AppCompatCacheParser extracts ShimCache (AppCompatCache) entries from the SYSTEM Registry hive. ShimCache records programs that the operating system checked for application compatibility. This provides evidence that the OS was aware of a program's existence on the system.

📦 Download & Install
⬇ Eric Zimmerman Tools Page

Included in the Eric Zimmerman tool bundle. Extract and run AppCompatCacheParser.exe from the command line. Requires .NET 6 Runtime.

🔧 Usage in This Course

Command Syntax

# Parse ShimCache from the SYSTEM hive AppCompatCacheParser.exe -f "C:\Evidence\SYSTEM" --csv "C:\Output" --csvf shimcache.csv
⚠️ Important Limitation

ShimCache proves the OS was aware of a program — it does not definitively prove the program was executed. A program can appear in ShimCache simply by being browsed in Explorer. Use ShimCache as supporting evidence alongside UserAssist or Prefetch, not as standalone proof of execution.

📝 Description

RECmd is the command-line companion to Registry Explorer. It allows you to run scripted queries against offline Registry hive files, which is useful for batch processing or when you want to extract specific values without opening the GUI. RECmd uses "batch files" (not Windows batch scripts — RECmd-specific YAML/text files) that define which keys and values to extract.

This tool is optional for this course. RegRipper covers the automated extraction needs for Lab Activity 7. RECmd is included here for students who want additional command-line options.

📦 Download & Install
⬇ Eric Zimmerman Tools Page 📄 AboutDFIR Reference

Distributed with the Eric Zimmerman Registry toolkit (same download as Registry Explorer). The executable is RECmd.exe. Documentation is included in the same PDF manual bundle as Registry Explorer.

🔧 Example Usage
# Run a batch file against a hive RECmd.exe -f "C:\Evidence\NTUSER.DAT" --bn BatchExamples\BatchExample.reb --csv "C:\Output" # Query a specific key RECmd.exe -f "C:\Evidence\SYSTEM" --kn "Select"

⚡ Quick-Start Installation Guide

The fastest way to get all Week 7 tools installed is to use Eric Zimmerman's bundle script for his tools, then download RegRipper separately.

Step 1: Install .NET 6 Desktop Runtime

Most Zimmerman tools require .NET 6. Download from Microsoft's .NET download page if not already installed.

Step 2: Download All Zimmerman Tools

# Option A: Use the PowerShell download script # Download Get-ZimmermanTools.ps1 from ericzimmerman.github.io .\Get-ZimmermanTools.ps1 -Dest C:\Tools\ZimmermanTools # Option B: Download individual ZIPs from ericzimmerman.github.io # Extract each to C:\Tools\ZimmermanTools\

This gives you: Registry Explorer, RECmd, JLECmd, RBCmd, AppCompatCacheParser, and many other useful forensic tools.

Step 3: Download RegRipper

# Download from GitHub https://github.com/keydet89/RegRipper3.0 # Click Code → Download ZIP → Extract to C:\Tools\RegRipper\

Step 4: Verify Installation

# Open a command prompt and test each tool C:\Tools\ZimmermanTools\RegistryExplorer.exe C:\Tools\RegRipper\rip.exe -h C:\Tools\ZimmermanTools\JLECmd.exe -h C:\Tools\ZimmermanTools\RBCmd.exe -h C:\Tools\ZimmermanTools\AppCompatCacheParser.exe -h
💡 Tip — Add to PATH

For convenience, add your tools directories to the system PATH environment variable. This allows you to run tools from any directory without typing the full path.
