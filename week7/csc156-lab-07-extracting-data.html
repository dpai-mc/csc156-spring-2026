Tutorial: Extracting Data from 4orensics.001 Using Autopsy

A step-by-step guide to loading the forensic image, navigating the file system, and exporting the specific files needed for Lab Activity 7.

💿 Image: 4orensics.001 🔬 Tool: Autopsy 📁 Export → External Tools ⏱ ~30 minutes

📋 What This Tutorial Covers

Lab Activity 7 requires you to extract specific files from a Windows forensic image and analyze them with specialized tools (Registry Explorer, RegRipper, JLECmd, RBCmd). Autopsy serves as your file browser and exporter — you will use it to navigate the NTFS file system inside the 4orensics.001 image and export the files you need to your local working folder.

This tutorial walks you through each extraction task in the order you will need them for the lab. Complete this tutorial before beginning the lab analysis tasks.

1
Add 4orensics.001 as a Data Source in Autopsy
Create a case and load the forensic image
Setup
  1. Launch Autopsy and click New Case.
  2. Fill in the case information:
    Case Name: Lab7_YourName Base Directory: C:\ForensicCases\ Case Number: CSC156-W7-L7 Examiner: [Your Full Name]
  3. When the Add Data Source wizard appears, select Disk Image or VM File.
  4. Browse to and select 4orensics.001. Autopsy will automatically follow subsequent segments (.002, .003, etc.) if they exist.
  5. On the Configure Ingest Modules screen, you have two options:

    Disable most modules and enable only Recent Activity and File Type Identification. This is faster and sufficient because you will primarily use external tools (Registry Explorer, RegRipper, JLECmd, RBCmd) for analysis.

    Advantage: Faster ingest time. You can start exporting files sooner.

    Leave all default ingest modules enabled. Autopsy will parse Registry hives, recent activity, and other artifacts automatically.

    Advantage: After ingest, you can use Autopsy's Extracted Content section to cross-check your findings from external tools (e.g., verify OS version, install date, user accounts).

    Disadvantage: Longer ingest time.

  6. Click Next and then Finish. Wait for Autopsy to ingest the image.
  7. Once ingest completes, you will see a Data Sources entry in the left panel representing the 4orensics disk with the full NTFS directory tree.
💡 Tip — You Can Browse While Ingesting

Autopsy allows you to browse the file tree and begin exporting files while ingest is still running. You don't need to wait for ingest to complete before starting the next steps.

Export System Registry Hive Files
SYSTEM, SAM, SOFTWARE — for Lab Tasks 1, 2, and 3
Lab Tasks 1–3

2a. Navigate to the Registry Hive Location

In the left Data Sources / Directory Tree panel, expand:

Data Sources 4orensics (or the name Autopsy assigns) C: Windows System32 config ← This is where the system hives are

2b. Identify the Hive Files

In the file list (right panel), you will see many files. Locate these specific files:

FilePurpose in Lab
SAMUser account enumeration (Task 3)
SAM.LOG1Transaction log for SAM
SAM.LOG2Transaction log for SAM
SYSTEMComputer name, time zone, ControlSet, USB history, shutdown time (Task 2)
SYSTEM.LOG1Transaction log for SYSTEM
SYSTEM.LOG2Transaction log for SYSTEM
SOFTWAREOS version, installed software, network history, ProfileList (Task 2)
SOFTWARE.LOG1Transaction log for SOFTWARE
SOFTWARE.LOG2Transaction log for SOFTWARE

2c. Export the Files

  1. Click on a hive file (e.g., SYSTEM) in the file list to select it.
  2. Right-click → Extract File(s) (the exact wording may vary by Autopsy version).
  3. Choose your working folder: C:\Lab7_Evidence\
  4. Repeat for each of the 9 files listed above.
⚠️ Critical — Keep LOG Files Together

Export the .LOG1 and .LOG2 files to the same directory as their parent hive file. When you later open a hive in Registry Explorer, it will automatically detect and offer to replay the transaction logs — but only if they are in the same folder.

2d. What You Will Do With These Files

FileOpen WithLab Task
SYSTEMRegistry Explorer / RegRipperTask 2: ControlSet, computer name, time zone, shutdown time
SOFTWARERegistry Explorer / RegRipperTask 2: OS version, install date, ProfileList
SAMRegRipper (samparse)Task 3: User account enumeration
🔍 Verification

After exporting, open C:\Lab7_Evidence\ in Windows Explorer and verify you see all 9 files. The hive files (SAM, SYSTEM, SOFTWARE) will have no file extension. The log files will have .LOG1 and .LOG2 extensions.

3
Export User Registry Hive Files (NTUSER.DAT)
Per-user activity data — for Lab Task 4
Lab Task 4

3a. Identify User Profiles

In the Autopsy directory tree, navigate to:

Data Sources 4orensics C: Users ← Expand this to see all user profile folders

You will see multiple folders. Look for real user profiles — typically named after a person or username. Ignore system profiles like Public, Default, and All Users.

💡 How to Identify Real User Profiles

Real user profiles contain an NTUSER.DAT file in their root directory. System profiles (Public, Default) may also contain NTUSER.DAT but are not typically relevant. Focus on profiles with user-recognizable names.

3b. Export NTUSER.DAT for Each User

  1. Click on a user profile folder (e.g., the main user's folder).
  2. In the file list, locate NTUSER.DAT in the root of that profile directory. It is a hidden, system file — Autopsy shows all files regardless of attributes.
  3. Right-click → Extract File(s).
  4. Save to C:\Lab7_Evidence\ and rename the file to include the username:
    # Rename to avoid confusion when multiple users exist username_NTUSER.DAT
  5. Also extract NTUSER.DAT.LOG1 and NTUSER.DAT.LOG2 if present, renaming similarly.
  6. Repeat for every real user profile in the image.
⚠️ Important — Extract ALL User Profiles

Even if you think only one user is relevant, extract NTUSER.DAT from every user profile. Other accounts may contain evidence of unauthorized activity, provide baseline comparisons, or reveal the suspect used a different account.

3c. What You Will Do With These Files

Run RegRipper plugins against each extracted NTUSER.DAT:

# Extract program execution history rip.exe -r username_NTUSER.DAT -p userassist # Extract recently opened files rip.exe -r username_NTUSER.DAT -p recentdocs # Extract Explorer search history rip.exe -r username_NTUSER.DAT -p wordwheelquery # Extract typed paths, Run dialog, and autoruns rip.exe -r username_NTUSER.DAT -p typedpaths rip.exe -r username_NTUSER.DAT -p runmru rip.exe -r username_NTUSER.DAT -p run # Or run ALL plugins at once rip.exe -r username_NTUSER.DAT -a > username_ntuser_report.txt
4
Export Jump List Files
Application-file relationships — for Lab Task 5
Lab Task 5

4a. Navigate to the AutomaticDestinations Folder

In the Autopsy directory tree, expand the following path for each user profile:

C: Users <username> AppData Roaming Microsoft Windows Recent AutomaticDestinations ← Jump List files here

4b. Export the Files

  1. In the file list, you will see files with names like 5f7b5f1e01b83767.automaticDestinations-ms.
  2. Select all .automaticDestinations-ms files in the list.
  3. Right-click → Extract File(s).
  4. Save to C:\Lab7_Evidence\AutomaticDestinations\ (create this subfolder).

4c. Analyze with JLECmd

JLECmd.exe -d "C:\Lab7_Evidence\AutomaticDestinations" --csv "C:\Lab7_Evidence\Output" --csvf jumplists.csv

Open jumplists.csv in Excel or Timeline Explorer and look for:

  • AppID → which application (cross-reference with AppID lookup tables)
  • Target paths → which files were opened
  • Volume type → Fixed, Removable, or Network
  • Timestamps → when files were accessed

Export Recycle Bin Artifacts
Deleted file evidence — for Lab Task 6
Lab Task 6

5a. Navigate to the Recycle Bin

C: $Recycle.Bin S-1-5-21-xxxxxxxxxx-xxxxxxxxxx-xxxxxxxxxx-1001 ← User SID folder S-1-5-21-xxxxxxxxxx-xxxxxxxxxx-xxxxxxxxxx-1002 ← Another user (if any)

5b. Identify Which SID Belongs to Which User

Use the SAM → ProfileList mapping you will create in Lab Task 3. The SID in the Recycle Bin folder name corresponds to the SID you mapped to a username. For example:

# From your SAM/ProfileList analysis: S-1-5-21-...-1001 → jsmith (C:\Users\jsmith) S-1-5-21-...-1002 → temp_user (C:\Users\temp_user)

5c. Export the SID Folder

  1. Right-click the SID folder for the user you want to examine.
  2. Select Extract File(s).
  3. Save to C:\Lab7_Evidence\Recycle_Bin_SID\ (create this subfolder).
  4. The exported folder will contain $I and $R file pairs.

5d. Analyze with RBCmd

RBCmd.exe -d "C:\Lab7_Evidence\Recycle_Bin_SID" --csv "C:\Lab7_Evidence\Output" --csvf recyclebin.csv

Open recyclebin.csv and record for each deleted file:

  • Original path — where the file was located before deletion
  • File size — how large the file was
  • Deletion timestamp — when it was deleted (convert to local time)

❓ Troubleshooting

NTUSER.DAT is marked with Hidden and System NTFS attributes. Autopsy displays all files regardless of attributes, so it should be visible. Make sure you are looking in the root of the user profile directory (e.g., C:\Users\jsmith\), not in a subdirectory. If you still cannot find it, try using Autopsy's search function to search for NTUSER.DAT.

This means the hive has uncommitted transactions. Ensure the .LOG1 and .LOG2 files are in the same folder as the hive file. When Registry Explorer loads the hive, it will detect the log files and offer to replay them. Click Yes. If the log files are missing, Registry Explorer will still open the hive but some recent changes may not be reflected.

$Recycle.Bin is a hidden system folder at the root of the C: drive. In Autopsy, navigate to C:\ and look for it in the file list. It should appear alongside other root-level folders like Windows, Users, Program Files, etc. If you don't see it, make sure you are looking at the root of the volume, not inside another directory.

Large forensic images can take time to ingest. To speed things up:

  • Use Option A (minimal ingest) from Step 1 — disable modules you don't need
  • You can browse the file tree and export files while ingest is still running
  • Ensure the image file is on a local drive (not a network share) for best performance
  • Close other applications to free up RAM

AppData is a hidden folder. Autopsy shows all files regardless of hidden attributes, but sometimes the directory tree can be slow to fully expand. Try clicking directly on the user profile folder and looking in the file list (right panel) rather than expanding the tree. You should see AppData listed as a subdirectory. Click into it and continue navigating to Roaming\Microsoft\Windows\Recent\AutomaticDestinations\.
